## 一、为什么需要 Inertialization？

### 1.1 传统 Crossfade 的根本缺陷

Standard Crossfade（标准交叉渐变）的本质是**姿态空间的线性插值**：

```
p(t) = lerp(p_source, p_target, alpha(t))
```

这种做法有一个根本性的假设：**过渡过程中，骨骼位置应该沿直线从源到目标移动**。但真实物体不是这样运动的——它们有**惯性**。

**具体问题表现：**

| 场景 | Crossfade 表现 | 物理上应该的表现 |
|---|---|---|
| 高速跑动 → 急停 | 骨骼直接从跑到停的位置线性滑动 | 身体因惯性前冲再减速 |
| 攻击被打断 | 手臂从挥砍位瞬移回待机位 | 手臂沿挥砍方向继续甩出再收回 |
| 快速转向 180° | 身体扭曲变形 | 身体先维持原方向再自然转过来 |
| Ragdoll → 恢复 | 从瘫软直接插值到站立 | 身体各部位按不同速度自然收拢 |

### 1.2 Inertialization 的核心思想

**一句话：不做姿态插值，而是让旧动画"自然衰减消亡"（Dead Blending）。**

- 在转换瞬间，**快照**旧动画的当前姿态和速度
- 从快照出发，按照物理惯性**外推**（extrapolate）：每根骨骼沿其当前运动方向继续移动，速度以指数衰减趋向零
- 将外推结果与新动画按时间比例**混合**
- 最终效果：骨骼先"甩出去"再自然回到新动画的轨道

## 二、核心算法 — Dead Blending 数学详解

> 🔧 **以下所有公式和代码均来自引擎源码验证**
> 源码路径：`Engine/Plugins/Experimental/UAF/UAFAnimGraph/Source/UAFAnimGraph/Private/EvaluationVM/Tasks/DeadBlending.cpp`（基于 UE 5.6 源码验证，行号可能随版本变动）

### 2.1 算法三阶段

整个 Inertialization 过程分为三个阶段：

```
┌──────────────────┐     ┌──────────────────────┐     ┌────────────────────┐
│  Phase 1:        │     │  Phase 2:            │     │  Phase 3:          │
│  Transition      │ ──► │  Apply (每帧)        │ ──► │  Complete          │
│  (转换时刻，1次) │     │  外推 + 混合          │     │  衰减完毕，停用     │
│                  │     │  (持续 BlendDuration)│     │                    │
│  • 快照姿态      │     │  • 指数衰减外推      │     │  • 清空状态        │
│  • 计算速度      │     │  • 与新动画混合      │     │  • 释放内存        │
│  • 拟合半衰期    │     │  • 旋转方向保护      │     │                    │
└──────────────────┘     └──────────────────────┘     └────────────────────┘
```

### 2.2 Phase 1：Transition（快照 + 速度计算 + 半衰期拟合）

#### Step 1：快照当前姿态

记录旧动画在转换时刻每根骨骼的 Translation、Rotation、Scale：

```cpp
// 源码：DeadBlending.cpp Line 247-253
const FVector SrcTranslationCurr = Curr.Translations[LODBoneIdx];
const FQuat SrcRotationCurr = Curr.Rotations[LODBoneIdx];
const FVector SrcScaleCurr = Curr.Scales3D[LODBoneIdx];

Source.Translations[LODBoneIdx] = SrcTranslationCurr;
Source.Rotations[LODBoneIdx] = SrcRotationCurr;
Source.Scales3D[LODBoneIdx] = SrcScaleCurr;
```

#### Step 2：计算速度（有限差分法）

利用 **当前帧（Curr）** 和 **前一帧（Prev）** 的差值除以 DeltaTime 得到速度：

**Translation 速度：**
```
TranslationVelocity = clamp((CurrTranslation - PrevTranslation) / DeltaTime, MaxTranslationVelocity)
```

**Rotation 速度（在旋转向量空间）：**
```
RotationDiff = (CurrRotation × Inv(PrevRotation)).GetShortestArcWith(Identity)
RotationVelocity = clamp(RotationDiff.ToRotationVector() / DeltaTime, MaxRotationVelocity)
```

**Scale 速度（在对数空间）：**
```
ScaleDiff = CurrScale / PrevScale   (逐分量除)
ScaleVelocity = clamp(log(ScaleDiff) / DeltaTime, MaxScaleVelocity)
```

源码实现：
```cpp
// 源码：DeadBlending.cpp Line 261-268
const FVector TranslationDiff = SrcTranslationCurr - SrcTranslationPrev;
const FQuat RotationDiff = (SrcRotationCurr * SrcRotationPrev.Inverse())
    .GetShortestArcWith(FQuat::Identity);
const FVector ScaleDiff = VectorDivMax(SrcScaleCurr, SrcScalePrev);

SourceBoneTranslationVelocities[LODBoneIdx] = 
    (FVector3f)(TranslationDiff / DeltaTime)
    .GetClampedToMaxSize(Parameters.MaximumTranslationVelocity);
SourceBoneRotationVelocities[LODBoneIdx] = 
    (FVector3f)(RotationDiff.ToRotationVector() / DeltaTime)
    .GetClampedToMaxSize(Parameters.MaximumRotationVelocity);
SourceBoneScaleVelocities[LODBoneIdx] = 
    (FVector3f)(VectorLogMax(ScaleDiff) / DeltaTime)
    .GetClampedToMaxSize(Parameters.MaximumScaleVelocity);
```

> 💡 **关键设计点：** 速度被 clamp 到最大值，防止帧率波动导致的异常速度。默认值：Translation 500 cm/s，Rotation 360°/s（内部转弧度），Scale 4.0。

#### Step 3：自适应衰减半衰期拟合

这是算法最巧妙的部分。每根骨骼的每个分量（x/y/z）都有**独立计算的半衰期**：

```
DecayHalfLife = clamp(
    HalfLife × (SrcDstDiff / SrcVelocity),
    HalfLifeMin,
    HalfLifeMax
)
```

源码实现：
```cpp
// 源码：DeadBlending.cpp Line 154-172
static inline float ComputeDecayHalfLifeFromDiffAndVelocity(
    const float SrcDstDiff,
    const float SrcVelocity,
    const float HalfLife,
    const float HalfLifeMin,
    const float HalfLifeMax,
    const float Epsilon = UE_KINDA_SMALL_NUMBER)
{
    return FMath::Clamp(
        HalfLife * (SrcDstDiff / ClipMagnitudeToGreaterThanEpsilon(SrcVelocity, Epsilon)), 
        HalfLifeMin, 
        HalfLifeMax);
}
```

**直觉理解（非常重要！）：**

| 速度 vs 差异 | SrcDstDiff / SrcVelocity | 半衰期 | 物理含义 |
|---|---|---|---|
| 速度方向朝向目标 | 正且大 | 大（衰减慢） | 旧动画本来就在往目标走，让它自然走 |
| 速度方向背离目标 | 负 | 被 clamp 到 Min（快速衰减） | 方向反了，赶紧纠正 |
| 速度很小，差异大 | 大 | 大 | 几乎不动但离目标远，慢慢过渡 |
| 速度很大，差异小 | 小 | 小 | 快到了，快速收敛 |

### 2.3 Phase 2：Apply（外推 + 混合）

每帧执行，直到 `TimeSinceTransition > BlendDuration`。

#### 外推公式（核心！）

**Translation 外推：**
```
C = ln(2) / DecayHalfLife           (衰减常数)
ExtrapolatedTranslation = SourceTranslation + (Velocity / C) × (1 - exp(-C × Time))
```

**物理意义：**
- `v(t) = v0 × exp(-C×t)` — 速度以指数方式衰减
- 对速度积分得到位移：`∫v dt = (v0/C) × (1 - exp(-C×t))`
- 当 t→∞ 时，总位移极限为 `v0/C`

源码实现：
```cpp
// 源码：DeadBlending.cpp Line 93-109
static inline FVector ExtrapolateTranslation(
    const FVector Translation,
    const FVector3f Velocity,
    const FVector3f DecayHalflife,
    const float Time,
    const float Epsilon = UE_SMALL_NUMBER)
{
    if (Velocity.SquaredLength() > Epsilon)
    {
        const FVector3f C = VectorDivMax(Ln2, DecayHalflife, Epsilon);
        return Translation + (FVector)(VectorDivMax(Velocity, C, Epsilon) 
            * (FVector3f::OneVector - VectorInvExpApprox(C * Time)));
    }
    else
    {
        return Translation;  // 速度为零直接返回
    }
}
```

**Rotation 外推（在旋转向量空间）：**
```
C = ln(2) / DecayHalfLife
ExtrapolatedRotation = MakeFromRotationVector((Velocity / C) × (1 - exp(-C × Time))) × SourceRotation
```

**Scale 外推（在对数空间）：**
```
C = ln(2) / DecayHalfLife
ExtrapolatedScale = exp((Velocity / C) × (1 - exp(-C × Time))) × SourceScale
```

#### 混合公式

```
Alpha = 1.0 - AlphaToBlendOption(TimeSinceTransition / BlendDuration, BlendMode)

FinalTranslation = lerp(DestTranslation, ExtrapolatedTranslation, Alpha)
FinalRotation    = RotationVector(Diff × Alpha) × DestRotation     // 旋转向量空间 slerp
FinalScale       = eerp(DestScale, ExtrapolatedScale, Alpha)       // 指数插值
```

其中 `eerp`（exponential interpolation）的公式为：
```
eerp(a, b, t) = pow(a, 1-t) × pow(b, t)
```

源码实现：
```cpp
// 源码：DeadBlending.cpp Line 377-379
const float Alpha = 1.0f - FAlphaBlend::AlphaToBlendOption(
    FMath::Clamp(TimeSinceTransition / FMath::Max(BlendDuration, UE_SMALL_NUMBER), 0.0f, 1.0f),
    BlendMode, CustomBlendCurve);
```

#### 旋转方向一致性保护

这是一个容易被忽视但非常重要的细节。在混合过程中，外推的旋转可能会导致突然"翻转方向"，引起视觉跳变。算法通过维护一个 `BoneRotationDirections` 数组来解决：

```cpp
// 源码：DeadBlending.cpp Line 427-439
// We need to enforce that the blend of the rotation doesn't suddenly "switch sides"
FQuat RotationDiff = ExtrapolatedRotation * Dest.Rotations[LODBoneIdx].Inverse();
RotationDiff.EnforceShortestArcWith((FQuat)BoneRotationDirections[LODBoneIdx]);

// Update BoneRotationDirections to match our current path
BoneRotationDirections[LODBoneIdx] = (FQuat4f)RotationDiff;

Dest.Rotations[LODBoneIdx] = FQuat::MakeFromRotationVector(
    RotationDiff.ToRotationVector() * Alpha) * Dest.Rotations[LODBoneIdx];
```

### 2.4 ISPC 向量化加速

UE5 的 Inertialization 实现还包含了 **ISPC（Intel SPMD Program Compiler）** 版本，对骨骼遍历进行 SIMD 向量化加速：

```c
// 源码：DeadBlending.ispc
export void AnimNextDeadBlendingApply(
    uniform double DestTranslations[],
    uniform double DestRotations[],
    ...
    const uniform int LODBoneNum,
    const uniform float Alpha,
    const uniform float TimeSinceTransition)
{
    foreach(BoneIdx = 0 ... LODBoneNum)
    {
        // SIMD 并行处理所有骨骼
        const Vector3d ExtrapolatedTranslation = ExtrapolateTranslation(
            Vector3dLoad(SourceTranslations, BoneIdx),
            Vector3fLoad(SourceBoneTranslationVelocities, BoneIdx),
            Vector3fLoad(SourceBoneTranslationDecayHalfLives, BoneIdx),
            TimeSinceTransition);
        ...
    }
}
```

> 💡 **性能意义：** ISPC 版本可以利用 AVX-512 等 SIMD 指令集，一次处理多根骨骼的外推计算，对于骨骼数量多的怪物（如 100+ 骨骼的四足怪）有显著性能提升。

## 三、Per-Bone（逐骨骼）特性详解

### 3.1 所有计算都是 Per-Bone 的

`FDeadBlendingState` 为**每根骨骼**分别维护独立的状态：

```cpp
// 源码：Inertialization.h (UAF)
struct FDeadBlendingState
{
    TArray<FQuat4f> BoneRotationDirections;         // 旋转方向一致性
    FTransformArraySoAHeap SourcePose;              // 快照的源姿态
    TArray<FVector3f> SourceBoneTranslationVelocities;      // 位移速度
    TArray<FVector3f> SourceBoneRotationVelocities;         // 旋转速度
    TArray<FVector3f> SourceBoneScaleVelocities;            // 缩放速度
    TArray<FVector3f> SourceBoneTranslationDecayHalfLives;  // 位移衰减半衰期
    TArray<FVector3f> SourceBoneRotationDecayHalfLives;     // 旋转衰减半衰期
    TArray<FVector3f> SourceBoneScaleDecayHalfLives;        // 缩放衰减半衰期
};
```

> 💡 **关键意义：** 半衰期是 `FVector3f`（xyz 三个分量独立），意味着同一根骨骼在不同轴向上的衰减速度可以不同。比如手臂在水平方向高速运动但垂直方向几乎不动，那水平方向会有较大半衰期（慢衰减），垂直方向会有较小半衰期（快衰减）。

### 3.2 为什么 Per-Bone 很重要

在怪物动画中，不同骨骼在同一转换时刻的运动状态差异巨大：

```
四足怪物从奔跑 → 急停：

骨盆（Pelvis）：高速前进 → 需要大半衰期 → 自然前冲减速
前腿（FrontLegs）：高频交替摆动 → 需要适中半衰期 → 让步态自然收住
尾巴（Tail）：随身体摇摆 → 需要大半衰期 → 让尾巴惯性甩动
头部（Head）：相对稳定 → 需要小半衰期 → 快速对齐新方向
```

如果用统一的衰减参数，要么头部过度甩动，要么尾巴过早停止。Per-Bone 自适应半衰期完美解决了这个问题。

## 四、引擎中的两条实现路径

### 4.1 路径 A：传统 AnimGraph（BlendStack / MotionMatching）

这是当前生产项目中最常用的路径。

**关键属性（🔧 来自源码）：**

| 属性名 | 类型 | 默认值 | 说明 |
|---|---|---|---|

| `InertialBlendNodeTag` | `FName` | `NAME_None` | 指定特定 Inertialization 节点的 Tag |
| `BlendTime` | `float` | `0.2f` | 混合持续时间（秒） |

| `BlendProfile` | `TObjectPtr<UBlendProfile>` | `nullptr` | Per-bone 权重配置 |

**触发流程（🔧 源码路径）：**

```
FAnimNode_BlendStack::BlendTo()
  → 如果 bUseInertialBlend == true:
    → 调用 RequestInertialBlend() 静态函数
    → 通过 IInertializationRequester 消息接口获取 requester
    → 创建 FInertializationRequest（Duration, BlendProfile, BlendMode, Tag）
    → requester->RequestInertialization(Request)
    → 将 BlendStack 自身的 BlendTime 设为 0（混合交给 Inertialization 节点处理）
```

```cpp
// 源码：AnimNode_BlendStack.cpp
static void RequestInertialBlend(const FAnimationUpdateContext& Context, 
    float BlendTime, const UBlendProfile* BlendProfile, 
    EAlphaBlendOption BlendOption, FName InertialBlendNodeTag)
{
    if (BlendTime > 0.0f)
    {
        auto* InertializationRequester = 
            Context.GetMessage<UE::Anim::IInertializationRequester>();
        if (InertializationRequester)
        {
            FInertializationRequest Request;
            Request.Duration = BlendTime;
            Request.BlendProfile = BlendProfile;
            Request.bUseBlendMode = true;
            Request.BlendMode = BlendOption;
            Request.Tag = InertialBlendNodeTag;
            InertializationRequester->RequestInertialization(Request);
        }
        else
        {
            FAnimNode_Inertialization::LogRequestError(Context, ...);
        }
    }
}
```

> ⚠️ **常见错误：** 如果 Anim Graph 中没有放置 Inertialization 节点但开启了 `bUseInertialBlend`，会在 Output Log 中输出错误日志（`LogRequestError`）。

#### Trait 2：`FBlendInertializerTrait`（混合转换器）

**显示名称：** "Blend Inertializer"

将普通的混合操作转换为 Inertialization 混合。它做的事情：
1. 在 `OnBlendTransition` 时创建 `FAnimNextInertializationRequestEvent`
2. 通过 `Context.RaiseOutputTraitEvent()` 向图的上游冒泡
3. 劫持正常的 BlendTime 返回 0（因为混合由 Dead Blending Trait 处理）

```cpp
// 源码：BlendInertializer.cpp Line 38-56
void FBlendInertializerCoreTrait::OnBlendTransition(...) const
{
    IDiscreteBlend::OnBlendTransition(Context, Binding, OldChildIndex, NewChildIndex);
    
    TTraitBinding<IInertializerBlend> InertializerBlendTrait;
    Binding.GetStackInterface(InertializerBlendTrait);
    
    const float BlendTime = InertializerBlendTrait.GetBlendTime(Context, NewChildIndex);
    if (BlendTime <= 0.0f) return;  // 禁用
    
    // 创建 Inertialization 请求事件
    auto Event = MakeTraitEvent<FAnimNextInertializationRequestEvent>();
    Event->Request.BlendTime = BlendTime;
    Context.RaiseOutputTraitEvent(Event);  // 向上游冒泡
}
```

#### Trait 3：`IInertializerBlend`（接口）

提供 `GetBlendTime()` 方法，返回 Inertialization 的混合时间。默认返回 0（禁用）。

**UAF 中的事件传播流程：**

```
BlendInertializerTrait::OnBlendTransition()
  → 创建 FAnimNextInertializationRequestEvent（BlendTime）
  → Context.RaiseOutputTraitEvent(Event)  // 向上游冒泡
  → 找到 InertializationDeadBlendTrait
    → OnInertializationRequestEvent()
      → 如果没有 pending 请求 或 新请求 BlendTime 更短：
        → 存储为 PendingRequest
      → Event.MarkHandled()
```

## 五、参数调优完全指南

### 5.1 核心参数关系图

```
                  ┌───────────────────────────────────────┐
                  │         参数相互关系                     │
                  │                                         │
                  │  DefaultBlendDuration (0.25s)           │
                  │    └─ 控制 "混合窗口" 的总时长            │
                  │    └─ 值越大 = 过渡越慢越柔和             │
                  │                                         │
                  │  ExtrapolationHalfLife (1.0s)            │
                  │    └─ 控制 "惯性甩出" 的距离和速度         │
                  │    └─ 值越大 = 骨骼甩得越远               │
                  │                                         │
                  │  ExtrapolationHalfLifeMin (0.05s)        │
                  │    └─ 控制 "快速纠正" 时的最小衰减时间     │
                  │    └─ 值越小 = 反方向运动时纠正越快        │
                  │                                         │
                  │  ExtrapolationHalfLifeMax (1.0s)         │
                  │    └─ 控制 "自然滑行" 时的最大衰减时间     │
                  │    └─ 值越大 = 顺方向运动时滑行越远        │
                  │                                         │
                  │  MaximumTranslationVelocity (500 cm/s)  │
                  │    └─ 安全阀：防止异常高速导致骨骼飞出     │
                  │                                         │
                  │  MaximumRotationVelocity (360°/s)       │
                  │    └─ 安全阀：防止旋转过快导致姿态崩溃     │
                  └───────────────────────────────────────┘
```

### 5.2 怪物体型分级调参表

| 参数 | 小型怪物（史莱姆/蝙蝠） | 中型怪物（兽人/猎犬） | 大型怪物（巨龙/Boss） | 调参直觉 |
|---|---|---|---|---|
| `DefaultBlendDuration` | 0.1 ~ 0.15s | 0.2 ~ 0.3s | 0.3 ~ 0.5s | 体型越大，过渡越慢 |
| `ExtrapolationHalfLife` | 0.3 ~ 0.5 | 0.8 ~ 1.2 | 1.5 ~ 2.5 | 体型越大，惯性越强 |
| `ExtrapolationHalfLifeMin` | 0.02 ~ 0.05 | 0.05 ~ 0.1 | 0.1 ~ 0.15 | 小值 = 快速纠正 |
| `ExtrapolationHalfLifeMax` | 0.3 ~ 0.5 | 0.8 ~ 1.2 | 1.5 ~ 2.0 | 大值 = 长滑行 |
| `MaxTranslationVelocity` | 300 | 500 | 800 | 匹配怪物最大移动速度 |
| `MaxRotationVelocity` | 540 | 360 | 180 | 小怪灵活，大怪笨重 |

### 5.3 场景化调参建议

#### 场景 1：怪物攻击被打断 → Hit React

这是 Inertialization 最佳应用场景。

```
攻击动画：手臂正在高速挥砍
打断时刻：手臂有很大的旋转速度和位移速度

Crossfade：手臂直接从挥砍位线性插值到受击位 → 生硬
Inertialization：手臂先沿挥砍方向继续甩出一点 → 然后自然收回到受击姿态 → 自然

推荐参数：
  DefaultBlendDuration = 0.15s（受击需要快速响应）
  ExtrapolationHalfLife = 0.8s（适度惯性）
  ExtrapolationHalfLifeMin = 0.03s（反方向快速纠正）
```

#### 场景 2：高速奔跑 → 急停

```
推荐参数：
  DefaultBlendDuration = 0.25s（给足过渡时间）
  ExtrapolationHalfLife = 1.5s（大惯性，模拟刹车距离）
  MaxTranslationVelocity = 设为怪物最大移速（如 600 cm/s）

关键：骨盆会自然前冲再停下，腿部会有自然的制动步态
```

#### 场景 3：Ragdoll 恢复到站立

```
推荐参数：
  DefaultBlendDuration = 0.4 ~ 0.6s（给充分时间恢复）
  ExtrapolationHalfLife = 2.0s（很大的惯性，因为 Ragdoll 状态速度混乱）
  MaxTranslationVelocity = 200（限制，防止从 Ragdoll 恢复时骨骼乱飞）
  MaxRotationVelocity = 180（限制旋转）
```

|---|---|---|

| `QuinticInOut` | 五次缓入缓出 | 极端的缓入缓出 |
| `CircularIn` | 圆弧缓入 | 开始快结束慢 |
| `CircularOut` | 圆弧缓出 | 开始慢结束快 |

## 八、Inertialization vs Crossfade vs Blend Stack 对比

| 维度 | Standard Crossfade | Inertialization / Dead Blending | Blend Stack |
|---|---|---|---|
| **算法本质** | 两个姿态的线性/曲线插值 | 旧姿态指数衰减外推 + 与新动画混合 | 多个动画层的加权混合 |
| **物理真实感** | ❌ 无惯性 | ✅ 天然的物理惯性 | ❌ 无惯性 |
| **Per-Bone 适应** | 有限（BlendProfile 调权重） | ✅ 自动（每骨骼独立半衰期） | 有限 |
| **性能开销** | 最低 | 中（需要维护快照+外推） | 中高（多层混合） |
| **内存开销** | 0 | 每角色额外存储 FDeadBlendingState | 每层一个完整 pose |
| **适用场景** | 低速/平缓过渡 | 高速中断、急停急转、Ragdoll恢复 | 需要同时播放多个动画 |
| **调参复杂度** | 低（只有 BlendTime） | 中（半衰期 + 最大速度 + 混合模式） | 中（MaxActiveBlends + Per-layer 配置） |

## 十、进阶话题

### 10.1 零速度 Transition（Single Pose）

当转换时刻只有当前帧的快照而没有前一帧时（`PrevPose` 为空），算法假设速度为零：

```cpp
// 源码：InertializationDeadBlend.cpp Line 82-108
else if (InstanceData->PrevPose.IsEmpty())
{
    // 只有单帧快照，假设零速度
    Context.AppendTask(FAnimNextDeadBlendingTransitionTask::Make(
        &InstanceData->State, 
        &InstanceData->CurrPose,  // 只传 Curr，不传 Prev
        Parameters));
}
```

此时所有骨骼的速度为零，半衰期统一设为 `HalfLifeMin`：

```cpp
// 源码：DeadBlending.cpp Line 341-356 (Transition with single pose)
SourceBoneTranslationVelocities[LODBoneIdx] = FVector3f::ZeroVector;
SourceBoneTranslationDecayHalfLives[LODBoneIdx] = 
    Parameters.ExtrapolationHalfLifeMin * FVector3f::OneVector;
```

**实践意义：** 这意味着即使在角色刚生成、还没有历史帧数据的情况下，Inertialization 也不会崩溃——它会退化为一种简化的过渡模式。

### 10.2 请求优先级

当短时间内收到多个 Inertialization 请求时（比如连续两次状态切换），系统采用**更短 BlendTime 优先**的策略：

```cpp
// 源码：InertializationDeadBlend.cpp Line 40-45
if (!InstanceData->bRequestPending ||
    Event.Request.BlendTime < InstanceData->PendingRequest.BlendTime)
{
    InstanceData->bRequestPending = true;
    InstanceData->PendingRequest = Event.Request;
}
```

**含义：** 快速连续的打断事件（如怪物被连续受击）会自动选择最快的过渡速度，确保响应及时。

### 10.3 Scale 的对数空间处理

Scale 的外推不是在线性空间做的，而是在**对数空间**：

```
ScaleVelocity = log(CurrScale / PrevScale) / DeltaTime
ExtrapolatedScale = exp((Velocity / C) × (1 - exp(-C × Time))) × SourceScale
```

混合用的是**指数插值（eerp）**而非线性插值：

```
eerp(a, b, t) = pow(a, 1-t) × pow(b, t)
```

这保证了 Scale 的变化是乘性的（比如从 1.0 缩放到 2.0 和从 2.0 缩放到 4.0 被视为相同的"距离"），避免了线性插值在缩放空间中的不自然行为。

## 十一、性能注意事项

### 11.1 内存开销

每个使用 Inertialization 的角色实例需要额外存储：

```
FDeadBlendingState 内存估算（100 骨骼）：
  BoneRotationDirections:           100 × 16 bytes = 1.6 KB
  SourcePose (Translation+Rot+Scale): 100 × 80 bytes ≈ 8.0 KB
  6 × Velocity/HalfLife arrays:      600 × 12 bytes = 7.2 KB
  CurrPose + PrevPose:               2 × 8 KB ≈ 16 KB
  ────────────────────────────────────────────────────
  总计约 33 KB / 角色实例
```

对于同时有 50 个怪物活跃的场景，总额外内存约 1.6 MB——完全可接受。

### 11.2 CPU 开销

- **Transition（一次性）：** 遍历所有骨骼计算速度和半衰期，约 O(N) 其中 N 为骨骼数
- **Apply（每帧）：** 遍历所有骨骼做外推和混合，约 O(N)
- **ISPC 加速：** 启用 ISPC 后可以利用 SIMD 指令并行处理，在骨骼数量多时显著降低开销

### 11.3 LOD 策略

建议根据距离摄像机的距离设置不同的 Inertialization 策略：

```
LOD0 (近距离 < 10m)：完整 Inertialization，所有参数按上述调优
LOD1 (中距离 10-25m)：简化参数（降低 MaxVelocity 以减少外推计算量）
LOD2 (远距离 > 25m)：关闭 Inertialization，回退到 Standard Crossfade
```

### 快速上手 Checklist

- [ ] 在 Anim Graph 中 State Machine 输出后放置 Inertialization 节点

- [ ] 为不同怪物体型创建不同的参数预设
- [ ] 使用 Rewind Debugger 的 Inertialization Track 调试

### 关键记忆点

1. **Inertialization ≠ Crossfade**：不是插值两个姿态，而是让旧动画"自然死亡"（Dead Blending）
2. **Per-Bone 自适应**：每根骨骼、每个分量（xyz）的衰减速度独立计算
3. **半衰期拟合公式**：`HalfLife × (SrcDstDiff / SrcVelocity)` — 速度朝向目标则慢衰减，背离则快纠正
4. **外推数学**：`位移 = (v0/C) × (1 - exp(-C×t))`，速度以指数衰减
5. **ISPC 加速**：大量骨骼时有 SIMD 并行优化
6. **两条路径**：传统 AnimNode（生产就绪）和 UAF Trait（实验性）
7. **调参核心**：`DefaultBlendDuration` 控制总时长，`ExtrapolationHalfLife` 控制惯性强度
