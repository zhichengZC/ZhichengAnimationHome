## 第 0 章 前置知识回顾 [L0 核心]

进入 VAE 之前，我们把需要用到的基础快速过一遍。**如果某条你不熟悉，先暂停回去补**。

### 0.1 神经网络基础（2 分钟自查）

一个神经网络做的事本质上只有三步：

```
输入 x  →  网络（一堆矩阵乘法 + 非线性激活）  →  输出 y'
               ↑
         参数 W、b（就是 "模型学到的东西"）

损失 L = 衡量 y' 和真实 y 差多远（比如 MSE）

反向传播：∂L/∂W 告诉我们"把 W 朝哪个方向调能让 L 下降"
梯度下降：W ← W − η · ∂L/∂W（η 是学习率）
```

**关键事实**：反向传播要求从 L 到 W **每一步都是可微的**（有导数）。这一点在讲第 4 章重参数化时会变成"拦路虎"，先记住。

### 0.2 三种学习范式的区别

| 范式 | 数据形式 | 训练目标 | VAE 属于谁？ |
|---|---|---|---|
| **监督学习**（Supervised） | (x, y) 配对 | 预测 y | ❌ |
| **无监督学习**（Unsupervised） | 只有 x，没有标签 | 发现结构 | ✅（广义）|
| **自监督学习**（Self-Supervised） | 从 x 本身构造"伪标签" | 预测 x 的某个部分 | ✅（精确）|
| **强化学习**（Reinforcement） | 环境反馈奖励 | 最大化累计回报 | ❌ |

**VAE 的本质**：给它一堆没有标签的数据（比如 10 万段动画，**不告诉它哪段是走哪段是跑**），它自己学会"压缩 → 重建"这张数据的结构。"伪标签"就是 x 自己——网络的目标是复原输入。

### 0.3 几个符号约定

| 符号 | 含义 |
|---|---|
| `x` | 原始数据（比如一张图、一段动画、一个 100 维向量） |
| `z` | 潜变量（Latent Variable），x 被"压缩"后的低维表示 |
| `E(x)` 或 `q(z|x)` | Encoder：把 x 映射到 z |
| `D(z)` 或 `p(x|z)` | Decoder：把 z 还原回 x' |
| `θ`、`φ` | 网络参数（encoder 参数 φ、decoder 参数 θ，仅推导时用） |

**术语中英对照小贴士**：
- Autoencoder = **自编码器**（"自"指的是输入和目标都是 x 自己）
- Latent = **潜的、隐藏的**（潜变量 = 隐藏在数据背后的低维表达）
- Variational = **变分的**（来自"变分推断 Variational Inference"，第 3 章再细讲）

## 第 1 章 为什么要学 VAE [L0 核心]

### 1.1 生成式 AI 的两大主流家族

你已经听过 "AI 生图"、"AI 生动画" 这类词。它们背后其实不是一个算法，而是**一整个家族**。粗分成两大流派：

```
生成模型家族
├── 显式概率派（Likelihood-based）
│   ├── VAE       ← 本篇主角
│   ├── Flow      ← 精确可逆，但结构受限
│   └── Diffusion ← 2020 年后霸主，扩散到去噪
│
└── 隐式对抗派（Adversarial）
    └── GAN       ← 4/21 学过（基础），质量锐利但难训
```

**直觉区别**（后面章节展开）：

| 流派 | 生成过程 | 特点 | 代表应用 |
|---|---|---|---|
| VAE | x → z → x'，用概率分布建模 | 训练稳定、潜空间结构好、但生成偏模糊 | Motion VAE、潜空间编辑 |
| GAN | 生成器 vs 判别器博弈 | 生成锐利，但训练不稳 | StyleGAN、AMP 动画 |
| Diffusion | 从噪声逐步去噪 | 质量最高，但推理慢 | Stable Diffusion、MDM |

### 1.2 动画领域里的 VAE 足迹

你做动画 TA，**VAE 不是遥远的学术名词**。看看它已经在哪些具体项目里跑：

| 项目 | 作用 | 引用 |
|---|---|---|
| **Motion VAE** | 把动画数据压到潜空间，在其中做运动控制 | Ling et al., SIGGRAPH 2020 [A] |
| **MotionCLIP** | 用 VAE 做文本-动作对齐，文字生成动作 | Tevet et al., ECCV 2022 [A] |
| **Phase-Functioned Neural Networks (PFNN)** | 非直接 VAE，但其 phase + latent 的思想同源 | Holden et al., SIGGRAPH 2017 [A] |
| **MLD (Motion Latent Diffusion)** | 先用 VAE 压到潜空间，再在潜空间做 Diffusion | Chen et al., CVPR 2023 [A] |

**关键点**：即使你未来学 Diffusion 做动画生成，十有八九还是在"**VAE 压出来的潜空间**"里跑 Diffusion。潜空间是**现代生成式动画的地基**。

## 第 2 章 潜空间：数据压缩背后的几何直觉 [L0 核心]

### 2.1 一个让你好奇的疑问

**场景**：你有一段 120 帧的走路动画，每帧 60 个关节，每个关节 3 个旋转值。这段动画的原始数据是：

```
120 帧 × 60 关节 × 3 = 21,600 个浮点数
```

现在告诉你：**用一个好的神经网络，可以把这 21,600 个数压到 32 个数**，然后再解压回来，**几乎完全还原**原动画。

**你的第一反应应该是**："等等，这怎么可能？信息不是丢了吗？"

答案是：**没丢，因为原始数据里有大量冗余**。这一章我们把这个事情讲清楚。

### 2.2 Shader 视角的类比：GPU 纹理压缩

你肯定用过 DXT、BC7 之类的 GPU 纹理压缩。一张 1024×1024 的 RGBA 贴图：

```
原始大小：1024 × 1024 × 4 bytes = 4 MB
BC7 压缩：                      = 1 MB（1/4）
解压后的视觉效果：几乎看不出差别
```

**为什么能压？** 因为**真实贴图的像素之间高度相关**——相邻像素颜色接近、大部分区域颜色变化平滑。满载 4 MB 里，大部分 bit 是在表达"这里和上个像素差不多"，真正"新信息"少得多。

**动画数据同理**。21,600 个浮点数看似很多，但：

- 相邻关节相互约束（肘关节不能转 720°）
- 相邻帧之间变化极小（60 FPS 下单帧位移 < 几厘米）
- 人体运动是**协调的**（左腿迈出时右手会摆动）

**这些约束让数据的"真实维度"远低于 21,600**。32 维潜空间就够装下一段典型动作的所有本质信息。

### 2.3 流形（Manifold）：数学家的说法

上面这个直觉，数学里叫**流形（Manifold）**：

> 高维空间里，真实数据只分布在一个**低维子结构**上。这个子结构就是流形。

**一个更形象的类比**：想象 3 维空间里的一张纸。

```
3 维空间 = 整个房间（x, y, z 都能任意取值）
一张纸 = 嵌在这个空间里的 2 维流形
       （沿纸面有两个自由度，垂直于纸面没有）
```

用 3 个坐标描述纸上任意一点是"够用但浪费"的——**用纸面的 2 个坐标 (u, v) 就完全够**。

**动画数据的流形**：21,600 维空间里，所有"**合法的人类动画**"只占其中一个约 32 维的扁平流形。流形外面的点（比如"头在脚下方"、"肘关节反转 180°"）都不是合法动画。

![fig2](/images/Autoencoder-VAE-DeepDive/fig2.png)
*图 2：高维数据的低维流形结构（纸的类比）*

### 2.4 自编码器（Autoencoder, AE）：学流形坐标的神经网络

现在我们给"学流形坐标"这件事起个名字——这就是 **Autoencoder (AE)**。

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  原始数据 x          潜变量 z          重建 x'       │
│  (21,600 维)  →→→→→  (32 维)  →→→→→   (21,600 维)   │
│                                                     │
│        Encoder E           Decoder D                │
│                                                     │
│  训练目标：x' 尽量等于 x（比如 MSE 损失）            │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**三个组件**：

1. **Encoder（编码器）**：一个神经网络 `z = E(x)`，把高维 x 压成低维 z。
2. **Decoder（解码器）**：一个神经网络 `x' = D(z)`，把 z 还原为 x'。
3. **损失函数**：`L = ‖x − x'‖²`（均方误差），强迫 x' 接近 x。

**训练完成后的效果**：
- Encoder 学会了"把输入投影到流形上"（即用流形坐标表示）
- Decoder 学会了"从流形坐标还原完整数据"
- **z 就是这段数据在流形上的坐标**

这个 z 就叫**潜变量（Latent）**，z 所在的 32 维空间就叫**潜空间（Latent Space）**。

### 2.5 具体数字演示：3 维潜空间的手算

为了让你摸到"潜空间"，我们用一个极简化的例子。

**假设**：有一个 AE，输入 5 维，潜空间 3 维。训练完后，对三个具体姿态编码：

```
x_行走    = [1.0, 0.2, 0.5, 0.3, 0.8]
x_跑步    = [1.2, 0.5, 0.7, 0.4, 1.1]
x_跳跃    = [0.8, 1.0, 0.3, 0.9, 0.6]

Encoder 处理后：
z_行走    = [0.3, -0.7, 1.2]
z_跑步    = [0.5, -0.5, 1.0]
z_跳跃    = [-0.4, 0.9, 0.3]
```

**观察 1**：行走和跑步的 z 很接近（它们在潜空间里是"邻居"）。
**观察 2**：跳跃的 z 远离另外两个（跳跃在潜空间里是"另一个区域"）。

**这就是潜空间的神奇之处**：**相似的姿态被映射到相似的 z**，语义关系被保留下来。

### 2.6 潜空间算术（手算）

既然 z 是一组坐标，那我们能在潜空间里"做算术"吗？

**实验**：取 z_行走 和 z_跑步 的中点：

```
z_mid = (z_行走 + z_跑步) / 2
     = ([0.3, -0.7, 1.2] + [0.5, -0.5, 1.0]) / 2
     = [0.4, -0.6, 1.1]
```

**把 z_mid 丢给 Decoder**，理想情况下会输出"**介于行走和跑步之间的姿态**"——大概是"快步走 / 慢跑"的样子。

**这就是潜空间插值（Latent Interpolation）**，在动画里极其有用：

- **动作混合**：输入起始动作和目标动作，在潜空间插值，得到连续的渐变帧
- **新动作生成**：对不同动作的 z 做线性组合，得到训练集里没有的新动作
- **风格编辑**：找到"愤怒 z"和"平静 z"的差向量，加到任意动作的 z 上实现风格迁移

![fig3](/images/Autoencoder-VAE-DeepDive/fig3.png)
*图 3：潜空间插值生成"快步走"（行走↔跑步的连续渐变）*

### 2.7 一个关键问题留给第 3 章

但这里有个**藏得很深的陷阱**，你现在可能还没意识到：

> **上面的"潜空间插值"真的能生成合理姿态吗？**
>
> 或者说，`z_mid = [0.4, -0.6, 1.1]` 这个点在潜空间里，它**真的被训练过**吗？

如果答案是"训练过"—— ✅ 恭喜，Decoder 能漂亮还原。

如果答案是"没训练过，它是两个训练点之间的空白地带"—— ⚠️ **问题来了**：Decoder 面对它时可能输出垃圾。

**这正是 AE 的致命缺陷，也是 VAE 要解决的问题**。下一章见。

### 2.8 章末自测（2 题，给答案）

**Q1**：如果我的动画数据是 60 帧 × 42 关节 × 3 旋转 = 7560 维，你觉得合适的潜空间维度大约是多少？为什么不选 100 维？为什么不选 5 维？

<details>
<summary>参考答案</summary>

合适的维度大约在 16-64 之间，常见选择 32。

- 选 100 维：潜空间维度太高，压缩比不够，流形还没被"卡紧"，模型可能学偏
- 选 5 维：潜空间太挤，容纳不下所有姿态变化，重建误差会很大
- 32 维：经验甜点区，足够装下人体运动的本质自由度（大约 20-40 个独立"运动模式"）

</details>

**Q2**：为什么说"潜空间里相邻 = 原空间里相邻"？如果 Encoder 不满足这个性质会怎样？

<details>
<summary>参考答案</summary>

训练完成的 AE 会自动让**语义相似的输入**（比如两个走路姿态）映射到相近的 z，因为这样解码器才能用"一套参数"同时还原多个相似姿态——损失函数推动了这个性质。

如果 Encoder 不满足：训练会变困难（损失难以下降），而且潜空间无法用于插值或生成任务（插值出来会是垃圾）。

</details>

## 第 3 章 从 AE 到 VAE：分布输出的突破 [L0 核心]

### 3.1 回到上一章的悬念

第 2 章末尾我们留了一个问题：

> `z_mid = (z_行走 + z_跑步) / 2 = [0.4, -0.6, 1.1]` —— 这个点**真的被训练过**吗？

答案是：**大概率没有**。

训练数据里有"行走 z_行走"和"跑步 z_跑步"，但 `z_mid` 在它们中间，是一个**训练时从未出现过的点**。Decoder 面对这种点时，会发生什么？

### 3.2 AE 的潜空间是"孤岛"

**关键事实**：Autoencoder 训练出来的潜空间**是"点阵"，不是"连续区域"**。

直觉图景：

```
训练数据 10 万个样本 → 潜空间里 10 万个孤立的点

  ·        ·              ·
         ·      ·     ·
      ·     ·         ·
   ·           ·   ·
         ·           ·
              ·
                 ← 点与点之间是"未知地带"
```

**每个训练样本对应一个精确的 z**（Encoder 是确定性的：同一个 x 总是压到同一个 z）。点与点之间的**空白区域**（占潜空间绝大多数体积）从来没有训练过 Decoder 怎么处理。

**结果**：

- 往已有训练点 `z_行走` 里丢，Decoder 能漂亮还原。
- 往空白点 `z_mid` 里丢，Decoder **输出无法预测**——可能是乱码、可能是扭曲、可能偶尔凑巧合理。

**这就是 AE 不能直接用来生成新数据的根本原因**：它的潜空间**没有被填满**。

### 3.3 VAE 的核心创新：Encoder 输出"分布"不是"点"

Kingma & Welling（2013）的 VAE 论文（*Auto-Encoding Variational Bayes*, ICLR 2014 [A]）提出的关键改动，用一句话概括：

> **Encoder 不再输出一个确定的 z，而是输出 z 的一个分布参数（μ, σ）。**

对比两者：

```
Autoencoder:                       VAE:
                                    
x → Encoder → z                    x → Encoder → (μ, σ)
     ↑                                  ↑
  一个确定的点                         一个高斯分布 N(μ, σ²)
                                    
  "你是 42 号座位"                    "你在 40-44 号区域里"
```

**通俗版**：AE 像"把照片塞进一个确切的编号格子"，VAE 像"把照片放到一片模糊区域里"。

这个"模糊"的改动看起来像是**降低了精度**，为什么反而变强了？答案在下一节。

### 3.4 为什么"模糊"反而更强？

关键是 VAE 训练时的机制：

```
x → Encoder → (μ, σ) → 从 N(μ, σ²) 采样 → z → Decoder → x'
                          ↑
                   每次前向都重新采样！
                   同一个 x 会得到略微不同的 z
```

**每个训练样本 x 对应一片"高斯云"**，而不是一个点：

```
训练过程中，同一段"行走"动画 x_行走 会生成：
  第 1 轮采样：z = [0.31, -0.68, 1.19]
  第 2 轮采样：z = [0.29, -0.72, 1.24]
  第 3 轮采样：z = [0.33, -0.69, 1.18]
  ...
  
这 1000 次训练，相当于告诉 Decoder：
  "以 μ 为中心、σ 为半径的这片区域里的所有点，
   你都要尽量解码回 x_行走"
```

**结果**：Decoder 被训练得**能处理整片区域**，不是单个点。

再配合下一节要讲的 KL 散度约束，**不同样本的高斯云会互相重叠**：

```
        z_行走 的高斯云          z_跑步 的高斯云
         ╭──────╮                ╭──────╮
         │·  ·  │                │·  ·  │
         │  ·   │ ←─重叠区域──→ │   ·  │
         │· · · │                │· ·   │
         ╰──────╯                ╰──────╯
                    ↑
            z_mid 落在重叠区
            Decoder 见过类似点 → 能合理解码
```

**这就是 VAE 能生成新数据的根源**：

1. 每个训练样本"铺"一片区域，而不是占一个点
2. 所有区域叠加后**填满整个潜空间**
3. 推理时从 `N(0, 1)` 任意采样一个 z，**总会落在某个被训练过的区域里**
4. Decoder 解码出合理的新样本

![fig4](/images/Autoencoder-VAE-DeepDive/fig4.png)
*图 4：AE 的孤立点 vs VAE 的高斯云覆盖*

### 3.5 KL 散度的作用（先给直觉）

但光"输出分布"还不够。想象如果 Encoder 把不同样本的 μ 学到离得特别远：

```
z_行走 的 μ = [100, -200, 500]
z_跑步 的 μ = [-800, 300, 1]
z_跳跃 的 μ = [9999, 9999, 9999]
```

那不同高斯云**根本不会重叠**，潜空间绝大多数区域仍然是"垃圾区"——推理时从 `N(0,1)` 采样，大概率落在没训练过的地方。

**解决方案**：加一个**正则项**，**逼着所有分布向标准正态 N(0, 1) 靠拢**。

这个正则项就是 **KL 散度（Kullback-Leibler Divergence）**：

$$
\text{KL}(q(z|x) \,\|\, p(z)) = \text{KL}(N(\mu, \sigma^2) \,\|\, N(0, 1))
$$

其中 `q(z|x)` 是 Encoder 输出的分布，`p(z)` 是先验（标准正态）。

**KL 散度的直觉**：
- KL = 0 ⇔ 两个分布完全一样
- KL 越大 ⇔ 两个分布差别越大
- **不是对称的**：KL(A||B) ≠ KL(B||A)，这里无需深究

### 3.6 手算一个 KL 散度

**目标**：手算 `KL(N(μ, σ²) || N(0, 1))`。

**闭式解**（对单维高斯，已由数学家推好）：

$$
\text{KL}(N(\mu, \sigma^2) \,\|\, N(0, 1)) = \frac{1}{2} \left( \mu^2 + \sigma^2 - \log(\sigma^2) - 1 \right)
$$

**手算 3 个场景**：

| 场景 | μ | σ² | μ² | σ²−log(σ²) | KL |
|---|---|---|---|---|---|
| ① 理想（贴近 N(0,1)） | 0 | 1 | 0 | 1 − 0 = 1 | ½(0+1−1) = **0** ✅ |
| ② μ 偏了 | 2 | 1 | 4 | 1 − 0 = 1 | ½(4+1−1) = **2** ⚠️ |
| ③ σ 太尖（方差太小） | 0 | 0.01 | 0 | 0.01 − log(0.01) ≈ 0.01 + 4.6 = 4.61 | ½(0+4.61−1) = **1.8** ⚠️ |
| ④ σ 太胖（方差太大） | 0 | 100 | 0 | 100 − log(100) ≈ 100 − 4.6 = 95.4 | ½(0+95.4−1) = **47** ⚠️⚠️ |

**观察**：
- KL 本质在**双重惩罚**：μ 偏离 0（让 μ²↑）**+** σ 偏离 1（让 σ²−log(σ²)−1 ↑，这是一个"碗状"函数，σ²=1 时为 0）
- **只有 μ=0 且 σ²=1 时 KL=0**，任何偏离都会被惩罚
- 场景 ④ 告诉我们：**方差过大也会被狠狠惩罚**（防止 encoder 偷懒把方差开太大）

**几何直觉**：

```
σ² 对 KL 的贡献 = σ² − log(σ²) − 1 (这是一个 "碗"，碗底在 σ²=1)

    KL
     │         ╲
     │          ╲             ╱
     │           ╲           ╱
     │            ╲         ╱
     │             ╲_______╱
     │                  ↑
     │                σ²=1 (碗底, KL=0)
     └──────────────────────────── σ²
         太小 ←          → 太大
```

![fig9](/images/Autoencoder-VAE-DeepDive/fig9.png)
*图 9：KL 散度对 σ² 的"碗状"惩罚函数——只有 σ²=1 时不付代价*

### 3.7 VAE 的损失函数全貌

把"重建"和"KL 正则"组合起来，就是 VAE 的损失函数：

$$
\boxed{\;L_{\text{VAE}} = \underbrace{\|x - x'\|^2}_{\text{重建损失}} + \beta \cdot \underbrace{\text{KL}(q(z|x) \,\|\, N(0, 1))}_{\text{KL 正则}}\;}
$$

**两项的物理意义**：

| 项 | 想让什么最小化？ | 如果单独优化会怎样？ |
|---|---|---|
| 重建损失 | Decoder 还原能力 | μ 各自飞远、σ → 0，退化成 AE |
| KL 正则 | 分布贴近 N(0,1) | 所有 μ=0、σ=1，样本混淆无法区分 |
| **两项平衡** | **潜空间既规整又能重建** | 这才是 VAE 的魔法 |

**β 是什么？** 一个权重超参数：
- β=1：原版 VAE（Kingma 2013）
- β>1：更强的正则（β-VAE, Higgins 2017 [A]），潜空间更"解耦"，重建变差
- β<1：更弱正则，潜空间变散但重建更清晰

（β-VAE 的细节我们留到第 7 章展开。）

### 3.8 小结一下这一章

```
┌─────────────────────────────────────────────────┐
│                                                 │
│  AE 的病：潜空间是孤立点，中间空白 → 不能生成   │
│                                                 │
│  VAE 的药：                                     │
│     1. Encoder 输出分布（μ, σ）而不是点         │
│     2. 训练时每次从分布采样，"铺"一片云         │
│     3. KL 正则逼着所有云贴近 N(0,1)，互相重叠   │
│     4. 潜空间被填满 → 推理时采样就能生成       │
│                                                 │
│  代价：重建会比 AE 略模糊（模糊 ≈ 生成能力费用）│
│                                                 │
└─────────────────────────────────────────────────┘
```

**但这里又冒出一个新问题**：

> VAE 训练时需要"从 N(μ, σ²) 采样"这一步——**采样是随机的，它怎么反向传播？**

下一章揭晓。

### 3.9 章末自测（2 题，给答案）

**Q1**：如果我把 VAE 的损失函数改成只保留重建项（去掉 KL），训练结束后会变成什么？

<details>
<summary>参考答案</summary>

退化成一个**有噪声的 AE**：Encoder 仍然输出 (μ, σ)，但 σ 会被训练到趋近 0（让采样点始终等于 μ），模型变成"用 μ 做 AE"。潜空间变回孤立点，**失去生成能力**。

</details>

**Q2**：β-VAE 把 β 调到 100，你预期重建质量和潜空间结构会怎样变化？

<details>
<summary>参考答案</summary>

- 重建质量**变差**（KL 项太强，逼着所有分布贴近 N(0,1)，Encoder 没有自由度区分不同样本）
- 潜空间更"规整"（所有云几乎都叠在 N(0,1) 上）
- **解耦性提升**（这是 β-VAE 的论文动机——极端情况下不同维度学到不同语义，如姿势维度 vs 光照维度分离）

</details>

## 第 4 章 重参数化技巧：让随机采样能反向传播 [L0 核心]

### 4.1 一个让 VAE 差点胎死腹中的问题

上一章末尾我们留了个炸弹：

> VAE 训练时需要从 `N(μ, σ²)` 采样得到 z。**采样是随机操作，怎么反向传播？**

这不是小问题。我们把数据流画出来：

```
x → Encoder → (μ, σ) → 采样 z ~ N(μ, σ²) → Decoder → x'
                          ↑
                     问题就在这里
```

**反向传播需要**：`∂L/∂参数` 沿着计算图一路传回去。

**采样是什么？** 它是"给定 (μ, σ)，从对应分布里**随机**抽一个数"。

- **确定性操作**（加法、乘法、矩阵乘、激活函数…）：PyTorch 的 autograd 知道如何求导
- **随机采样**：autograd 不知道 `∂z/∂μ` 是多少——因为 z 是"随机抽"出来的，它和 μ 的关系不是函数关系

**后果**：梯度到了 z 这里就**断了**。encoder 的参数 φ 永远收不到梯度信号，没法训练。

### 4.2 几条错误的尝试

在 Kingma 的天才想法之前，人们试过：

**尝试 1：直接用 μ，不采样**
- 相当于退化成 AE（见第 3.9 节 Q1），失去 VAE 的本意

**尝试 2：用 Policy Gradient / REINFORCE 那套随机梯度估计**
- 有效，但方差极大，训练几乎无法收敛
- 强化学习里常用，但 VAE 用这套效率太低

**尝试 3：??? 直到 Kingma 2013 想到了一个优雅的办法**

### 4.3 天才的办法：把随机性"外置"

Kingma 的洞察是这样一个数学事实：

> **从 `N(μ, σ²)` 采样 z**，等价于 **从 `N(0, 1)` 采样 ε，然后令 `z = μ + σ·ε`**。

一行公式：

$$
\boxed{\; z = \mu + \sigma \cdot \varepsilon, \quad \varepsilon \sim N(0, 1) \;}
$$

**这叫重参数化技巧（Reparameterization Trick）**。

**看起来像啥都没干？** 对，从"采样出来的 z 服从什么分布"这个角度看，新写法和旧写法**完全等价**——生成的 z 数值分布完全一样。

**但从"计算图结构"的角度看，天翻地覆**：

```
原版：                      重参数化版：
                             
μ ─┐                        μ ─┐
    ├─→ N(μ,σ²) → z          ├─→ (+) ─→ z
σ ─┘    ↑                    σ ─┤        ↑
        随机                      ├─(×)   梯度通！
                              ε ─┘
                              (外部噪声)
```

**核心差别**：
- 原版：z 由 `N(μ, σ²)` 直接采样，**(μ, σ) → z** 是随机过程，不可导
- 新版：ε **已经在外部采好了**（随机性被"外置"），再用 `z = μ + σ·ε` 计算 z，**这是一个确定性函数**，可导！

### 4.4 具体数字手算

**场景**：μ=2，σ=0.5，需要采一个 z。

**原版做法**：

```
调用 torch.distributions.Normal(2, 0.5).sample()
→ 随机输出 z，比如 z = 2.6
但 PyTorch 不知道这个 2.6 是怎么来的，无法求 ∂z/∂μ
```

**重参数化做法**：

```
Step 1: 从标准正态采 ε → 比如 ε = 1.2
Step 2: z = μ + σ·ε = 2 + 0.5 × 1.2 = 2.6  ← 和上面数值一致！

但这次 PyTorch 看到的是：
  ε = 1.2          （常数，不参与梯度）
  z = 2 + 0.5 × 1.2 = 2.6
  
它能算：
  ∂z/∂μ = 1         （加法的导数）
  ∂z/∂σ = ε = 1.2   （乘法的导数）
  
梯度 → 顺利反向传播回 μ 和 σ，再回到 encoder 参数 φ
```

**关键洞察**：
- `z = 2.6` 这个**数值**两种方法完全一样
- `μ → z` 这条**梯度通路**在新方法里畅通了

### 4.5 PyTorch 实现一行代码

```python
# μ 和 σ（实际代码里用 log_var 数值稳定性更好）由 encoder 输出
mu, log_var = encoder(x)
sigma = torch.exp(0.5 * log_var)

# 重参数化核心（就这一行）
z = mu + sigma * torch.randn_like(mu)
#          ↑
#   randn_like 生成 ε，requires_grad=False
#   ε 不进计算图，梯度只通过 mu 和 sigma 回流
```

就这三行，搞定了 VAE 的核心机制。

### 4.6 几个容易混淆的点

**Q: 训练每一步 ε 都重新采样吗？**
A: 对，每次 forward 都重新采 ε。这保证了"每个样本每次走过的 z 不同"，形成"高斯云"的覆盖。

**Q: 推理时还需要重参数化吗？**
A: 推理（生成新样本）时直接从 `N(0, 1)` 采 z 就行，不需要 encoder 和重参数化。只有训练时要。

**Q: 为什么不能 `z = μ + σ·常数`（比如总是用 ε=0.5）？**
A: 这就变成确定性函数了，每次同一个 x 对应同一个 z，又退化成 AE。**随机性是 VAE 训练的本质**，必须保留 ε 的随机性。

**Q: 重参数化有没有泛化意义？**
A: 极大的意义。这是"**把随机性从计算图外部注入**"的通用模式，后面你会在 Diffusion（用同样思路加/去噪声）、**Normalizing Flow（标准化流，一类用可逆神经网络做生成的模型）**、**Gumbel-Softmax（处理离散分布的重参数化变体）** 等多处看到同一个技巧。

![fig5](/images/Autoencoder-VAE-DeepDive/fig5.png)
*图 5：重参数化前后的计算图对比——梯度通路从断到通*

### 4.7 为什么能这么做？数学上的严格依据

（这一节是 [L3 深坑]，不懂可跳，不影响工程使用。）

重参数化能做的本质原因是：**高斯分布是一个"位置-尺度族"**（location-scale family）。这类分布有个性质：

> 对任意 (μ, σ)，`N(μ, σ²)` 的样本都可以通过**仿射变换**（线性变换）从一个标准分布（N(0,1)）的样本获得。

常见属于位置-尺度族的分布：正态、均匀、拉普拉斯。**重参数化对这些分布都成立**。

不属于的：离散分布（如分类 / 多项式）。所以 VQ-VAE 那种输出离散码本的情形要用 Gumbel-Softmax 等替代技巧。

### 4.8 章末自测（1 题，给答案）

**Q**：假设 encoder 输出 μ=1.5，σ=2.0，某次前向采样得到 ε=-0.3。请手算 z，并计算 ∂z/∂μ 和 ∂z/∂σ。

<details>
<summary>参考答案</summary>

- z = μ + σ·ε = 1.5 + 2.0 × (−0.3) = **0.9**
- ∂z/∂μ = 1（加法对 μ 求导 = 1）
- ∂z/∂σ = ε = **−0.3**（z = μ + σ·ε 对 σ 求导 = ε）

反向传播时：encoder 会收到"σ 要朝 −0.3 的反方向调（如果 ∂L/∂z 是正）"的信号。

</details>

## 第 5 章 VAE 训练全流程：把三个核心机制串成闭环 [L0 核心]

前面三章各自讲清了一件事。本章把它们**串成一个完整的训练-推理闭环**，给你一个"从无到有"的工程实感。

### 5.1 完整数据流一图看懂

```
┌─────────────────────── 训练阶段 ────────────────────────┐
│                                                        │
│                                   重建损失              │
│  x ─→ Encoder ─→ (μ, σ)             ║                  │
│                    │                ║                  │
│                    ↓                ║                  │
│                 z = μ + σ·ε ─→ Decoder ─→ x'  ─║─→ L_rec│
│                    ↑                                   │
│                 ε ~ N(0,1)                             │
│                                                        │
│  (μ, σ) ─→ KL(N(μ,σ²) ‖ N(0,1)) ───────────────→ L_kl  │
│                                                        │
│  Total Loss = L_rec + β · L_kl   → 反向传播更新 E, D    │
│                                                        │
└────────────────────────────────────────────────────────┘

┌─────────────────────── 推理阶段 ───────────────────────┐
│                                                        │
│  z ~ N(0, 1)  ─→  Decoder  ─→  新样本 x'               │
│      ↑                                                 │
│  不需要 Encoder！                                       │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**几个关键观察**：

1. **训练时 Encoder 和 Decoder 都在用**；推理时**只需要 Decoder**。
2. **Encoder 学"怎么压"，Decoder 学"怎么解压"**——两者同时训，损失强迫它们配合。
3. **ε 只在训练时用**（为了梯度），推理时直接从 N(0,1) 采 z。
4. **潜空间被填满是训练的副产物**，推理时白嫖这个好处。

![fig6](/images/Autoencoder-VAE-DeepDive/fig6.png)
*图 6：VAE 的训练-推理两个阶段*

### 5.2 完整 PyTorch 伪代码（~50 行，带详细注释）

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# ========================================
# 1. 定义 VAE 网络
# ========================================
class VAE(nn.Module):
    def __init__(self, input_dim=784, hidden_dim=400, latent_dim=20):
        super().__init__()
        # Encoder：输入 → 隐藏 → (μ, log σ²)
        self.enc_fc1 = nn.Linear(input_dim, hidden_dim)
        self.enc_mu  = nn.Linear(hidden_dim, latent_dim)   # 输出 μ
        self.enc_lv  = nn.Linear(hidden_dim, latent_dim)   # 输出 log σ²（数值稳定）
        
        # Decoder：z → 隐藏 → 输出
        self.dec_fc1 = nn.Linear(latent_dim, hidden_dim)
        self.dec_fc2 = nn.Linear(hidden_dim, input_dim)
    
    def encode(self, x):
        h = F.relu(self.enc_fc1(x))
        return self.enc_mu(h), self.enc_lv(h)   # μ, log σ²
    
    def reparameterize(self, mu, log_var):
        sigma = torch.exp(0.5 * log_var)        # σ = exp(0.5 · log σ²)
        eps = torch.randn_like(mu)              # 外部噪声 ε
        return mu + sigma * eps                 # ★ 重参数化核心 ★
    
    def decode(self, z):
        h = F.relu(self.dec_fc1(z))
        return torch.sigmoid(self.dec_fc2(h))   # MNIST 像素 [0,1]
    
    def forward(self, x):
        mu, log_var = self.encode(x)
        z = self.reparameterize(mu, log_var)
        x_recon = self.decode(z)
        return x_recon, mu, log_var

# ========================================
# 2. 损失函数
# ========================================
def vae_loss(x_recon, x, mu, log_var, beta=1.0):
    # 重建损失：二元交叉熵（适合 MNIST 像素），对连续数据用 MSE
    rec_loss = F.binary_cross_entropy(x_recon, x, reduction='sum')
    
    # KL 散度（闭式解，对每一维求和）
    # KL(N(μ, σ²) ‖ N(0,1)) = 0.5 · Σ (μ² + σ² - log σ² - 1)
    kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())
    
    return rec_loss + beta * kl_loss, rec_loss, kl_loss

# ========================================
# 3. 训练循环
# ========================================
model = VAE(input_dim=784, latent_dim=20)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(50):
    for x, _ in train_loader:          # MNIST，忽略标签
        x = x.view(-1, 784)            # 展平
        x_recon, mu, log_var = model(x)
        loss, rec, kl = vae_loss(x_recon, x, mu, log_var, beta=1.0)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f"Epoch {epoch}: Recon={rec.item():.2f}, KL={kl.item():.2f}")

# ========================================
# 4. 推理：生成新样本
# ========================================
model.eval()
with torch.no_grad():
    z = torch.randn(16, 20)            # 从 N(0,1) 采 16 个
    x_new = model.decode(z)            # 生成 16 张新图
    # 可以 reshape(16, 28, 28) 后保存或展示
```

**几个工程细节**：

- **为什么 encoder 输出 `log σ²` 而不是 σ**：log_var 数值稳定、无正负约束（σ 必须 > 0，直接输出要加 softplus 等激活），而且和 KL 闭式解里的 log σ² 吻合。
- **为什么用 BCE 做重建损失**：MNIST 像素是 [0,1]，BCE 比 MSE 更合适；对连续数据（动画关节角）用 MSE。
- **β 从 0 逐步退火到 1（Annealing）**：实际训练中常见技巧，先让模型专心学重建（β=0），再慢慢加 KL 约束（β→1），防止训练早期 KL 太强把模型压死。

### 5.3 训练曲线的典型形状

一个健康的 VAE 训练过程大概长这样：

```
损失
 ▲
 │╲
 │ ╲  重建损失（快速下降到稳定）
 │  ╲_____________________
 │
 │         ╭─────────
 │        ╱  KL 损失（慢涨，达到平衡）
 │       ╱
 │   ╱─╯
 │__╱─────────────────────→ epoch
```

**读曲线**：
- 重建损失先降后稳 → 模型学会了重建
- KL 损失不是一路降到 0（如果降到 0 说明 σ → 无穷 = "模型放弃"，或 μ = 0 → "所有样本混一块"）
- 两项达到平衡 → 潜空间规整且能重建

![fig10](/images/Autoencoder-VAE-DeepDive/fig10.png)
*图 10：健康的 VAE 训练曲线——两项损失达到动态平衡*

**异常情形**：
- KL 崩到 0：**Posterior Collapse**（后验崩溃），encoder 摆烂输出 N(0,1)，decoder 彻底忽略 z。β 调小或用 KL annealing 缓解。
- 重建一直高：模型容量不足或学习率不合适。
- KL 爆炸：β 太大，或 encoder 输出的 log_var 没做数值范围约束。

### 5.4 章末自测（1 题，给答案）

**Q**：假设你训练完 VAE 后，发现生成的新样本**非常模糊但有结构**。可能的原因和缓解方法是什么？

<details>
<summary>参考答案</summary>

**原因**（可能多个）：
1. β 过大，KL 正则太强，逼着潜空间完全贴近 N(0,1)，损失了细节（β-VAE 过头）
2. 潜空间维度太小，表达能力不足
3. Decoder 容量不够（模糊是 VAE 的固有倾向，因为重建损失是"逐像素平均"，天然倾向输出"均值图"）
4. 训练不够充分

**缓解**：
- 降低 β
- 增大潜空间维度
- Decoder 使用更强的网络（如 PixelCNN decoder、autoregressive decoder）
- 换对抗损失（VAE-GAN）或换 Diffusion（在 VAE 压缩的潜空间上跑 Diffusion，即 MLD/LDM）

"模糊" 是 VAE 家族的**著名缺陷**，也是后来 GAN 和 Diffusion 崛起的动因之一。

</details>

## 第 6 章 动画应用：Motion VAE 深度解读 [L0 核心]

> 📌 **TA 读法**：**本章 L0，必须掌握**。Motion VAE（Ling et al., SIGGRAPH 2020）是动画 TA 的直接相关工作——用 VAE 学动作潜空间，再在潜空间里做动作编辑/混合。你后面读 MDM/MLD 时会看到"VAE 潜空间"的回响。手算 ELBO 那段（如果前面第 5 章没完全消化）本章会再给一个 Motion 场景的实例。

### 6.1 Motion VAE 是什么？

**Motion VAE（MVAE）**：Ling, Zinno, Cheng, van de Panne, *Character Controllers Using Motion VAEs*, SIGGRAPH 2020 [A]。

- **作者单位**：UBC（温哥华）+ EA Vancouver（Electronic Arts）
- **开源**：GitHub 有官方代码和训练数据
- **工业影响**：EA 旗下多款足球、格斗类游戏的动画系统受此思路影响

**核心贡献**：**把 VAE 用到人物动作序列上，让"潜空间"成为角色控制的接口**。

### 6.2 架构拆解

```
┌─────────────── Motion VAE 架构 ───────────────┐
│                                               │
│   前一帧骨骼状态 s_{t-1} ─┐                    │
│                           ├─→ Encoder (MLP)   │
│   当前帧骨骼状态 s_t    ─┘      │             │
│                                 ↓             │
│                            (μ, σ) 潜空间      │
│                                 │             │
│                                 ↓ 采样         │
│                                 z             │
│                                 │             │
│    s_{t-1} ──┐                  ↓             │
│              ├─→ Decoder (MLP) ─→ s_t (预测)  │
│    z  ────── ┘                                │
│                                               │
└───────────────────────────────────────────────┘
```

**几个关键设计**：

1. **条件化**：不是把整段动画编码成一个 z，而是**每一帧都编码**，且 encoder 和 decoder 都接收 `s_{t-1}`。
   - 这让 z 只需要编码"这一帧相对前一帧的变化"，而不是绝对姿态
   - 极大降低了潜空间需要表达的复杂度

2. **潜空间维度**：通常 32 维左右

3. **Decoder 的角色**：给定前一帧和一个随机 z，预测下一帧。**这使 VAE 变成了一个"随机运动控制器"**——每步采不同的 z，得到不同的后续动作。

### 6.3 Motion VAE 的"玩法"

MVAE 论文展示了几种用 MVAE 做动画的方式：

**玩法 1：随机生成**
- 从 `N(0,1)` 采 z → decode → 得到下一帧 → 作为新的 s_{t-1} 继续 → 无限生成
- 像"让角色即兴走路"

**玩法 2：目标驱动**
- 在训练好的潜空间上训一个 **Policy**（小神经网络），输入目标位置/速度，输出 z
- Policy 相当于"选择一个语义向量"，再让 decoder 翻译成动作
- 实现方式：可以用 RL（奖励 = 是否达到目标）或监督学习

**玩法 3：风格插值**
- 找两个不同风格动作的 z（比如"自信走"和"疲惫走"），线性插值
- 配合潜空间算术可实现动作风格渐变

![fig7](/images/Autoencoder-VAE-DeepDive/fig7.png)
*图 7：Motion VAE 的训练/生成流程图*

### 6.4 为什么 MVAE 比 Motion Matching 香？

对比你熟悉的 Motion Matching（MM，EA Ubisoft 通用技术）：

| 对比项 | Motion Matching | Motion VAE |
|---|---|---|
| 动画库 | 离线存储所有片段（几百 MB~几 GB） | 训练一个网络（几 MB~几十 MB）|
| 运行时 | 查询最近邻片段 → 插值 | 一次前向 → 一帧 |
| 新动作生成 | ❌ 只能匹配库里已有的 | ✅ 可以"采样"出库外新动作 |
| 过渡平滑性 | 依赖匹配质量，偶有抖动 | 潜空间连续，天然平滑 |
| 可控性 | 通过轨迹/控制信号匹配 | 通过潜空间或条件变量控制 |

**实际项目的走向**：MVAE 不是要"干掉" MM，而是**互补**。业界常见做法：

- MM 提供"基础库级的保真度"
- MVAE 提供"潜空间层的新动作 + 风格混合"
- 两者混合部署，得到"又真实又灵活"的系统（业界实践，Ubisoft / EA 内部技术分享见 GDC 2022-2023，具体混合架构 [待验证]）

### 6.5 MVAE 之后：家族延伸

| 论文 | 改进 | 关键思路 |
|---|---|---|
| **Motion VAE (2020)** | 基线 | CondVAE + 逐帧建模 |
| **MoGlow (2020)** | 用 Normalizing Flow 替代 VAE | 精确可逆、采样更快 |
| **ACTOR (2021)** | + Transformer encoder | 看整段动作上下文 |
| **MotionCLIP (2022)** | + CLIP 文本对齐 | 文字 → 动作生成 |
| **MLD (2023)** | VAE 压缩 + Diffusion | 潜空间 Diffusion，质量起飞 |
| **MotionLCM (2024)** | MLD + 一步采样 | 实时潜空间 Diffusion |

**可以看出一条清晰脉络**：MVAE 奠定"**用潜空间做动作控制**"的范式，后来的工作换骨架（Transformer）、换生成机制（Flow / Diffusion）、换条件（文本 / CLIP），但"潜空间+解码器"的核心结构一直保留。

### 6.6 章末小结

Motion VAE 给动画 TA 的启示：

1. **潜空间 = 一个可控的动作语义接口**。把角色的"性格、情绪、动作意图"编码到 z 里，controller 只管操作 z，不直接碰骨骼
2. **每帧一个 z + 前帧条件 = 紧凑的随机运动模型**，比整段编码更易训练
3. **后面所有"潜空间动画生成"工作都建立在 MVAE 范式上**

## 第 7 章 变体家族：β-VAE / CVAE / VQ-VAE [L0 核心]

> **TA 读法**：本章 **7.2 CVAE 和 7.3 VQ-VAE 是 L0 必须掌握**（CVAE 是 Motion VAE 的原理，VQ-VAE 是下一篇文档的主题，不懂就断层）。**7.1 β-VAE 是 L1 扩展**——解耦性（disentanglement）是研究向概念，TA 日常工作不直接依赖，了解即可。

VAE 提出后衍生出一大家族。我们挑 3 个讲清楚——**7.2 和 7.3 必须读，7.1 可跳**。

### 7.1 β-VAE：用权重 β 换取解耦性

**论文**：Higgins et al., *β-VAE: Learning Basic Visual Concepts with a Constrained Variational Framework*, ICLR 2017 [A]

**一句话版**：把 VAE 的损失改成

$$
L = L_{\text{rec}} + \boldsymbol{\beta} \cdot L_{\text{KL}}, \quad \beta > 1
$$

就这么一个改动。β 从 1 变大（比如 4、10、20）会发生什么？

**变化**：

| β 值 | 重建质量 | 潜空间解耦性 |
|---|---|---|
| β = 1（原版）| 好 | 一般 |
| β = 4~10 | 下降 | 大幅提升 |
| β = 50+ | 很差 | 可能过头崩塌 |

**什么叫"解耦性（Disentanglement）"**？

> 理想潜空间里，**每一维对应一个独立的语义因子**。比如动画里：
> - 维度 0 = 速度
> - 维度 1 = 左右转向
> - 维度 2 = 情绪（开心/愤怒）
> - ...
>
> 调某一维只改对应语义，其他不变。

**β-VAE 的机制**：更强的 KL 约束**逼潜空间各维独立**（因为先验 N(0,I) 本身是独立的）。代价是重建模糊。

**动画 TA 的用法**：如果你想用 VAE 做"旋钮式动画控制"（拖条控制速度、转向、情绪），β-VAE 比原版更合适。

### 7.2 CVAE：带条件的 VAE

**论文**：Sohn, Lee, Yan, *Learning Structured Output Representation using Deep Conditional Generative Models*, NeurIPS 2015 [A]

**一句话版**：Encoder 和 Decoder 都**额外接收一个条件 c**。

```
原版 VAE:                    CVAE:
                              
x → Encoder → (μ, σ)         x + c → Encoder → (μ, σ)
                                       
z → Decoder → x'             z + c → Decoder → x'
```

**条件 c 可以是**：
- 动作类别（跑/跳/走）
- 文本描述（"愤怒走路"）
- 目标位置
- 另一段动画（用于风格迁移）

**用法**：生成时固定 c，对 z 采样 → 得到"指定风格 / 类别 / 目标下的多样动作"。

**Motion VAE 其实就是 CVAE**：条件 c 是前一帧 `s_{t-1}`。你已经学过 MVAE，CVAE 的原理同源。

### 7.3 VQ-VAE：把连续潜空间离散化

> ⚠️ **前置说明**：你之前反馈说 VQ-VAE 没学过，这里我从零讲清。后面 4/15 学过的部分是"VQ-VAE 的动画应用"，本节补原理。

**论文**：van den Oord, Vinyals, Kavukcuoglu, *Neural Discrete Representation Learning*, NeurIPS 2017 [A]

#### 7.3.1 VAE 的一个"遗憾"

原版 VAE 的潜空间是**连续**的——z 是一个 32 维实数向量，取值无限。

但很多下游任务（比如语言模型、动作分类）更喜欢**离散符号**：
- 一段动画 → 一串离散 token [12, 7, 45, 3, 7, 45, ...]
- 这串 token 可以用 Transformer 自回归建模（就像预测下一个字）

**怎么把连续 z 变成离散符号？**

#### 7.3.2 VQ-VAE 的核心思路

**引入一个"码本（Codebook）"**：

```
Codebook = {e_1, e_2, e_3, ..., e_K}     
其中每个 e_k 是一个 D 维向量（比如 512 维，K=1024 个）
```

**训练时的工作流**：

```
x → Encoder → z_e (连续向量, D 维)
     ↓
   找到码本里距离 z_e 最近的 e_k
     ↓
   z_q = e_k （离散化后的 z，变成了"第 k 个码"）
     ↓
   Decoder(z_q) → x'
```

**关键步骤**：`z_q = argmin_k ‖z_e − e_k‖` —— **把连续的 z_e 替换成离散码本里最接近的 e_k**。

一段动画输入 → 一系列 k 索引 [12, 7, 45, ...]，这就是"动作离散化"。

#### 7.3.3 两个棘手问题

**问题 1：argmin 不可导，怎么反向传播？**

**解决**：**Straight-Through Estimator（STE，直通估计）**。前向时用 z_q，反向时直接把 z_q 的梯度**原封不动复制**给 z_e。相当于假装"z_q ≈ z_e"，让梯度"直通"过离散化这一关。

**问题 2：码本 e_k 也要训练**

**解决**：多引入两个损失项。

$$
L = \underbrace{\|x - D(z_q)\|^2}_{\text{重建}} + \underbrace{\|sg[z_e] - e_k\|^2}_{\text{码本学 Encoder}} + \beta \underbrace{\|z_e - sg[e_k]\|^2}_{\text{承诺损失 Commitment}}
$$

（`sg[·]` 表示 **stop-gradient（梯度截断）**，反向传播时把梯度按 0 处理。PyTorch 里用 `.detach()` 实现。）

**三个损失的物理意义**：
- 重建：保证 z_q 能解回 x
- 码本学 Encoder：`e_k → z_e`（码本每个向量去学匹配 encoder 输出）
- **承诺损失（Commitment Loss）**：`z_e → e_k`（让 encoder 输出别乱跑，"承诺"贴近码本向量以稳定训练）

#### 7.3.4 动画领域的回响

4/15 你学的 **VQ-VAE 动作离散化** 就是把这套用到动画上：
- 一段动作 → encoder → 一串 k 索引
- 索引可以像"字母表"一样被 Transformer 建模
- 然后可以做 **文本 → 动作索引串 → 解码回动作**（即 T2M-GPT 的路线）

如果当时 4/15 那篇你觉得懵，回头再看一下原理部分会豁然开朗——**VQ-VAE = VAE + 离散码本替换 + STE 梯度技巧**。

### 7.4 三者的对比速查表

| 变体 | 改动的部分 | 主要用途 |
|---|---|---|
| **VAE（基线）** | — | 生成连续数据、潜空间插值 |
| **β-VAE** | KL 权重 β>1 | 需要"旋钮式"解耦控制 |
| **CVAE** | 加条件 c | 条件生成（文字→动作、类别→动作）|
| **VQ-VAE** | 离散化潜空间 | 序列建模、与 Transformer 结合 |

**Q**：如果我想做一个"给定类别（走/跑/跳）和强度（0-1），生成对应动作"的工具，应该用哪个变体？简述架构。

## 第 8 章 横向对比：VAE vs GAN vs Diffusion [L2 后续]

> 📖 **本章定位**：你已经学过 VAE（本篇）和 GAN（4/21）。Diffusion 还没学（后续课程会专讲）。本章给一个**三巨头速览**，方便你建立全局视野。

### 8.1 三种生成哲学

> 📌 **首次出现术语**：**ELBO（Evidence Lower Bound，证据下界）** = `log p(x)` 的一个数学下界，VAE 的优化目标就是最大化它（详见附录 A）。**SDE（Stochastic Differential Equation，随机微分方程）** 是 Diffusion 的连续时间数学基础，**Score Matching（分数匹配）** 是 Diffusion 训练去噪器的核心思想——这两个都留到 Diffusion 课再讲。

| 家族 | 生成方式 | 核心机制 |
|---|---|---|
| **VAE** | 从潜空间采样 → 解码 | 最大化数据似然的变分下界 ELBO |
| **GAN** | 生成器 vs 判别器博弈 | 让判别器"分不出真假" |
| **Diffusion** | 从噪声逐步去噪 | 反向"加噪-去噪"的马尔可夫链 |

### 8.2 关键指标对比

> 📌 **Mode Collapse（模式崩塌）** = GAN 训练的一种失败模式：生成器只学会生成少数几种样本（比如只会画猫不会画狗），牺牲多样性换"判别器分不出"的表面成功。

| 指标 | VAE | GAN | Diffusion |
|---|---|---|---|
| **训练稳定性** | 高 ✅ | 低（Mode Collapse）❌ | 高 ✅ |
| **生成质量** | 偏模糊 | 锐利 ✅ | 最高 ✅✅ |
| **多样性** | 好 ✅ | 一般（可能塌模式）| 最好 ✅✅ |
| **推理速度** | 快（1 次前向）✅ | 快（1 次前向）✅ | 慢（几十-千步）❌ |
| **可编辑性** | 潜空间语义清晰 ✅ | 潜空间可用但不稳 | 近年可控技术火速发展 |
| **数学基础** | 变分推断 | 博弈论/散度 | SDE / Score Matching |

### 8.3 动画领域的组合拳

2024-2026 主流管线往往是**组合**：

**组合 1：VAE + Diffusion（Latent Diffusion）**
- VAE 先把动作压到潜空间（节省 Diffusion 计算）
- 在潜空间跑 Diffusion
- 代表：MLD（Chen et al., CVPR 2023 [A]）、Stable Diffusion 在图像领域也是这套
- **现代文生动画的事实标准**

**组合 2：VAE + Transformer（VQ-VAE + GPT）**
- VQ-VAE 把动作离散化
- Transformer 自回归生成动作 token 序列
- 代表：T2M-GPT（Zhang et al., 2023 [A]）、MotionGPT（Jiang et al., 2023 [A]）

**组合 3：RL + VAE（Physics-based）**
- VAE 提供动作生成先验
- RL 在潜空间做策略搜索
- 代表：AMP（Peng et al., SIGGRAPH 2021 [A]）、MaskedMimic（2024）

### 8.4 Diffusion 预告（留到后面专讲）

Diffusion 的直觉三步走：

1. **前向加噪**：给一张真实图加 T 步高斯噪声，最后变纯噪声
2. **训练去噪器**：训练一个神经网络，学会"从第 t 步噪声图预测第 t-1 步（去掉一点噪声）"
3. **推理生成**：从纯噪声出发，反复调用去噪器 T 次 → 生成样本

**和 VAE 的联系**：
- **前向加噪 = 可看作 encoder**（但没有参数，固定加噪）
- **反向去噪 = 多步 decoder**
- **重参数化**：同样大量用到 `z = μ + σ·ε` 这类形式
- **潜空间**：MLD 就是在 VAE 的潜空间做 Diffusion

**后续课程**会从扩散过程的数学（加噪-去噪的马尔可夫链）到 DDPM（Ho et al., NeurIPS 2020 [A]）到 CFG（Classifier-Free Guidance）完整展开。

## 附录 A：ELBO 完整数学推导 [L3 深坑]

> ⚠️ 本附录是**纯数学补充**。动画 TA 工作中用不到，但想彻底搞懂 VAE 背后数学的可以读。
>
> **不读也完全不影响前面所有章节的理解**。

### A.1 问题设定

**目标**：最大化数据对数似然 `log p(x)`。

问题是 `p(x) = ∫ p(x|z) p(z) dz` 这个积分没法算（z 是连续高维）。

**VAE 的思路**：不直接优化 `log p(x)`，而是优化它的一个**下界**（Lower Bound）。只要下界一直在涨，`log p(x)` 肯定也在涨。

### A.2 推导过程

引入 Encoder 输出的分布 `q(z|x)`：

$$
\log p(x) = \log \int p(x|z) p(z) \, dz = \log \int q(z|x) \cdot \frac{p(x|z) p(z)}{q(z|x)} \, dz
$$

由 **Jensen 不等式**（`log` 是凹函数，所以 `log E[Y] ≥ E[log Y]`）：

$$
\log p(x) \geq \int q(z|x) \log \frac{p(x|z) p(z)}{q(z|x)} \, dz
$$

展开右边：

$$
\text{ELBO} = \underbrace{E_{q(z|x)}[\log p(x|z)]}_{\text{重建项}} - \underbrace{\text{KL}(q(z|x) \,\|\, p(z))}_{\text{KL 项}}
$$

**最大化 ELBO ⇔ 最小化 −ELBO = L_rec + KL**。这就是 VAE 的损失函数。

### A.3 KL 散度的闭式解推导

对高斯 `q = N(μ, σ²)` 和 `p = N(0, 1)`：

$$
\text{KL}(q \| p) = \int q(z) \log \frac{q(z)}{p(z)} \, dz
$$

代入两个高斯的 PDF，利用 $E_q[z] = \mu$ 和 $E_q[z^2] = \mu^2 + \sigma^2$：

$$
= -\frac{1}{2}\log(2\pi\sigma^2) - \frac{1}{2} + \frac{1}{2}\log(2\pi) + \frac{1}{2}(\mu^2 + \sigma^2)
$$

$$
= \frac{1}{2}\left(\mu^2 + \sigma^2 - \log\sigma^2 - 1\right)
$$

这就是第 3.6 节用的那个闭式解。多维时每一维独立，求和即可。

### A.4 ELBO 和对数似然的"缺口"

$$
\log p(x) = \text{ELBO}(x) + \text{KL}(q(z|x) \,\|\, p(z|x))
$$

其中 `p(z|x)` 是**真后验**（真实但不可计算），`q(z|x)` 是 encoder 学到的**近似后验**。

ELBO 和真对数似然的差距，正好等于 `q(z|x)` 和 `p(z|x)` 的 KL 距离。`q(z|x)` 学得越好，缺口越小。

**"Variational"（变分）**这个词就来自这里：用一个容易计算的 `q` 去"变分地逼近"一个不可计算的 `p`。
