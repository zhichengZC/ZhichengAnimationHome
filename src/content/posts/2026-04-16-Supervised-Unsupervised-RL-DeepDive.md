## 第一章：前置知识回顾

![fig1](/images/Supervised-Unsupervised-RL-DeepDive/fig1.png)


🧠 **本章目标**：用最快速度回顾上节课的核心概念，然后引出"三种范式"的核心问题。

### 1.1 上节课关键概念速览

上一课我们学了神经网络的基础。在深入之前，快速确认你的"行李"还在：

| 概念 | 一句话回顾 | 本课衔接点 |
|------|-----------|-----------|
| **前向传播(Forward Propagation)** | 数据从输入层流向输出层，每层做加权求和+激活 | 三种范式都用前向传播做预测 |
| **反向传播(Backpropagation)** | 从损失出发，逐层计算每个权重该怎么调 | 监督/无监督依赖反传；RL用策略梯度变体 |
| **损失函数(Loss Function)** | 衡量预测与"正确答案"之间的差距 | 监督学习的核心驱动力；RL中被"奖励"替代 |
| **激活函数(Activation Function)** | 引入非线性，让网络能学复杂模式 | 所有范式的网络都需要激活函数 |
| **梯度下降(Gradient Descent)** | 沿损失函数下降最快的方向调整权重 | 三种范式共享的优化基座 |
| **过拟合(Overfitting)** | 网络把训练数据的噪声也"背"下来了 | 三种范式都会面临过拟合问题 |

### 1.2 引出核心问题：网络"怎么学、学什么、从哪学"？

上节课我们搞明白了神经网络**内部怎么运转**——数据怎么流、误差怎么传、权重怎么调。但我们回避了三个更根本的问题：

> 1. **"学什么"**——网络学的目标是什么？学一个"正确答案"？学数据内部的结构？还是学一个"策略"？
> 2. **"从哪学"**——学习信号从哪来？人类标注的标签？数据本身的规律？还是环境给的奖惩？
> 3. **"怎么学"**——训练过程的反馈循环长什么样？

这三个问题的不同回答，定义了机器学习的三种核心范式：

| 问题 | 监督学习 | 无监督学习 | 强化学习 |
|------|---------|-----------|---------|
| 学什么？ | 输入到输出的映射关系 | 数据内部的结构/规律 | 最优行动策略 |
| 从哪学？ | 人类标注的标签 | 数据本身 | 环境反馈的奖惩 |
| 怎么学？ | 对比"预测"和"答案" | 发现隐藏模式 | 试错、探索、积累经验 |

### 1.3 三种范式的生活类比锚点

先在脑子里种三颗"类比种子"，后面每章会反复浇灌：

**🍎 监督学习 = 有老师的课堂**
> 老师出题、给标准答案、你做题、老师批改。目标：下次遇到新题也能做对。

**🔍 无监督学习 = 整理一堆没标签的照片**
> 没人告诉你这些照片该怎么分类。你自己翻看，发现"这几张都是海边的""那几张是山里的"——你自己发现了规律。

**🎮 强化学习 = 训练一只小狗**
> 小狗做了"坐下"你给零食（奖励），它咬了沙发你说"不行"（惩罚）。小狗通过不断试错，学会了拿到最多零食的行为策略。

## 第二章：监督学习（Supervised Learning）

![fig2](/images/Supervised-Unsupervised-RL-DeepDive/fig2.png)


📐 **本章目标**：彻底理解监督学习的工作原理，把它和上节课学的神经网络"接上线"。

### 2.1 核心直觉：老师给标准答案

**监督学习(Supervised Learning)** —— 用带有"正确答案"的数据来训练模型，让模型学会从输入预测输出。

这是最直觉、最好理解的方式，因为它和我们上学的学习方式几乎一模一样：

```
你的学习过程：
  1. 老师给你一道数学题（输入）和标准答案（标签）
  2. 你做出自己的答案（预测）
  3. 老师批改，告诉你差了多少（损失）
  4. 你复习改正，下次做得更好（参数更新）
  5. 期末考试：遇到没见过的新题，也能做对（泛化）
```

用机器学习术语翻译：

| 你的学习 | 监督学习对应 |
|---------|------------|
| 数学题 | **输入数据(Input Data)** |
| 标准答案 | **标签(Label / Ground Truth)** —— 人类提前标注好的"正确答案" |
| 你的答案 | **预测值(Prediction)** —— 模型的输出 |
| 老师批改 | **损失函数(Loss Function)** —— 计算预测和标签之间的差距 |
| 复习改正 | **反向传播 + 梯度下降** —— 调整网络权重 |
| 期末考新题 | **泛化(Generalization)** —— 对没见过的数据也能正确预测 |

> **关键术语**：**训练集(Training Set)** —— 用来训练模型的数据；**测试集(Test Set)** —— 训练完后评估模型的数据，模型在训练时从未见过。

### 2.2 工作原理详解

#### 数据的样子：输入-标签对

监督学习的核心燃料是 **输入-标签对(Input-Label Pair)**。每一条训练数据都长这样：

```
(输入, 标签)
(  x,    y  )
```

动画领域的具体例子：

| 任务 | 输入 x | 标签 y |
|------|--------|--------|
| 动作分类 | 一段动捕数据的关节角度序列 | "走路" / "跑步" / "跳跃" |
| 关节角度预测 | 当前帧的身体状态 + 期望速度 | 下一帧的关节角度 |
| 面部表情识别 | 一张角色面部截图 | "开心" / "悲伤" / "愤怒" |

#### 训练流程

注意——这和上节课学的完全一致，只是换了一个视角来看：

> **轮次(Epoch)** —— 把整个训练数据集从头到尾过一遍，叫做一个epoch。通常需要很多个epoch才能让模型收敛。

```python
# 监督学习训练循环（伪代码）

for epoch in range(num_epochs):              # 把所有数据过一遍叫一个 epoch
    for (x, y) in training_data:             # 取出一对 (输入, 标签)
        y_pred = model.forward(x)            # 第1步：前向传播，得到预测值
        loss = loss_function(y_pred, y)      # 第2步：计算损失（预测 vs 标签）
        gradients = loss.backward()          # 第3步：反向传播，计算梯度
        model.update_weights(gradients, lr)  # 第4步：梯度下降，更新权重
```

> 💡 **衔接上节课**：这就是上节课学的那整套流程！只不过上节课我们关注"内部机制"（梯度怎么算、权重怎么更新），这节课关注"为什么这样做"——**因为我们有标签，所以能计算损失，所以能反向传播。没有标签，这一套就走不通。** 这就是监督学习的本质特征。

### 2.3 回归 vs 分类：深化理解

上节课我们提到了MSE和交叉熵，现在用三种范式的视角再看一遍。

**回归(Regression)** —— 预测一个连续的数值。

> **直觉类比：射箭🏹** ——你在预测箭会落在靶的哪个坐标位置。偏了多少就罚多少。

- 输出：连续值（如关节角度 37.5°，位置坐标 (1.2, 3.4, 0.5)）
- 损失函数：MSE（均方误差）
- 最后一层激活：通常无激活（直接输出数值）

**分类(Classification)** —— 预测属于哪个离散类别。

> **直觉类比：选择题📝** ——A/B/C/D四个选项，你需要选出正确答案。

- 输出：每个类别的概率（如 走路:0.8, 跑步:0.15, 跳跃:0.05）
- 损失函数：交叉熵
- 最后一层激活：Sigmoid（二分类）或 Softmax（多分类）

> **Softmax(归一化指数函数)** —— 把任意数值转换为概率分布（所有输出加起来等于1）。"我把每个选项的原始分数转换成概率，让你看到每个类别有多大可能。"

| 对比项 | 回归 | 分类 |
|--------|------|------|
| 输出类型 | 连续数值 | 离散类别 |
| 损失函数 | MSE（差多少罚多少） | 交叉熵（猜错概率越大罚越重） |
| 最后一层 | 线性输出 | Sigmoid/Softmax |
| 动画示例 | 预测下一帧关节角度 | 识别当前动作类别 |

### 2.4 经典模型举例

| 模型 | 核心思想 | 复杂度 | 动画相关度 |
|------|---------|--------|-----------|
| **线性回归(Linear Regression)** | 用直线/超平面拟合数据 | ⭐ | 理解基础，实际少用 |
| **逻辑回归(Logistic Regression)** | 线性回归 + Sigmoid → 输出概率 → 二分类 | ⭐ | 简单动作分类的基准方法(Baseline)——用来和更复杂方法做对比的"及格线"方案 |
| **多层感知机(MLP)** | 多层全连接神经网络 | ⭐⭐ | 动画系统中的主力模型 |
| **CNN(卷积神经网络)** | 卷积提取空间/时序特征 | ⭐⭐⭐ | 动作识别、姿态估计（下节课详解） |

> 📝 **关键洞察**：线性回归和逻辑回归可以看作"只有一层、没有隐藏层的神经网络"——和上节课的知识无缝衔接。加上隐藏层和激活函数，就变成了MLP。

### 2.5 监督学习在动画中的应用

#### 应用一：动作标注与分类

**场景**：动捕数据库中有数万段动作片段，人工标注太慢 → 训练分类器自动标注。

```
输入：一段动作的关节角度序列（如 30帧 × 22关节 × 3轴 = 1980维向量）
标签：动画师手动标注的类别 {"Walk", "Run", "Jump", "Idle", ...}
模型：MLP 或 1D-CNN
```

训练完成后，新的动捕数据送进网络，自动输出动作类别。再也不用人工一段一段标了。

> 📄 **参考**：Holden, Saito, Komura, "A Deep Learning Framework for Character Motion Synthesis and Editing" (SIGGRAPH 2016) —— 该论文主要关注运动合成与编辑，其中使用的卷积自编码器对动作进行高层特征提取的思路，为后续的动作分类方法奠定了基础。

#### 应用二：关节角度预测（Learned Motion Matching）

**场景**：Motion Matching 需要在运行时搜索海量动作数据库，开销大 → 用神经网络"记住"搜索结果。

```
输入：角色当前状态（位置、速度、朝向）+ 玩家摇杆输入
标签：传统 Motion Matching 搜索出的最佳匹配姿态
模型：MLP（多层全连接）
损失函数：MSE（预测姿态 vs 搜索结果）
```

这是典型的**回归问题**——用监督学习训练网络，让它一次前向传播就能输出原本需要暴力搜索才能找到的最佳姿态。

> 📄 **参考**：Holden et al., "Learned Motion Matching" (SIGGRAPH 2020)
> 
> **对你的意义**：如果项目中听到"我们要把 Motion Matching 换成神经网络版本"，本质就是用监督学习做回归预测。

#### 应用三：PFNN 中的相位预测

**场景**：Phase-Functioned Neural Network，网络权重是步态相位的函数。

```
输入：角色状态 + 地形信息 + 相位变量 φ
标签：下一帧的关节姿态
特色：权重本身 = f(φ)，不同相位使用不同权重
```

核心创新：走路周期的不同阶段（脚刚离地、空中、脚落地...），网络的权重完全不同。这让同一个网络能处理步态的各个阶段。

> 📄 **参考**：Holden et al., "Phase-Functioned Neural Networks for Character Control" (SIGGRAPH 2017)
>

## 第三章：无监督学习（Unsupervised Learning）

![fig3](/images/Supervised-Unsupervised-RL-DeepDive/fig3.png)


🔍 **本章目标**：理解"没有标签也能学"是怎么做到的，以及为什么这对动画AI很重要。

### 3.1 核心直觉：没有老师，自己找规律

**无监督学习(Unsupervised Learning)** —— 从没有标签的数据中，自动发现数据内部的结构和规律。

回到我们的类比：

> 🔍 你拿到一堆**未分类的动捕文件**——几千段动作，没有任何标签。你自己翻看了一遍，发现"这些看起来都是走路""那些像是在跳"——**你在做无监督学习**。

与监督学习的核心区别：

| | 监督学习 | 无监督学习 |
|---|---------|-----------|
| 数据 | (输入, 标签) 对 | 只有输入，没有标签 |
| 目标 | 预测正确标签 | 发现数据结构 |
| 信号来源 | 标签提供的误差 | 数据本身的统计规律 |
| 类比 | 有标准答案的考试 | 自己整理归纳笔记 |

> **为什么需要无监督学习？** 因为**标注数据太贵了**。给1万段动捕数据手动标注类别、情绪、风格，可能需要动画师几个月的工作。但如果机器能自己发现这些规律呢？

### 3.2 聚类（Clustering）：自动分组

**聚类(Clustering)** —— 把相似的数据自动归为一组，不需要提前告诉它分几组或每组叫什么。

#### K-Means 算法直觉

**K-Means(K均值聚类)** —— 最经典的聚类算法。K是你指定的分组数量。

> **直觉类比：班主任分小组🏫**
> 
> 想象你是班主任，要把30个同学分成3个学习小组（K=3），原则是"兴趣相近的人在一组"：
> 1. 你随机指定3个"组长"（初始化3个聚类中心）
> 2. 每个同学找离自己兴趣最近的组长，加入那个组（分配到最近的中心）
> 3. 每个组重新选一个"兴趣最居中"的人当组长（更新中心为组内平均值）
> 4. 重复步骤2-3，直到小组成员不再变化（收敛）

用伪代码表示：

```python
# K-Means 伪代码
def kmeans(data, K):
    centers = random_pick(data, K)       # 1. 随机选K个初始中心
    while not converged:
        # 2. 每个数据点分配到最近的中心
        clusters = assign_to_nearest(data, centers)
        # 3. 更新每个中心为其组内成员的平均值
        centers = compute_means(clusters)
    return clusters, centers
```

#### 在动画中的应用

**场景**：你有5000段MoCap数据，没有标签。用K-Means自动分组：

```
输入：5000个动作片段，每段用特征向量表示
      （平均速度、根节点高度变化、关节角度方差等）
K = 10（你猜数据大概有10种主要动作类型）

输出：
  聚类0 → 1200段（发现主要是走路类）
  聚类1 → 800段（发现主要是跑步类）
  聚类2 → 350段（发现主要是跳跃类）
  聚类3 → 500段（发现是站立/Idle类）
  ...
```

你甚至不需要提前告诉它什么是"走路"——它自己就把相似的动作归在一起了。

### 3.3 降维（Dimensionality Reduction）：压缩表示

**降维(Dimensionality Reduction)** —— 把高维数据压缩到更低的维度，同时尽量保留重要信息。

> **为什么需要降维？** 一个人体骨骼有 ~22 个关节 × 3 个旋转轴 = 66 维。如果考虑位置那就是 22 × 3 + 22 × 3 = 132 维。再加上速度、加速度……维度爆炸。
>
> 但直觉告诉我们：人不可能做出 132 维空间里的任何一个点对应的姿态——大部分人体姿态其实在一个**低维的"流形(Manifold)"上**——直觉上可以理解为"高维空间中数据实际分布的低维曲面"，就像地球表面是3D空间中的一张2D曲面。降维就是找到这个低维表示。

#### PCA 直觉

**PCA(主成分分析, Principal Component Analysis)** —— 找到数据中"变化最大"的方向，保留这些方向，忽略变化小的方向。

> **直觉类比：压扁一个橄榄球🏈**
>
> 想象你有一堆数据点，它们在3D空间中排列成一个长长的橄榄球形状。PCA做的事情就是：
> 1. 找到橄榄球最长的方向（第一主成分）
> 2. 找到垂直于最长方向的第二长方向（第二主成分）
> 3. 如果你只保留这两个方向，就把3D数据"压扁"成了2D——丢失了最短方向上的少量信息，但保留了大部分结构

**在动画中**：把132维的骨骼数据做PCA，可能发现只需要 10-20 个主成分就能表示95%以上的姿态变化。这意味着人体运动的"真正自由度"远没有132那么多。

### 3.4 自编码器（Autoencoder）预告

**自编码器(Autoencoder, AE)** —— 一种用神经网络做降维的方法：把数据压缩到低维再还原，迫使网络学会保留最重要的信息。

```
输入 x → [编码器 Encoder] → 低维表示 z → [解码器 Decoder] → 重建 x'
                                ↑
                        潜空间(Latent Space)
                     也叫"瓶颈层(Bottleneck)"
```

> **直觉类比：电话传话游戏📞**
>
> 想象你要通过一根只能传3个数字的电话线，把一张照片传给对方。你必须想出一种方法，用3个数字"概括"这张照片的核心信息（编码），对方收到后再"脑补"出完整照片（解码）。
>
> 编码器 = 你压缩信息的过程
> 3个数字 = 潜空间表示
> 解码器 = 对方还原信息的过程

训练目标：让重建 x' 尽可能接近原始输入 x。

$$L_{AE} = \|x - x'\|^2$$

> 🗣️ **人话翻译**：重建出来的东西和原始输入的差距越小越好。这和监督学习不同——**标签就是输入本身**！网络要学的不是"预测答案"，而是"学会压缩和还原"。

> ⏭️ **预告**：自编码器的升级版 **VAE（变分自编码器）** 和 **VQ-VAE（矢量量化变分自编码器）** 是动画AI中最重要的生成模型之一。你已经在VQ-VAE那课中学过了，现在知道它的"祖先"在哪了！

### 3.5 无监督学习在动画中的应用

#### 应用一：动作库自动归类

用K-Means或层次聚类，把海量MoCap数据自动分组——减少手动标注工作量。

```
实际工作流：
1. 提取每段动作的特征（速度、高度、周期性等）
2. 用 K-Means 分成 N 组
3. 动画师只需要看每组的代表性样本，给整组贴标签
   （比如"这一组都是走路类"）
4. 效率提升：从标注5000段 → 只需检查50段代表样本
```

#### 应用二：动作风格发现

同一种动作（走路）可能有不同风格（自信地走、疲惫地走、偷偷摸摸地走）。无监督学习可以自动发现这些风格差异。

#### 应用三：潜空间（Latent Space）概念

**潜空间(Latent Space)** —— 数据被编码后所在的低维空间。在这个空间里，语义相近的东西距离也近。

> **这是整个生成式AI的核心概念之一。**

在动画潜空间中：
- "走路"的编码和"跑步"的编码距离较近（都是移动类）
- "走路"和"跳舞"的编码距离较远（运动模式不同）
- 在潜空间中**做插值**，可以生成"介于走路和跑步之间"的过渡动作

```
潜空间示意（简化为2D）：

        快乐跳跃 ●
                      ● 开心跑步
   悲伤行走 ●
                ● 正常走路
        ● 疲惫站立       ● 正常跑步

  [X轴：运动强度（静止→剧烈）]
  [Y轴：情绪（消极→积极）]
```

> **衔接VQ-VAE**：你在之前的课程中学过VQ-VAE会把动作编码成离散的token。那个"码本中的向量"就住在潜空间里。

## 第四章：强化学习（Reinforcement Learning）

![fig4](/images/Supervised-Unsupervised-RL-DeepDive/fig4.png)


🎮 **本章目标**：理解强化学习的独特思维方式——不需要标签、不需要预先准备的数据集，只需要一个"环境"和"奖惩规则"。
>
> ⚠️ **限定说明**：这里说的是经典的在线RL。实际应用中，很多RL方法仍然依赖参考数据——比如DeepMimic需要参考动作片段来计算奖励。此外还有"离线强化学习(Offline RL)"是直接在预收集的数据上训练的。

### 4.1 核心直觉：通过试错学习

**强化学习(Reinforcement Learning, RL)** —— 智能体(Agent)通过与环境交互，根据获得的奖励和惩罚，学习最优的行动策略。

回到类比：

> 🎮 **训练小狗**
>
> - 小狗（Agent/智能体）在客厅（Environment/环境）里
> - 它做"坐下"动作 → 你给它零食（Reward/正奖励 +1）
> - 它咬了沙发 → 你说"不行"（Reward/负奖励 -1）
> - 它走到门口 → 没有反应（Reward/中性 0）
> - 经过反复试错，小狗学会了一套策略：多做"坐下"，不咬沙发

**关键区别**：你**没有给小狗一份"标准行为手册"**（没有标签），也**没有让它看其他狗怎么做**（没有数据集）。它是通过**自己的尝试和你的反馈**学会的。

### 4.2 五要素

强化学习有五个核心概念，记住这五个就掌握了RL的基本框架：

| 要素 | 英文 | 通俗解释 | 动画类比 |
|------|------|---------|---------|
| **智能体** | Agent | 做决策的"大脑" | 虚拟角色的控制器 |
| **环境** | Environment | Agent生活的世界 | 物理模拟器（重力、碰撞） |
| **状态** | State | 当前世界的快照 | 角色的姿态+位置+速度 |
| **动作** | Action | Agent的一次操作 | 每个关节施加的力矩/扭矩 |
| **奖励** | Reward | 环境给的反馈信号（数字） | +1（成功前翻）/ -1（摔倒了） |

**还有一个核心概念：策略(Policy)**

**策略(Policy, π)** —— 在给定状态下选择什么动作的"规则"。策略就是Agent的"大脑"。

$$\pi(a|s) = P(\text{在状态 } s \text{ 下选择动作 } a)$$

> 🗣️ **人话翻译**：策略是一个函数，输入当前状态（"我现在是站着的、面朝前方、重心偏左"），输出该做什么动作（"右腿向前迈、左臂向后摆"）。RL的目标就是**找到那个能拿到最多总奖励的最优策略**。

交互循环：

```
                    ┌──────────┐
         动作 a     │          │   状态 s'
  Agent ──────────→│  环境     │──────────→ Agent
    ↑               │Environment│     ↓
    │               └──────────┘     │
    │                                │
    └────────── 奖励 r ──────────────┘
    
  每一步：
  1. Agent 观察当前状态 s
  2. Agent 根据策略 π 选择动作 a
  3. 环境根据 a 产生新状态 s' 和奖励 r
  4. Agent 用这个经验更新策略
  5. 重复
```

### 4.3 探索 vs 利用（Exploration vs Exploitation）

这是RL中最经典的两难困境：

**探索(Exploration)** —— 尝试没做过的动作，可能发现更好的策略。

**利用(Exploitation)** —— 重复已知的好策略，稳妥地拿奖励。

> **直觉类比：选餐厅🍜**
>
> 你每天中午都去同一家面馆（利用）——确定不会踩雷，但也不会发现更好的店。
> 或者你每天换一家试试（探索）——可能发现宝藏店，也可能踩到雷。
>
> 最优策略是**平衡两者**：大部分时候去已知好吃的店，偶尔试试新的。

在动画RL中：
- **探索**：让角色尝试各种奇怪的动作（可能发现一种新的走路方式）
- **利用**：重复已经学会的动作（稳定地走路不摔倒）
- 训练初期多探索（什么都不会，先瞎试），后期多利用（已经学了不少，微调就好）

### 4.4 与监督学习的关键区别

| 对比项 | 监督学习 | 强化学习 |
|--------|---------|---------|
| **数据来源** | 预先准备好的数据集 | Agent自己与环境交互生成 |
| **学习信号** | "正确答案"（标签） | "好不好"（奖励数字） |
| **反馈时机** | 每个样本立即有反馈 | 可能要做很多步之后才知道好不好 |
| **数据分布** | 固定的（数据集不变） | 变化的（策略变了→行为变了→数据变了） |
| **最大挑战** | 需要大量标注数据 | 奖励稀疏、训练不稳定、计算量巨大 |

> **延迟奖励(Delayed Reward)** —— RL中一个核心难点。想象下棋：你走的第10步可能在第50步才体现出好坏。奖励不是立即给的，这让学习困难得多。

### 4.5 强化学习在动画中的应用

#### 应用一：DeepMimic——让虚拟角色模仿真人动作

**核心思路**：给虚拟角色看一段人类MoCap动作（参考动作），让它在物理模拟器中自己学会模仿。

```
环境：物理模拟器（有重力、有地面碰撞）
状态：角色当前的关节角度、速度、位置
动作：每个关节施加的力矩（不是直接设置角度！）
奖励：当前姿态与参考动作的相似度
      r = 相似度(当前姿态, 参考动作对应帧)
```

**为什么用RL而不是监督学习？**

- 监督学习可以直接预测关节角度（比如PFNN），但它**不考虑物理**——预测的姿态可能物理上不可能（穿地、飘浮等）
- RL让角色在**物理模拟器**中学习，所有动作都必须**物理可行**——如果站不稳就会摔倒（得到负奖励）
- 结果：RL学出来的动作**自然带有物理真实感**（重心转移、身体摆动等）

> 📄 **参考**：Peng et al., "DeepMimic: Example-Guided Deep Reinforcement Learning of Physics-Based Character Skills" (SIGGRAPH 2018)

#### 应用二：为什么RL特别适合 Physics-based 动画

传统动画（Motion Matching、AnimBP）是**运动学(Kinematics)**方法——直接操纵关节角度/位置。

RL做的是**动力学(Dynamics)**方法——通过施加力和扭矩来驱动角色。

| | 运动学方法 | RL动力学方法 |
|---|---------|------------|
| 操控方式 | 直接设置关节角度 | 施加力/扭矩 |
| 物理真实感 | 需要额外处理 | 天然具备 |
| 地形适应 | 需要IK(Inverse Kinematics, 反向运动学)等后处理 | 自动适应 |
| 摔倒恢复 | 极难处理 | 可以学会 |
| 训练成本 | 低 | 非常高 |
| 运行时性能 | 快 | 较快（推理时只需前向传播） |

> **现实差距**：RL训练非常贵（需要几百万次模拟），且不稳定。目前3A游戏**几乎不直接用RL做运行时动画控制**——主流还是Motion Matching等运动学方法。但RL在**离线内容生成**（比如生成怪物走路动画数据）和**研究前沿**领域非常活跃。

#### 应用三：AMP——对抗运动先验（预告）

**AMP(Adversarial Motion Priors)** 结合了RL和**GAN(Generative Adversarial Network, 生成对抗网络)** —— 两个网络对抗训练：一个生成假数据，一个判断真假，在对抗中双方都变强 —— 的思想：

> **勘误说明**：AMP论文全称为"AMP: Adversarial Motion Priors for Stylized Physics-Based Character **Control**"（注意是"Control"非"Animation"）。

- 不直接定义奖励函数，而是训练一个**判别器(Discriminator)** —— 专门判断"真还是假"的神经网络 —— 来区分"像不像真人动作"
- 奖励 = 判别器给出的"像真人"的分数
- 这样RL学出的动作不仅物理可行，而且看起来自然

> 📄 **参考**：Peng et al., "AMP: Adversarial Motion Priors for Stylized Physics-Based Character Control" (SIGGRAPH 2021)
>
> ⏭️ **预告**：将在 Physics-RL 专题课中深入讲解。

## 第五章：三种范式大对比

![fig5](/images/Supervised-Unsupervised-RL-DeepDive/fig5.png)


📊 **本章目标**：把前三章放在一起比较，建立全局视角，并认识两种"混血"范式。

### 5.1 核心对比表

| 对比维度 | 监督学习 | 无监督学习 | 强化学习 |
|---------|---------|-----------|---------|
| **数据** | (输入, 标签) 对 | 只有输入 | 无需预备数据集 |
| **标签** | 必须有 | 不需要 | 不需要 |
| **学习信号** | 预测与标签的误差 | 数据本身的结构 | 环境给的奖惩 |
| **反馈类型** | 精确的误差值 | 隐式的（数据模式） | 标量奖励信号 |
| **优点** | 效果好、容易调试 | 不需要昂贵标注 | 能处理序列决策问题 |
| **缺点** | 标注数据昂贵 | 效果难以量化评估 | 训练不稳定、计算量大 |
| **典型动画应用** | 动作分类、姿态预测 | 动作聚类、降维 | 物理角色控制 |
| **类比** | 做有答案的试卷 | 自己整理笔记 | 训练小狗 |

### 5.2 半监督学习与自监督学习简介

真实世界的ML系统往往不是纯粹的三种范式之一，而是"混血"：

#### 半监督学习（Semi-supervised Learning）

**半监督学习(Semi-supervised Learning)** —— 少量标注数据 + 大量无标注数据，结合两者的优势。

> **类比**：你只做了10道有答案的题目（监督），但老师给了你100道没答案的练习题。你做那100道时，会参考有答案的10道的规律。

**在动画中**：你有50段手动标注的MoCap数据 + 5000段未标注的。半监督方法可以利用那5000段无标注数据来提升模型性能。

#### 自监督学习（Self-supervised Learning）

**自监督学习(Self-supervised Learning)** —— 从数据本身构造"伪标签"，然后用监督学习的方式训练。

> **类比**：遮住一句话中间的几个字，然后让自己猜被遮住的是什么——BERT就是这样训练的。

**核心技巧**：把无标签数据变成有标签数据。

| 自监督方法 | 伪标签构造方式 | 动画应用 |
|-----------|---------------|---------|
| **Masked Prediction** | 遮住一部分，预测被遮的内容 | MoMask：遮住一些运动token，预测它们 |
| **对比学习(Contrastive)** | 拉近相似样本、推开不相似样本 | 学习动作的通用特征表示 |
| **时序预测** | 用过去预测未来 | 预测下几帧的关节角度 |

> 📄 **参考**：MoMask: Generative Masked Modeling of 3D Human Motions (CVPR 2024, Guo et al.) 使用了自监督中的掩码预测方法。
>
> 💡 **重要洞察**：自监督学习是当前AI发展的**核心趋势**——大语言模型（GPT系列）、视觉基础模型（MAE）都是自监督学习。它本质上模糊了监督学习和无监督学习的边界。

### 5.3 动画任务决策树：选哪种范式？

```
你的动画AI任务
  │
  ├─ 有大量标注数据？
  │   ├─ 是 → 监督学习（精度最高）
  │   │   ├─ 预测连续值？ → 回归（MSE）
  │   │   └─ 预测类别？  → 分类（交叉熵）
  │   │
  │   └─ 否 → 有少量标注 + 大量无标注？
  │       ├─ 是 → 半监督学习 / 自监督学习
  │       └─ 否 → 无监督学习（聚类/降维/AE）
  │
  ├─ 需要物理真实感？
  │   ├─ 是 → 强化学习（DeepMimic/AMP）
  │   └─ 否 → 运动学方法可能就够了
  │
  └─ 需要生成新动作？
      ├─ 是 → 无监督/自监督（VAE/VQ-VAE/Diffusion）
      └─ 否 → 可能只需要数据库检索（Motion Matching）
```

### 6.2 监督学习 + CNN/MLP → 动作识别与运动匹配

**工业案例：Learned Motion Matching（Holden et al., SIGGRAPH 2020）**

育碧在实际项目中使用的方法。传统 Motion Matching 需要暴力搜索海量数据库 → 用监督学习训练神经网络"记住"搜索结果：

```
训练阶段（离线）：
  输入：角色当前状态（姿态 + 轨迹特征）
  标签：Motion Matching搜索出的最佳匹配姿态
  方法：监督学习，MSE损失

运行时（实时）：
  玩家按下摇杆 → 提取特征 → 网络前向传播 → 输出骨骼姿态 → 渲染
```

**工业案例：PFNN（Holden et al., SIGGRAPH 2017）**

网络权重是步态相位的函数。走路周期的不同阶段使用不同权重。本质仍然是监督学习——输入状态，标签是下一帧姿态。

### 6.3 无监督学习 + VAE → 运动生成

**衔接VQ-VAE（上一课）**：

```
运动生成流水线（VAE系）：
  大量MoCap数据（无需标签）
       │
       ▼ 无监督训练
  ┌──────────────────┐
  │  VQ-VAE Encoder  │  将动作编码为离散token
  └───────┬──────────┘
          │
          ▼
  ┌──────────────────┐
  │  语言模型 / GPT   │  学习token序列的分布（自监督）
  └───────┬──────────┘
          │ 生成新token序列
          ▼
  ┌──────────────────┐
  │  VQ-VAE Decoder  │  将token解码回动作
  └──────────────────┘
```

> **T2M-GPT、MotionGPT** 等系统就是这个流水线。你在VQ-VAE课中已经学了其中的编码-解码部分。

### 6.4 强化学习 + 策略网络 → 物理角色控制

```
DeepMimic 架构简图：
  
  物理模拟器（IsaacGym / MuJoCo / PhysX）
       │ 状态 s（关节角度、速度、位置）
       ▼
  ┌──────────────────┐
  │  策略网络(MLP)    │  输入状态 → 输出关节力矩
  │  π(a|s)          │  用PPO算法训练
  └───────┬──────────┘
          │ 动作 a（关节力矩）
          ▼
  物理模拟器执行 → 新状态 s' → 计算奖励 r
                              （与参考动作的相似度）
```

> **PPO(Proximal Policy Optimization, 近端策略优化)** —— 当前最常用的RL训练算法。直觉：每次只做小幅度的策略更新，避免"步子太大扯到蛋"。将在RL专题课中详细讲解。

### 7.1 监督学习实践：用NumPy实现简单关节角度预测

**场景**：用最简单的线性回归，根据角色速度预测膝关节角度。

```python
import numpy as np

# ==========================================
# 监督学习示例：预测膝关节角度
# ==========================================
# 场景：已知角色行走速度，预测膝关节弯曲角度
# 这是一个最简单的回归问题

# --- 1. 准备数据（模拟MoCap数据）---
np.random.seed(42)

# 输入：角色行走速度（m/s），范围 0.5~3.0
speeds = np.random.uniform(0.5, 3.0, size=100)

# 标签：膝关节最大弯曲角度（度）
# 真实关系：速度越快，弯曲角度越大（跑步时膝盖抬得更高）
# 加一点噪声模拟真实数据的不完美
true_angles = 30 + 15 * speeds + np.random.normal(0, 3, size=100)

# --- 2. 模型：最简单的线性回归 y = w*x + b ---
w = np.random.randn()  # 权重，随机初始化
b = np.random.randn()  # 偏置，随机初始化
learning_rate = 0.01   # 学习率

# --- 3. 训练循环 ---
for epoch in range(200):
    # 前向传播：预测
    predictions = w * speeds + b
    
    # 计算损失（MSE）
    loss = np.mean((predictions - true_angles) ** 2)
    
    # 反向传播：手动计算梯度
    # ∂Loss/∂w = (2/n) * Σ(pred - true) * speed
    # ∂Loss/∂b = (2/n) * Σ(pred - true)
    dw = 2 * np.mean((predictions - true_angles) * speeds)
    db = 2 * np.mean(predictions - true_angles)
    
    # 梯度下降：更新参数
    w = w - learning_rate * dw
    b = b - learning_rate * db
    
    if epoch % 50 == 0:
        print(f"Epoch {epoch}: Loss = {loss:.2f}, w = {w:.2f}, b = {b:.2f}")

# --- 4. 测试 ---
test_speed = 2.0  # 速度2.0 m/s
predicted_angle = w * test_speed + b
print(f"\n速度 {test_speed} m/s 时，预测膝关节角度 = {predicted_angle:.1f}°")
print(f"（真实关系应该约为 {30 + 15 * test_speed:.1f}°）")
```

**预期输出**（以下数值仅为示意，实际运行结果会略有不同）：
```
Epoch 0: Loss = 2847.32, w = 1.23, b = 0.45
Epoch 50: Loss = 12.45, w = 14.56, b = 30.89
Epoch 100: Loss = 9.02, w = 14.89, b = 30.12
Epoch 150: Loss = 8.98, w = 14.92, b = 30.05
训练结束后收敛

速度 2.0 m/s 时，预测膝关节角度 = 59.9°
（真实关系应该约为 60.0°）
```

> 1. 如果把 learning_rate 改成 1.0，会发生什么？（提示：Loss可能越来越大——发散了！）
> 2. 如果数据中速度和角度不是线性关系（比如有拐点），这个线性模型还能用吗？（提示：需要加激活函数和隐藏层→变成神经网络）

### 7.2 无监督学习实践：K-Means 聚类动作数据

**场景**：给定一组动作特征（速度+高度变化），自动分成几组。

```python
import numpy as np

# ==========================================
# 无监督学习示例：K-Means 聚类动作数据
# ==========================================
# 场景：有一批动作数据，用速度和跳跃高度两个特征来自动分组

np.random.seed(42)

# --- 1. 生成模拟数据（3种动作，但我们"假装不知道"标签）---
# 走路：低速、低高度变化
walk = np.random.randn(40, 2) * [0.3, 0.1] + [1.0, 0.0]
# 跑步：高速、低高度变化
run = np.random.randn(30, 2) * [0.4, 0.15] + [3.0, 0.1]
# 跳跃：中速、高高度变化
jump = np.random.randn(30, 2) * [0.3, 0.3] + [1.5, 2.0]

# 合并数据——注意：我们故意"忘记"标签
data = np.vstack([walk, run, jump])  # shape: (100, 2)
np.random.shuffle(data)  # 打乱顺序

print(f"数据集大小: {data.shape[0]} 个样本, 每个有 {data.shape[1]} 个特征")
print(f"特征含义: [行走速度(m/s), 跳跃高度(m)]")

# --- 2. K-Means 实现 ---
K = 3  # 我们猜测有3种动作类型

# 随机选3个数据点作为初始中心
indices = np.random.choice(len(data), K, replace=False)
centers = data[indices].copy()

for iteration in range(20):
    # 步骤A：分配——每个点归入最近的中心
    distances = np.array([
        np.sqrt(np.sum((data - c) ** 2, axis=1))  # 每个点到中心c的距离
        for c in centers
    ])  # shape: (K, N)
    labels = np.argmin(distances, axis=0)  # 每个点的最近中心编号
    
    # 步骤B：更新——每个中心移到其组内平均位置
    new_centers = np.array([
        data[labels == k].mean(axis=0) if np.sum(labels == k) > 0 else centers[k]
        for k in range(K)
    ])
    
    # 检查是否收敛（中心不再移动）
    shift = np.sqrt(np.sum((new_centers - centers) ** 2))
    centers = new_centers
    
    if shift < 1e-6:
        print(f"第 {iteration} 轮收敛！")
        break

# --- 3. 查看结果 ---
print(f"\n聚类结果：")
for k in range(K):
    cluster_data = data[labels == k]
    avg_speed = cluster_data[:, 0].mean()
    avg_height = cluster_data[:, 1].mean()
    print(f"  聚类 {k}: {np.sum(labels==k)} 个样本, "
          f"平均速度={avg_speed:.2f}m/s, 平均高度变化={avg_height:.2f}m")
    
    # 根据特征猜测类别
    if avg_speed < 1.5 and avg_height < 0.5:
        guess = "走路 (Walk)"
    elif avg_speed > 2.0 and avg_height < 0.5:
        guess = "跑步 (Run)"
    else:
        guess = "跳跃 (Jump)"
    print(f"         → 猜测是: {guess}")
```

**预期输出**：
```
数据集大小: 100 个样本, 每个有 2 个特征
特征含义: [行走速度(m/s), 跳跃高度(m)]
第 5 轮收敛！

聚类结果：
  聚类 0: 40 个样本, 平均速度=1.02m/s, 平均高度变化=0.01m
         → 猜测是: 走路 (Walk)
  聚类 1: 30 个样本, 平均速度=2.98m/s, 平均高度变化=0.11m
         → 猜测是: 跑步 (Run)
  聚类 2: 30 个样本, 平均速度=1.48m/s, 平均高度变化=1.97m
         → 猜测是: 跳跃 (Jump)
```

> **🤔 思考题**：
> 1. 如果把 K 改成 2，会怎样？（"走路"和"跳跃"可能被合为一组，因为它们速度相近）
> 2. 如果走路和跑步的速度范围有大量重叠，K-Means还能分得清吗？
> 3. K-Means的弱点：你必须提前指定K。如果不知道有几种动作，怎么办？（提示：Elbow Method/肘部法则）

### 7.3 强化学习概念演示：Grid World

**场景**：一个4×4网格世界，Agent从左上角出发，目标是到达右下角的"宝藏"。

```
┌────┬────┬────┬────┐
│ 起 │    │    │    │
├────┼────┼────┼────┤
│    │ ██ │    │    │   ██ = 墙壁（不能走）
├────┼────┼────┼────┤
│    │    │ ⚡ │    │   ⚡ = 陷阱（奖励 -10）
├────┼────┼────┼────┤
│    │    │    │ 🏆 │   🏆 = 宝藏（奖励 +100）
└────┴────┴────┴────┘

Agent可以做的动作：上(↑)、下(↓)、左(←)、右(→)
每走一步：奖励 -1（鼓励走最短路径）
撞墙/出界：停在原地，奖励 -1
到达陷阱：奖励 -10，回到起点
到达宝藏：奖励 +100，游戏结束
```

**Q-Learning 直觉**（不用完整实现）：

> 📖 **选读/进阶内容**：以下Q-Learning的细节第一遍阅读时可以跳过，只需理解Q表的直觉概念即可。RL专题课会详细讲解。

```
Q-Table（Q表）的概念：

一张表格，记录"在某个状态下做某个动作，预期能拿多少总奖励"

         ↑       ↓       ←       →
(0,0)  [-5.2]  [12.3]  [-5.2]  [8.7]    ← 在起点，往下走(12.3)比往右(8.7)好
(0,1)  [-3.1]  [15.8]  [-2.0]  [7.2]    ← 在(0,1)，继续往下走最好
(1,0)  [-8.0]  [4.5]   [-8.0]  [18.9]   ← 在(1,0)，往右走最好（绕过墙壁）
...

训练过程（Q-Learning伪代码）：

for episode in range(10000):  # 玩10000局
    state = (0, 0)            # 从起点开始
    
    while state != treasure:
        # 探索 vs 利用
        if random() < epsilon:
            action = random_action()      # 探索：随机选
        else:
            action = best_action(Q, state) # 利用：选Q值最大的
        
        # 执行动作
        new_state, reward = environment.step(action)
        
        # 更新Q值（核心公式）
        Q[state][action] = Q[state][action] + α * (
            reward + γ * max(Q[new_state]) - Q[state][action]
        )
        
        state = new_state
```

**Q值更新公式**的人话翻译：

$$Q(s,a) \leftarrow Q(s,a) + \alpha \Big[ r + \gamma \cdot \max_{a'} Q(s', a') - Q(s,a) \Big]$$

> 🗣️ **人话**：我对"在状态s做动作a"的价值预估，根据实际收到的奖励r和下一步最好的预估价值来调整。如果实际比预估好，就调高；如果实际比预估差，就调低。

其中：
- **α(学习率)**：每次调整多大幅度（和监督学习的学习率概念一样）
- **γ(折扣因子, Discount Factor)** —— 未来奖励打几折。γ=0.99表示"我很看重未来"，γ=0.1表示"我只看眼前"
- **ε(epsilon)** —— 探索概率。训练初期ε=0.9（90%时间随机探索），后期ε=0.1（90%时间利用经验）

> **🤔 连接动画**：Grid World的Agent "上下左右"移动 → DeepMimic中角色的"各关节施加力矩"。原理完全一样，只是状态空间从4×4网格变成了高维关节空间，动作空间从4个方向变成了连续的力矩值。
