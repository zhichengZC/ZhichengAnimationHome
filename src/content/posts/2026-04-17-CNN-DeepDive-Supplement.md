# 专题1：Conv1D 手算详解

![fig1](/images/CNN-DeepDive-Supplement/fig1.png)


## 前置知识

你在 Shader 里用过的卷积（比如边缘检测、模糊）是 **Conv2D**：一个 3×3 的核在图像上**二维滑动**。Conv1D 的思路完全一样，只是核变成了**一维的条**，沿着**一个方向**滑动。

关键区别只有一个：
- Conv2D 的核在**宽×高**两个方向滑 → 处理图像
- Conv1D 的核只在**一个方向**滑 → 处理时间序列（比如动画帧序列）

## 1.1 最简单的例子：单通道 Conv1D

### 场景设定

假设你有一个角色的**髋关节 Y 轴位置**随时间变化的数据（比如走路时的上下弹跳），采样了 7 帧：

```
时间轴 →
帧:     t0   t1   t2   t3   t4   t5   t6
髋关节Y: 1.0  0.0  2.0  3.0  0.0  1.0  1.0

输入向量: [1, 0, 2, 3, 0, 1, 1]   ← 长度=7
```

我们用一个大小为 3 的卷积核来提取"局部运动模式"：

```
卷积核(Kernel): [2, 1, 3]   ← 长度=3（可学习参数）
```

### 手算过程（padding='valid'，stride=1）

卷积核从左到右滑动，每次看 3 帧，做"逐元素相乘再求和"：

```
步骤1：核对准 t0-t2
  输入窗口: [1, 0, 2]
  卷积核:   [2, 1, 3]
  计算: 1×2 + 0×1 + 2×3 = 2 + 0 + 6 = 8
                                        ↑ 输出第1个值

步骤2：核右移1步，对准 t1-t3
  输入窗口: [0, 2, 3]
  卷积核:   [2, 1, 3]
  计算: 0×2 + 2×1 + 3×3 = 0 + 2 + 9 = 11

步骤3：核右移1步，对准 t2-t4
  输入窗口: [2, 3, 0]
  卷积核:   [2, 1, 3]
  计算: 2×2 + 3×1 + 0×3 = 4 + 3 + 0 = 7

步骤4：核右移1步，对准 t3-t5
  输入窗口: [3, 0, 1]
  卷积核:   [2, 1, 3]
  计算: 3×2 + 0×1 + 1×3 = 6 + 0 + 3 = 9

步骤5：核右移1步，对准 t4-t6
  输入窗口: [0, 1, 1]
  卷积核:   [2, 1, 3]
  计算: 0×2 + 1×1 + 1×3 = 0 + 1 + 3 = 4
```

### 完整过程一图看懂

```
输入:  [ 1    0    2    3    0    1    1 ]
        ├──┤
        [2,1,3] → 1×2+0×1+2×3 = 8
             ├──┤
             [2,1,3] → 0×2+2×1+3×3 = 11
                  ├──┤
                  [2,1,3] → 2×2+3×1+0×3 = 7
                       ├──┤
                       [2,1,3] → 3×2+0×1+1×3 = 9
                            ├──┤
                            [2,1,3] → 0×2+1×1+1×3 = 4

输出:  [ 8   11    7    9    4 ]
```

### 输出长度公式

```
输出长度 = 输入长度 - 核大小 + 1
         = 7 - 3 + 1 = 5
```

> **和 Shader 的联系**：这跟你在 Shader 里做 1D 模糊（比如沿 U 方向的高斯模糊）是一模一样的操作。不同之处在于，Shader 的核是你手动设定的（如 [0.25, 0.5, 0.25]），而 CNN 的核是**网络自己学出来的**。

## 1.2 填充（Padding）：控制输出长度

上面的例子输入 7 帧，输出只有 5 帧——边缘丢了。动画里这很麻烦：你输入 60 帧走路，卷积完变成 58 帧，帧数对不上了。

### 解决：在输入两端补零

**padding='same'**（两端补零，输出与输入等长）：

```
原始输入:    [1, 0, 2, 3, 0, 1, 1]       长度=7
补零后:    [0, 1, 0, 2, 3, 0, 1, 1, 0]   长度=9（左右各补1个0）

新增的首位计算：
  窗口 [0, 1, 0]: 0×2 + 1×1 + 0×3 = 1

新增的末位计算：
  窗口 [1, 1, 0]: 1×2 + 1×1 + 0×3 = 3

完整输出: [1, 8, 11, 7, 9, 4, 3]         长度=7 ✅ 和输入一样长了！
```

### padding 类型速查

| 类型 | 补零方式 | 输出长度（stride=1） | 用途 |
|------|---------|---------------------|------|
| `valid` | 不补零 | 输入 - 核 + 1 | 不关心边缘时 |
| `same` | 两端补零 | 与输入等长 | 大多数情况 |
| `causal` | **只在左边补零** | 与输入等长 | 实时/时序场景（后面详解） |

## 1.3 步长（Stride）：跳着看

stride=2 意味着核每次跳 2 步而不是 1 步，输出长度减半：

```
stride=2 的效果（padding='same'）：

完整输出（stride=1）:  [1,  8, 11,  7,  9,  4,  3]
                        ↑       ↑       ↑       ↑
stride=2时只保留:      [1,     11,      9,      3]

输出长度: ⌈7/2⌉ = 4
```

> **直觉**：stride 就是"下采样"。stride=2 相当于卷积之后隔一个取一个。在动画里，这就像把 60fps 的动画数据"压缩"成 30fps 的特征表示。

## 1.4 多通道 Conv1D —— 动画数据的真实场景

真实的动画数据不会只有 1 个数字，一帧数据包含**所有关节的位置**。

### 场景：简化骨骼，3 个关节，每帧 3 个 Y 坐标

```
帧:        t0    t1    t2    t3    t4
─────────────────────────────────────
髋关节Y:   1.0   0.5   1.2   0.8   1.0    ← 通道0
膝关节Y:   0.5   0.3   0.8   0.4   0.6    ← 通道1  
脚踝Y:     0.0   0.1   0.3   0.1   0.0    ← 通道2

输入形状: (5帧, 3通道)
即 PyTorch 的 (batch=1, channels=3, length=5)
```

### 多通道卷积的核是什么样

一个核大小=3、输入通道=3 的 Conv1D 核：

```
核形状: (3通道, 3时间步) —— 一个 3×3 的矩阵

       t  t+1  t+2
髋关节  [w00  w01  w02]    ← 通道0的核权重
膝关节  [w10  w11  w12]    ← 通道1的核权重
脚踝    [w20  w21  w22]    ← 通道2的核权重
```

### 计算一个输出值

以 t0-t2 为例：

```
通道0（髋关节）: [1.0, 0.5, 1.2] · [w00, w01, w02] = sum_0
通道1（膝关节）: [0.5, 0.3, 0.8] · [w10, w11, w12] = sum_1
通道2（脚踝）  : [0.0, 0.1, 0.3] · [w20, w21, w22] = sum_2

输出值 = sum_0 + sum_1 + sum_2 + bias
```

**关键理解**：多通道 Conv1D 在**同一个时间窗口内**，同时看所有通道（所有关节），然后把结果加起来。它能学到的是"**在这几帧内，各关节之间的协调模式**"。

### 多个输出通道 = 多个"检测器"

如果我们要 8 个输出通道（out_channels=8），就需要 **8 个不同的核**，每个核都是 (3通道, 3时间步)。每个核学习检测一种不同的运动模式。

```
参数总量 = out_channels × in_channels × kernel_size
         = 8 × 3 × 3 = 72 个参数
```

### PyTorch 验证

```python
import torch
import torch.nn as nn

# 模拟：1个样本，3个关节通道，5帧
x = torch.tensor([
    [[1.0, 0.5, 1.2, 0.8, 1.0],   # 髋关节Y
     [0.5, 0.3, 0.8, 0.4, 0.6],   # 膝关节Y
     [0.0, 0.1, 0.3, 0.1, 0.0]]   # 脚踝Y
])
# x.shape = (1, 3, 5) → (batch, channels, length)

conv = nn.Conv1d(
    in_channels=3,     # 3个关节
    out_channels=8,    # 8种运动模式检测器
    kernel_size=3,     # 看3帧窗口
    padding=1          # same padding，保持帧数
)

y = conv(x)
print(y.shape)  # torch.Size([1, 8, 5])
# 输出：1个样本，8个特征通道，5帧 → 帧数没变！
```

## 1.5 Conv1D 自查：你能手算吗？

**练习题**：给定输入 `[3, 1, 4, 1, 5]`，核 `[1, -1]`（核大小=2），padding='valid'，stride=1。

先自己算，答案在下面。

```
答案：
  [3,1]: 3×1 + 1×(-1) = 2
  [1,4]: 1×1 + 4×(-1) = -3
  [4,1]: 4×1 + 1×(-1) = 3
  [1,5]: 1×1 + 5×(-1) = -4

输出: [2, -3, 3, -4]

有没有发现？核 [1, -1] 计算的是相邻帧的差分！
正值=值在增大，负值=值在减小。
这就是一个"变化率检测器"——检测关节速度的方向。
```

> **这就是 Conv1D 的力量**：不同的核能自动学会检测不同的运动特征——加速、减速、周期性节拍——而不需要你手动设计。

# 专题2：膨胀卷积 + 因果卷积详解

![fig2](/images/CNN-DeepDive-Supplement/fig2.png)


## 前置知识

- Conv1D 的基本操作（专题1）
- 感受野(Receptive Field)：一个输出值能"看到"多少帧的输入

## 2.1 问题引出：普通 Conv1D 看不远

假设你在做实时角色动画，当前帧 t 的动作需要参考之前的历史帧。

用普通 Conv1D（核大小=3，stride=1）：
- 第1层：每个输出能看到 **3帧** 的输入
- 第2层：每个输出能看到 **5帧** 的输入（3 + 2）
- 第3层：每个输出能看到 **7帧** 的输入（5 + 2）
- ...
- 第n层：能看到 **2n+1 帧**

```
感受野（普通Conv1D）= 1 + n × (k-1)    其中 n=层数, k=核大小

要看到60帧前的历史：
  k=3 时: 60 = 1 + n × 2  → n = 29.5 → 需要30层！
  k=5 时: 60 = 1 + n × 4  → n = 14.75 → 需要15层
```

**30层全是 Conv1D，参数量巨大，训练困难。** 有没有办法用更少的层看到更远？

## 2.2 膨胀卷积（Dilated Convolution）：跳着看

### 核心思想

普通卷积：核看**连续**的帧  
膨胀卷积：核看**间隔**的帧，中间跳过若干帧

### 用动画帧来理解

**普通 Conv1D（dilation=1）**，核大小=3：
```
输入帧: t0  t1  t2  t3  t4  t5  t6  t7
         ↑   ↑   ↑
         ├───┤───┤
         核看这3帧（连续的）

感受野 = 3帧
```

**膨胀 Conv1D（dilation=2）**，核大小=3：
```
输入帧: t0  t1  t2  t3  t4  t5  t6  t7
         ↑       ↑       ↑
         ├───×───┤───×───┤
         核看这3帧（每隔1帧取一个）

感受野 = 5帧（从t0到t4）
```

**膨胀 Conv1D（dilation=4）**，核大小=3：
```
输入帧: t0  t1  t2  t3  t4  t5  t6  t7  t8
         ↑               ↑               ↑
         ├───×──×──×─────┤───×──×──×─────┤
         核看这3帧（每隔3帧取一个）

感受野 = 9帧（从t0到t8）
```

### 手算一个膨胀卷积

输入：`[1, 0, 2, 3, 0, 1, 1]`，核：`[2, 1, 3]`，**dilation=2**

```
dilation=2 意味着核元素间隔2个位置（跳过1个）

步骤1：核的3个元素分别对准 t0, t2, t4
  输入取: [t0, t2, t4] = [1, 2, 0]
  计算: 1×2 + 2×1 + 0×3 = 4

步骤2：核右移1步，对准 t1, t3, t5
  输入取: [t1, t3, t5] = [0, 3, 1]
  计算: 0×2 + 3×1 + 1×3 = 6

步骤3：核右移1步，对准 t2, t4, t6
  输入取: [t2, t4, t6] = [2, 0, 1]
  计算: 2×2 + 0×1 + 1×3 = 7

输出: [4, 6, 7]
```

### 膨胀卷积的感受野公式

单层膨胀卷积的感受野：

```
单层感受野 = dilation × (kernel_size - 1) + 1
```

| dilation | kernel_size=3 的感受野 | 看多少帧 |
|----------|----------------------|---------|
| 1 | 1×2+1 = 3 | 3帧 |
| 2 | 2×2+1 = 5 | 5帧 |
| 4 | 4×2+1 = 9 | 9帧 |
| 8 | 8×2+1 = 17 | 17帧 |
| 16 | 16×2+1 = 33 | 33帧 |

**关键洞察**：膨胀因子翻倍，感受野几乎翻倍，但参数量没有变！核还是只有 3 个参数。

## 2.3 TCN 的杀手锏：指数膨胀堆叠

TCN 不只用一层膨胀卷积，而是**堆叠多层**，膨胀率指数增长：d=1, 2, 4, 8, 16...

```
                    TCN 感受野增长示意（kernel_size=2）
                    
第1层 (d=1):   ●──●                              感受野=2
第2层 (d=2):   ●─────●                           感受野=4  (看4帧)
第3层 (d=4):   ●───────────●                      感受野=8  (看8帧)
第4层 (d=8):   ●───────────────────────●           感受野=16 (看16帧)

4层就能看16帧！
```

### TCN 感受野总公式

```
感受野 = 1 + (kernel_size - 1) × Σ dilation_i
                                    i=0 到 L-1

当 dilation_i = 2^i 时（最常见的配置）：

感受野 = 1 + (kernel_size - 1) × (2^L - 1)
```

### 动画实例计算

你在做游戏角色动画，30fps，需要看到过去 2 秒（60帧）的历史：

```
目标感受野 ≥ 60帧

配置1：kernel_size=2
  60 ≤ 1 + (2-1) × (2^L - 1)
  59 ≤ 2^L - 1
  2^L ≥ 60
  L ≥ 6 (因为 2^6=64)
  → 6层即可，感受野 = 1 + 1 × 63 = 64帧 ✅

配置2：kernel_size=3
  60 ≤ 1 + (3-1) × (2^L - 1)
  59 ≤ 2 × (2^L - 1)
  29.5 ≤ 2^L - 1
  2^L ≥ 30.5
  L ≥ 5 (因为 2^5=32)
  → 5层即可，感受野 = 1 + 2 × 31 = 63帧 ✅
```

对比普通 Conv1D 需要 30 层，TCN 只需要 **5-6 层**就能覆盖同样的历史！

## 2.4 因果卷积（Causal Convolution）："不能偷看未来"

### 为什么需要因果性

在游戏里，角色动画是**实时**的——当前帧的动作只能基于已经发生的帧，不能使用还没发生的未来帧。

```
普通卷积（non-causal），核大小=3：
  输出[t3] 看的是 t2, t3, t4
                           ↑ 未来帧！在实时系统中不可能拿到！

因果卷积（causal），核大小=3：
  输出[t3] 看的是 t1, t2, t3
                           ↑ 最新的是当前帧，不偷看未来 ✅
```

### 实现方式：只在左边补零

```
普通 padding='same'（两边补零）：
  [0, 1, 0, 2, 3, 0, 1, 1, 0]    ← 左右各补1个0
   ↑                          ↑

因果 padding='causal'（只左边补零）：
  [0, 0, 1, 0, 2, 3, 0, 1, 1]    ← 只在左边补2个0
   ↑  ↑
  补零数 = kernel_size - 1 = 3 - 1 = 2
```

### 因果卷积手算

输入：`[1, 0, 2, 3, 0, 1, 1]`，核：`[2, 1, 3]`，**causal padding**

```
左补2个零后: [0, 0, 1, 0, 2, 3, 0, 1, 1]

步骤1：窗口 [0, 0, 1]
  0×2 + 0×1 + 1×3 = 3       → 输出t0（只看到了t0自己）

步骤2：窗口 [0, 1, 0]
  0×2 + 1×1 + 0×3 = 1       → 输出t1（看到了t0和t1）

步骤3：窗口 [1, 0, 2]
  1×2 + 0×1 + 2×3 = 8       → 输出t2（看到了t0,t1,t2）

步骤4：窗口 [0, 2, 3]
  0×2 + 2×1 + 3×3 = 11      → 输出t3（看到了t1,t2,t3）

步骤5：窗口 [2, 3, 0]
  2×2 + 3×1 + 0×3 = 7       → 输出t4（看到了t2,t3,t4）

步骤6：窗口 [3, 0, 1]
  3×2 + 0×1 + 1×3 = 9       → 输出t5（看到了t3,t4,t5）

步骤7：窗口 [0, 1, 1]
  0×2 + 1×1 + 1×3 = 4       → 输出t6（看到了t4,t5,t6）

输出: [3, 1, 8, 11, 7, 9, 4]    长度=7 ✅ 等长且因果！
```

**验证因果性**：输出[t3]=11，它看的窗口是 [0, 2, 3]，对应原始输入的 t1, t2, t3 —— 确实没用到 t4 以后的数据 ✅

### 对比三种 padding

用同样的输入 `[1, 0, 2, 3, 0, 1, 1]` 和核 `[2, 1, 3]`：

```
valid:   [        8,  11,  7,  9,  4        ]   长度=5，边缘丢失
same:    [   1,   8,  11,  7,  9,  4,  3    ]   长度=7，对称
causal:  [   3,   1,   8, 11,  7,  9,  4    ]   长度=7，因果

注意：causal 的输出和 same 数值不同！因为看的帧不同。
```

## 2.5 因果膨胀卷积：TCN 的完整组件

把因果 + 膨胀组合起来：

```
因果膨胀卷积 (dilation=4, kernel_size=3)：

时间轴:  t0  t1  t2  t3  t4  t5  t6  t7  t8  t9  t10  t11  t12
          ↑               ↑               ↑
          t2              t6              t10   ← 输出[t10] 看这3帧
          
左补零: dilation × (kernel_size - 1) = 4 × 2 = 8个零

只看当前和过去 ✅  间隔地看更远 ✅
```

### TCN 4 层堆叠示意（kernel_size=2）

```
输入:   t0  t1  t2  t3  t4  t5  t6  t7  t8  t9  t10 t11 t12 t13 t14 t15
         ●   ●   ●   ●   ●   ●   ●   ●   ●   ●   ●   ●   ●   ●   ●   ●

d=1:     ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●  ●─●
              │        │         │         │         │         │         │
d=2:          ●────●   ●────●   ●────●   ●────●   ●────●   ●────●   ●────●
                   │              │              │              │
d=4:               ●──────────●  ●──────────●   ●──────────●   ●──────────●
                               │                              │
d=8:                            ●────────────────────────────●
                                                              ↑
                                                        输出[t15]
                                                     感受野覆盖 t0-t15
                                                     共16帧 ✅
```

> **用游戏的语言说**：第1层检测"相邻两帧的变化"（脚刚离地），第2层检测"隔帧的模式"（步态节拍），第3层检测"远距离的关联"（走→跑的过渡），第4层检测"整体趋势"（完整的运动循环）。

## 2.6 PyTorch 代码验证

```python
import torch
import torch.nn as nn

# 模拟16帧、3通道的动画数据
x = torch.randn(1, 3, 16)  # (batch, channels, frames)

# TCN 的 4 层因果膨胀卷积
layers = []
for i in range(4):
    d = 2 ** i  # dilation = 1, 2, 4, 8
    # 因果卷积 = 手动在左边补零
    padding = d * (3 - 1)  # dilation × (kernel_size - 1)
    layers.append(nn.Conv1d(3, 3, kernel_size=3, dilation=d, padding=padding))

# 注意：PyTorch 的 padding 参数是两边对称补零的
# 要实现因果卷积，需要补零后截断右侧多余部分
# 实际实现中通常用 nn.ConstantPad1d 单独在左边补零

# 演示手动因果填充
class CausalConv1d(nn.Module):
    def __init__(self, in_ch, out_ch, kernel_size, dilation=1):
        super().__init__()
        self.pad_len = dilation * (kernel_size - 1)  # 左侧补零数
        self.conv = nn.Conv1d(in_ch, out_ch, kernel_size, 
                              dilation=dilation, padding=0)  # 不自动补零
    
    def forward(self, x):
        # 只在左边补零（因果！）
        x = torch.nn.functional.pad(x, (self.pad_len, 0))
        return self.conv(x)

# 搭建4层因果膨胀卷积
tcn = nn.Sequential(
    CausalConv1d(3, 8, kernel_size=3, dilation=1),
    nn.ReLU(),
    CausalConv1d(8, 8, kernel_size=3, dilation=2),
    nn.ReLU(),
    CausalConv1d(8, 8, kernel_size=3, dilation=4),
    nn.ReLU(),
    CausalConv1d(8, 8, kernel_size=3, dilation=8),
)

y = tcn(x)
print(f"输入: {x.shape}")   # (1, 3, 16)
print(f"输出: {y.shape}")   # (1, 8, 16) — 帧数不变！因果！
```

## 2.7 感受野计算自查

**练习题**：你需要设计一个 TCN 处理 60fps 动画数据，要求能看到过去 1 秒（60帧）的历史。kernel_size=3，膨胀因子按 2^i 增长。最少需要几层？

```
答案：

感受野 = 1 + (k-1) × (2^L - 1)
60 ≤ 1 + 2 × (2^L - 1)
59 ≤ 2 × (2^L - 1)
29.5 ≤ 2^L - 1
30.5 ≤ 2^L
L ≥ 5 (因为 2^5 = 32)

5层，感受野 = 1 + 2 × 31 = 63帧 ≥ 60帧 ✅

每层膨胀率: d=1, 2, 4, 8, 16
参数量（单通道）: 5层 × 3参数/核 = 仅15个参数就能看63帧！
```

# 专题3：残差连接的梯度流

![fig3](/images/CNN-DeepDive-Supplement/fig3.png)


## 前置知识

- 什么是梯度下降（用导数的方向调参数）—— 第1课已学
- 什么是反向传播（从输出往输入逐层算梯度）—— 第1课已学
- 链式法则：复合函数求导 = 各环节导数相乘

## 3.1 问题：为什么深层网络训不动？

### 先理解链式法则

假设你有一条 3 层的处理链：

```
输入x → [第1层 f₁] → [第2层 f₂] → [第3层 f₃] → 输出y → 损失L
```

要更新第1层的参数，需要计算"损失 L 对第1层参数的梯度"：

```
∂L/∂w₁ = (∂L/∂y) × (∂f₃/∂h₂) × (∂f₂/∂h₁) × (∂f₁/∂w₁)
           ↑           ↑            ↑            ↑
         最后一层     第3层的      第2层的      第1层的
         的梯度      局部梯度     局部梯度     局部梯度
```

**关键**：这些梯度是**连续相乘**的。

### 梯度消失的直觉

想象每层的局部梯度平均值是 0.3（小于1）：

```
3层:  0.3 × 0.3 × 0.3 = 0.027            ← 还行
10层: 0.3^10 = 0.0000059                   ← 第1层几乎学不动了
20层: 0.3^20 = 0.0000000000348             ← 完全死了
50层: 0.3^50 ≈ 7 × 10^(-27)               ← 比原子还小
```

> **直觉类比**：想象你在 UE5 蓝图里连了 50 个节点，每个节点把数值乘以 0.3。最后一个节点输出的数字还能有意义吗？几乎就是 0 了。这就是深层网络的第1层面对的困境——从末端传回来的梯度信号已经衰减到几乎为零。

### He et al. 的实验证据

何恺明等人在 ResNet 论文中做了一个关键实验（He et al., 2016, CVPR）：

```
CIFAR-10 数据集上的训练误差:

普通网络 (plain)：
  20层: 训练误差 ≈ 5%
  56层: 训练误差 ≈ 7%   ← 更深反而更差！

不是过拟合（过拟合是训练好但测试差），
而是训练本身就失败了——优化器找不到好的解。
```

## 3.2 残差连接如何解决问题

### 残差块的结构

```
        输入 x
        │
        ├─────────────────────┐
        │                     │
        ↓                     │
  ┌───────────┐               │
  │  Conv 层  │               │ （恒等映射
  │  + BN     │               │  Identity
  │  + ReLU   │               │  Shortcut）
  ├───────────┤               │
  │  Conv 层  │               │
  │  + BN     │               │
  └─────┬─────┘               │
        │                     │
        ↓                     │
      F(x)     +              x
        │                     │
        └──────── ⊕ ──────────┘
                 │
                 ↓
            H(x) = F(x) + x     ← 输出
                 │
                 ↓
               ReLU
```

**用一句话说**：输出 = 经过卷积处理的结果 + 原始输入。

### 这个"+"就是全部的秘密

## 3.3 梯度流的数学推导（一步步来）

### 前向传播

```
H(x) = F(x) + x

其中：
  x = 输入（来自上一层）
  F(x) = 两层 Conv 的变换结果
  H(x) = 这个残差块的输出
```

### 反向传播：关键的求导

损失 L 对输入 x 的梯度：

```
∂L/∂x = ∂L/∂H × ∂H/∂x
```

现在算 ∂H/∂x：

```
H(x) = F(x) + x

∂H/∂x = ∂F(x)/∂x + ∂x/∂x
       = ∂F(x)/∂x + 1        ← 这个 +1 是关键！！！
```

所以：

```
∂L/∂x = ∂L/∂H × (∂F(x)/∂x + 1)
       = ∂L/∂H × ∂F(x)/∂x  +  ∂L/∂H × 1
       = ∂L/∂H × ∂F(x)/∂x  +  ∂L/∂H
         ↑                       ↑
         通过卷积层的梯度         直接传过来的梯度
         (可能很小)              (不会衰减！)
```

### 为什么这解决了梯度消失

**没有残差连接时**：

```
梯度 = ∂L/∂H × ∂F(x)/∂x

如果 ∂F(x)/∂x ≈ 0.3，经过N层：
总梯度 ≈ 0.3^N → 趋近于0（消失！）
```

**有残差连接时**：

```
梯度 = ∂L/∂H × ∂F(x)/∂x  +  ∂L/∂H

即使 ∂F(x)/∂x ≈ 0（第一项消失了），
第二项 ∂L/∂H 仍然完好无损地传回来了！
```

> **直觉类比**：想象你在一栋 50 层的大楼里传递消息（梯度）。  
> - **没有残差连接**：消息只能一层一层往下传，每层传话都会损失信息。传到第1层，消息已经模糊到没人能理解了。  
> - **有残差连接**：每一层都有一部电梯直达楼顶！即使楼梯传话的通道堵了（卷积层梯度很小），电梯通道（恒等映射）仍然能把清晰的消息直接送到任何一层。

## 3.4 多层残差块的梯度流

假设有 3 个残差块串联：

```
x₀ → [残差块1] → x₁ → [残差块2] → x₂ → [残差块3] → x₃ → 损失L

其中每个残差块: xᵢ₊₁ = F(xᵢ) + xᵢ
```

从 x₃ 回传到 x₀ 的梯度：

```
∂L/∂x₀ = ∂L/∂x₃ × ∂x₃/∂x₂ × ∂x₂/∂x₁ × ∂x₁/∂x₀

其中每个 ∂xᵢ₊₁/∂xᵢ = ∂F(xᵢ)/∂xᵢ + 1

展开乘积（用更紧凑的记号，令 fᵢ = ∂F(xᵢ)/∂xᵢ）：

∂L/∂x₀ = ∂L/∂x₃ × (f₂+1) × (f₁+1) × (f₀+1)
```

把 (f₂+1)(f₁+1)(f₀+1) 展开：

```
= 1                            ← 纯恒等路径（跳过所有块）
  + f₀                         ← 只经过块1
  + f₁                         ← 只经过块2
  + f₂                         ← 只经过块3
  + f₀f₁                       ← 经过块1和块2
  + f₀f₂                       ← 经过块1和块3
  + f₁f₂                       ← 经过块2和块3
  + f₀f₁f₂                     ← 经过所有块
```

**关键观察**：这里有 2³=8 条梯度路径，其中**始终有一条纯恒等路径（=1）**，不管网络多深，这条路径的梯度永远不会衰减！

```
即使 f₀ = f₁ = f₂ ≈ 0（卷积层梯度全部接近0），
总梯度 ≈ ∂L/∂x₃ × 1 = ∂L/∂x₃

梯度至少能原封不动地从末端传到起点！
```

推广到 N 个残差块：有 2^N 条梯度路径，总有不衰减的路径存在。

## 3.5 "残差学习"的另一个直觉

不只是梯度流——残差连接还改变了网络要学的东西。

### 不用残差：学完整变换

```
目标: 让网络学 H(x) = 某个复杂变换

网络需要从零学会整个变换——很难
```

### 用残差：只学"需要修正多少"

```
H(x) = F(x) + x

F(x) = H(x) - x ← 网络只需要学"输出和输入的差异"

如果理想的变换接近恒等（输出≈输入），那 F(x) ≈ 0
学习"接近零的修正量"比学习"完整变换"容易得多
```

> **用动画的语言说**：假设你有一个基础走路动画（输入 x），想得到一个走路+微调的动画（输出 H(x)）。  
> - 不用残差：网络需要**从零重建**整个走路动画  
> - 用残差：网络只需要学**"微调量"**（比如脚步稍微抬高一点）—— 这简单多了

## 3.6 残差连接的实际影响

## 3.7 自查练习

**问题**：假设有 2 个残差块，每个块的卷积梯度 f ≈ 0.1。

不用残差连接时，从末端传到起点的梯度是多少？
用残差连接时呢？

```
答案：

不用残差:
  梯度 = f₁ × f₀ = 0.1 × 0.1 = 0.01（衰减了100倍）

用残差:
  梯度 = (f₁+1)(f₀+1) = 1.1 × 1.1 = 1.21
  
  展开: 1 + f₀ + f₁ + f₀f₁ = 1 + 0.1 + 0.1 + 0.01 = 1.21

  几乎无衰减！纯恒等路径贡献了1.0，保底了。
```
