## 第1章：什么是神经网络

![fig1](/images/NeuralNetwork-Basics-DeepDive/fig1.png)

### 前置知识
无。从零开始。

### 1.1 从生物神经元说起

你的大脑里有大约860亿个**神经元（Neuron）**。每个神经元的工作方式大致是这样的：

```
其他神经元的信号 ──→ [树突] 接收信号
                      ↓
                 [细胞体] 汇总所有输入信号
                      ↓
              如果总信号强度超过某个阈值 → [轴突] 发出信号给下一个神经元
              如果不够强 → 不发信号
```

**关键直觉**：神经元是一个"判断器"——接收多个输入，汇总后决定是否"激活"。

> **重要提醒**：人工神经网络是**受生物启发**，但不是精确模拟大脑。就像飞机受鸟类启发但不是在扇翅膀一样。

来源: 3Blue1Brown "But what is a neural network?" (YouTube, Chapter 1)

### 1.2 人工神经元（Artificial Neuron）

把生物神经元的核心逻辑抽象成数学：

```
人工神经元的工作流程：

输入 x₁ ──(×权重 w₁)──┐
输入 x₂ ──(×权重 w₂)──┤
输入 x₃ ──(×权重 w₃)──┼──→ [求和: Σ(wᵢxᵢ) + b] ──→ [激活函数 f] ──→ 输出 y
  ...                  │
输入 xₙ ──(×权重 wₙ)──┘

数学公式：
y = f(w₁x₁ + w₂x₂ + ... + wₙxₙ + b)
简写：y = f(w·x + b)
```

**每个概念对应什么**：

| 数学符号 | 含义 | 生物类比 | 动画类比 |
|---------|------|---------|---------|
| x₁, x₂, ... | **输入(Inputs)** — 来自外部或上一层的数据 | 树突接收的信号 | 角色当前的关节角度、速度 |
| w₁, w₂, ... | **权重(Weights)** — 每条连接的"重要性" | 突触的连接强度 | 每个输入特征对输出的影响大小 |
| b | **偏置(Bias)** — 调整激活的基准线 | 激活阈值的高低 | 一个额外的可调参数 |
| f | **激活函数(Activation Function)** — 引入非线性 | 是否激活的判断 | 详见第3章 |
| y | **输出(Output)** — 这个神经元的结果 | 轴突发出的信号 | 预测的下一帧某个关节角度 |

**一个具体例子**：

假设你要判断一个怪物是否应该攻击玩家。输入是：
- x₁ = 与玩家的距离（0~100米）
- x₂ = 怪物当前血量百分比（0~1）
- x₃ = 玩家是否在视野内（0或1）

```
一个可能的神经元：
y = f(-0.5 × 距离 + 0.3 × 血量 + 2.0 × 在视野内 + 0.1)

解读：
  距离近（x₁小）→ -0.5 × 小值 ≈ 接近0 → 倾向攻击
  血量高（x₂大）→ +0.3 × 大值 → 倾向攻击
  在视野内（x₃=1）→ +2.0 → 强烈倾向攻击

权重w反映了"距离比血量重要"，"视野内最重要"
```

来源: 3Blue1Brown "But what is a neural network?" (Chapter 1)

### 1.3 感知机（Perceptron）— 最早的人工神经元

**感知机(Perceptron)** 是1958年由Frank Rosenblatt提出的，是最简单的人工神经元：

```
感知机 = 线性加权求和 + 阶跃函数

阶跃函数：如果 Σ(wᵢxᵢ) + b > 0 → 输出1（激活）
          如果 Σ(wᵢxᵢ) + b ≤ 0 → 输出0（不激活）
```

**感知机能做什么**：
- ✅ AND（与）：两个输入都为1时输出1
- ✅ OR（或）：至少一个输入为1时输出1
- ❌ XOR（异或）：两个输入不同时输出1 — **单个感知机做不到！**

XOR问题暴露了单层神经元的根本局限 → 这就是为什么我们需要**多层网络**。

来源: Rosenblatt, "The Perceptron: A Probabilistic Model for Information Storage and Organization in the Brain" (1958); Michael Nielsen "Neural Networks and Deep Learning" Chapter 1 (免费在线书: neuralnetworksanddeeplearning.com)

### 1.4 从单个神经元到网络

把多个神经元连接起来，就形成了**神经网络(Neural Network)**：

```
输入层          隐藏层1        隐藏层2        输出层
(Input)        (Hidden)       (Hidden)       (Output)

 x₁ ──┐     ┌── h₁ ──┐     ┌── h₅ ──┐     ┌── y₁
       ├─→  ├── h₂ ──├─→  ├── h₆ ──├─→  ├── y₂
 x₂ ──┤     ├── h₃ ──┤     ├── h₇ ──┤     └── y₃
       ├─→  └── h₄ ──┘─→  └── h₈ ──┘
 x₃ ──┘

全连接(Fully Connected)：每个神经元与下一层的每个神经元都相连
```

| 术语 | 含义 |
|------|------|
| **输入层(Input Layer)** | 接收原始数据，不做计算 |
| **隐藏层(Hidden Layer)** | 中间层，做实际的特征提取和变换 |
| **输出层(Output Layer)** | 产生最终预测结果 |
| **深度学习(Deep Learning)** | 使用多个隐藏层的神经网络 |
| **参数(Parameters)** | 所有权重w和偏置b的总和 |

**参数量怎么算**：
```
一个 784 → 16 → 16 → 10 的网络：
  第1层连接数: 784 × 16 + 16(偏置) = 12,560
  第2层连接数: 16 × 16 + 16 = 272
  第3层连接数: 16 × 10 + 10 = 170
  总参数: 13,002
```

来源: 3Blue1Brown "But what is a neural network?" (Chapter 1)

## 第2章：前向传播（Forward Propagation）

![fig2](/images/NeuralNetwork-Basics-DeepDive/fig2.png)

### 前置知识
第1章的神经元和网络结构。

### 2.1 数据如何流过网络

**前向传播(Forward Propagation)** — 数据从输入层到输出层的一次完整计算过程。

```
直觉：像流水线一样，原材料（输入）进入第一道工序（第一层），
      加工后传给第二道工序（第二层），一直到出成品（输出）。
      中间不回头，只往前走 → 所以叫"前向"传播。
```

逐层计算：
```
输入 x
  ↓
第1层: a⁽¹⁾ = f(W⁽¹⁾ · x + b⁽¹⁾)     ← 权重矩阵W × 输入 + 偏置，过激活函数
  ↓
第2层: a⁽²⁾ = f(W⁽²⁾ · a⁽¹⁾ + b⁽²⁾)   ← 权重矩阵W × 上一层输出 + 偏置，过激活函数
  ↓
输出: ŷ = a⁽²⁾                          ← 最后一层的输出就是预测结果
```

**符号约定**：
- a⁽ˡ⁾ = 第l层的**激活值(activation)**，就是这层所有神经元的输出
- W⁽ˡ⁾ = 第l层的**权重矩阵**
- b⁽ˡ⁾ = 第l层的**偏置向量**
- f = **激活函数**（下一章详讲）

### 2.2 一个完整的走通示例

以手写数字识别（MNIST）为例——这是ML领域的"Hello World"：

```
任务：输入一张手写数字图片(28×28像素)，输出它是0-9中的哪个数字

网络结构：784 → 16 → 16 → 10

第0步：输入
  x = [0.0, 0.0, ..., 0.8, 0.9, ..., 0.0]  (784个像素值，0=黑，1=白)

第1步：第1隐藏层
  z⁽¹⁾ = W⁽¹⁾·x + b⁽¹⁾        (784×16的矩阵 × 784维向量 + 16维偏置 → 16维)
  a⁽¹⁾ = ReLU(z⁽¹⁾)           (对每个元素取max(0, x) → 16维)

第2步：第2隐藏层
  z⁽²⁾ = W⁽²⁾·a⁽¹⁾ + b⁽²⁾     (16×16的矩阵 × 16维 + 16维 → 16维)
  a⁽²⁾ = ReLU(z⁽²⁾)           (→ 16维)

第3步：输出层
  z⁽³⁾ = W⁽³⁾·a⁽²⁾ + b⁽³⁾     (16×10的矩阵 × 16维 + 10维 → 10维)
  ŷ = Softmax(z⁽³⁾)           (→ 10维概率，加起来=1)

结果：ŷ = [0.01, 0.02, 0.05, 0.80, 0.02, 0.03, 0.02, 0.03, 0.01, 0.01]
                                ↑
                        模型认为这是数字"3"（80%置信度）
```

### 2.3 动画TA类比

**PFNN (Holden et al., SIGGRAPH 2017)** 的前向传播也是类似的过程：

```
PFNN的前向传播：
  输入 x = [角色当前姿态(关节角度) + 地形高度图 + 目标轨迹]
    ↓
  隐藏层1: a⁽¹⁾ = ELU(W⁽¹⁾(φ) · x + b⁽¹⁾(φ))   ← 注意：权重随步态相位φ变化！
    ↓
  隐藏层2: a⁽²⁾ = ELU(W⁽²⁾(φ) · a⁽¹⁾ + b⁽²⁾(φ))
    ↓
  输出 ŷ = [下一帧的关节角度 + 根节点位移 + 相位增量Δφ]
```

**一次前向传播 ≈ 0.5毫秒**，可以实时运行在游戏中。

来源: Holden, Komura, Saito. "Phase-Functioned Neural Networks for Character Control." SIGGRAPH 2017.

## 第3章：激活函数（Activation Functions）

![fig3](/images/NeuralNetwork-Basics-DeepDive/fig3.png)

### 前置知识
第1章的人工神经元公式 y = f(w·x + b)，第2章的前向传播。

### 3.1 为什么需要非线性？

**这是整个深度学习最关键的一个洞察之一。**

如果没有激活函数（或者只用线性函数 f(x) = x），那么不管你叠多少层：

```
第1层：z₁ = W₁·x + b₁
第2层：z₂ = W₂·z₁ + b₂ = W₂·(W₁·x + b₁) + b₂ = (W₂·W₁)·x + (W₂·b₁ + b₂)

化简后：z₂ = W'·x + b'   ← 仍然是一个线性变换！
```

**多层线性变换 = 一层线性变换。** 层数再多也没用。

```
动画类比：
  线性网络 = 只能做线性插值（直线）
  加了非线性 = 能拟合任意曲线（贝塞尔曲线、样条、任何形状）
  
  你的动画曲线（缓入缓出、回弹、预期/过冲）全是非线性的！
  所以网络必须有非线性才能学到这些。
```

来源: Stanford CS231n Lecture 6 "Training Neural Networks I"

### 3.2 常用激活函数

#### Sigmoid

```
σ(x) = 1 / (1 + e⁻ˣ)

输出范围: (0, 1) — 可以理解为"概率"

形状:
         1 ┤      ___________
           │    /
     0.5 ┤───/
           │ /
         0 ┤/___________
           ├────────────→ x
```

- ✅ 输出有概率含义
- ❌ **梯度消失**：当x很大或很小时，梯度接近0 → 深层网络训练困难
- ❌ 输出不是零中心的（都是正数）
- 📍 现在主要用于**输出层的二分类**

#### Tanh

```
tanh(x) = (eˣ - e⁻ˣ) / (eˣ + e⁻ˣ)

输出范围: (-1, 1) — 零中心

形状:
         1 ┤      ___________
           │    /
         0 ┤───/
           │ /
        -1 ┤/___________
           ├────────────→ x
```

- ✅ 比Sigmoid好：零中心输出
- ❌ 仍有梯度消失问题

#### ReLU（Rectified Linear Unit）— 目前最常用

```
ReLU(x) = max(0, x)

输出范围: [0, +∞)

形状:
           │      /
           │    /
           │  /
         0 ┤/___________
           ├────────────→ x
```

- ✅ **计算极快**（只是一个max操作）
- ✅ **缓解梯度消失**（正区间梯度恒为1）
- ✅ **稀疏激活**（部分神经元输出0 → 更高效）
- ❌ **Dead ReLU**：如果某个神经元的输入总是负数，梯度永远为0 → 这个神经元"死了"
- 📍 **隐藏层的默认选择**

#### 选择指南

| 场景 | 推荐激活函数 | 理由 |
|------|------------|------|
| 隐藏层（默认） | **ReLU** | 训练快、效果好 |
| 二分类输出层 | **Sigmoid** | 输出(0,1)概率 |
| 多分类输出层 | **Softmax** | 输出概率分布（和为1） |
| 回归输出层 | **无激活（线性）** | 不限制输出范围 |
 
来源: Stanford CS231n Lecture 6; Goodfellow et al. "Deep Learning" Chapter 6

## 第4章：损失函数（Loss Functions）

![fig4](/images/NeuralNetwork-Basics-DeepDive/fig4.png)

### 前置知识
前向传播（第2章），知道网络能产生预测输出ŷ。

### 4.1 什么是"损失"？

**损失(Loss)** = 模型的预测值 ŷ 与真实答案 y 之间的"差距"。

```
直觉类比：射箭
  
  靶心 = 真实答案 y
  你射中的位置 = 模型预测 ŷ
  损失 = 箭离靶心有多远
  
  训练 = 不断调整姿势（权重），让箭越来越靠近靶心（损失越来越小）
```

**训练的目标就是：找到一组权重和偏置，使得损失尽可能小。**

来源: 3Blue1Brown "Gradient descent, how neural networks learn" (Chapter 2)

### 4.2 均方误差 MSE（Mean Squared Error）— 回归用

```
L = (1/N) × Σ(yᵢ - ŷᵢ)²

翻译成人话：预测值和真实值之间差距的平方的平均值
```

- **适用场景**：预测连续数值（价格、角度、坐标...）
- **为什么用平方**：① 让正负误差不互相抵消 ② 放大大误差的惩罚 ③ 数学上更好求导
- **动画中的应用**：PFNN预测下一帧关节角度时，就是用MSE衡量预测姿态和真实姿态的差距

```
PFNN的损失函数（简化版）：

L = ||预测的下一帧姿态 - 真实的下一帧姿态||²
  = Σ(预测关节角度ᵢ - 真实关节角度ᵢ)²
```

### 4.3 交叉熵损失 Cross-Entropy — 分类用

```
多分类交叉熵:
L = -Σ yᵢ × log(ŷᵢ)

其中 yᵢ 是真实标签（one-hot），ŷᵢ 是模型预测的概率
```

**直觉**：惩罚模型对正确答案给出的低置信度。

```
例子：真实答案是"猫"(y = [1, 0, 0])

如果模型预测：ŷ = [0.9, 0.05, 0.05] → L = -log(0.9) = 0.105 → 小损失，很好
如果模型预测：ŷ = [0.1, 0.5, 0.4]  → L = -log(0.1) = 2.303 → 大损失，很差
如果模型预测：ŷ = [0.01, 0.5, 0.49] → L = -log(0.01) = 4.605 → 巨大损失！
```

- **适用场景**：分类问题
- **动画中的应用**：T2M-GPT预测下一个运动token时，就是用交叉熵（从512个码字中选一个 = 512分类问题）

来源: Stanford CS231n Lecture 3 "Loss Functions and Optimization"

### 4.4 损失景观（Loss Landscape）

**损失景观(Loss Landscape)** — 把所有可能的参数组合画成一个"地形图"：

```
想象一片山地：
  - 横轴/纵轴 = 不同的权重值
  - 高度 = 对应的损失值
  
  山峰 = 损失很大的参数组合
  山谷 = 损失很小的参数组合
  最低点 = 我们想找的最优参数
  
  训练 = 在这片山地上寻找最低点的过程
```

来源: Li et al. "Visualizing the Loss Landscape of Neural Nets." NeurIPS 2018.

## 第5章：反向传播（Backpropagation）

![fig5](/images/NeuralNetwork-Basics-DeepDive/fig5.png)

### 前置知识
前向传播（第2章）、损失函数（第4章）。需要知道导数的直觉含义（变化率）。

### 5.1 核心问题

前向传播让我们得到了预测 ŷ 和损失 L。

现在的问题是：**如何知道该调整哪个权重、朝哪个方向调、调多少？**

答案是计算**梯度(Gradient)**：

```
梯度 = ∂L/∂w = 损失L对权重w的偏导数

直觉：如果我把这个权重稍微增大一点点，损失会怎么变？
  - 梯度是正的 → 增大权重会让损失增大 → 应该减小权重
  - 梯度是负的 → 增大权重会让损失减小 → 应该增大权重
  - 梯度接近0 → 这个权重对损失影响不大 → 不怎么需要调
```

来源: 3Blue1Brown "Backpropagation, what is it really doing?" (Chapter 3)

### 5.2 链式法则（Chain Rule）

神经网络是一个复合函数的层层嵌套：

```
L = Loss(a⁽³⁾(a⁽²⁾(a⁽¹⁾(x))))

就像俄罗斯套娃：每一层都是上一层的"外壳"
```

**链式法则**允许我们逐层求导：

```
df/dx = df/dg × dg/dx

例如：f(g(x)) = sin(x²)
  df/dx = cos(x²) × 2x = df/dg × dg/dx
```

**反向传播 = 从输出层开始，逆向逐层应用链式法则**：

```
前向传播（从左到右）：
  x → a⁽¹⁾ → a⁽²⁾ → L

反向传播（从右到左）：
  ∂L/∂a⁽²⁾ ← ∂L/∂a⁽¹⁾ ← ∂L/∂W⁽¹⁾

每一步只需要"局部导数"乘以"传过来的梯度"
```

### 5.3 3Blue1Brown 的类比

3Blue1Brown 用了一个很好的类比：

```
你在黑暗中调一架巨大的钢琴：
  - 你弹了一个和弦（前向传播）
  - 和弦听起来不对（损失很大）
  - 你需要知道是哪根弦太紧或太松（计算梯度）
  - 从最终的声音出发，反向追溯到每根弦的贡献（反向传播）
  - 然后微调每根弦（梯度下降更新权重）
```

**实际中你不需要手写反向传播**——PyTorch/TensorFlow会自动完成！你只需要定义前向传播，框架帮你自动求导。

来源: 3Blue1Brown "Backpropagation, what is it really doing?" (Chapter 3); "Backpropagation calculus" (Chapter 4)

### 5.4 梯度消失与梯度爆炸

**梯度消失(Vanishing Gradients)**：
- 在深层网络中，梯度通过链式法则层层相乘
- 如果每层的梯度都小于1（比如Sigmoid的梯度最大0.25），N层后梯度 ≈ 0.25^N → 接近0
- 结果：前面几层几乎学不到东西
- 解决：ReLU激活函数（正区间梯度=1）、残差连接(ResNet)

**梯度爆炸(Exploding Gradients)**：
- 如果每层的梯度都大于1，N层后梯度指数增长
- 结果：权重更新过大，模型发散
- 解决：梯度裁剪(Gradient Clipping)、适当的权重初始化

来源: Stanford CS231n Lecture 6; Goodfellow et al. "Deep Learning" Chapter 8

## 第6章：梯度下降（Gradient Descent）

![fig6](/images/NeuralNetwork-Basics-DeepDive/fig6.png)

### 前置知识
损失函数（第4章）、梯度的概念（第5章）。

### 6.1 蒙眼下山

```
想象你蒙着眼站在一座山上，目标是走到最低点（损失最小值）：

1. 摸一下脚下的地面，感受哪个方向最陡（计算梯度）
2. 朝最陡的下坡方向走一小步（更新权重）
3. 重复，直到感觉已经到了谷底（损失不再降低）

这就是梯度下降！
```

**权重更新公式**：

```
w_new = w_old - η × ∂L/∂w

η (eta) = 学习率 = 每一步走多远
∂L/∂w = 梯度 = 最陡上升方向
负号 = 朝相反方向走（下坡）
```

来源: 3Blue1Brown "Gradient descent, how neural networks learn" (Chapter 2)

### 6.2 学习率（Learning Rate）

**学习率(Learning Rate, η)** — 每次更新权重时走的"步长"：

```
太大 (η = 1.0):
  ╲      ╱╲     ← 跳来跳去，越过最低点，甚至发散
   ╲    ╱  ╲
    ╲  ╱    ╲
     ╲╱

太小 (η = 0.00001):
  ╲
   •
   •  ← 像蜗牛一样慢慢爬，可能永远到不了最低点
   •

刚好 (η = 0.001):
  ╲
   ╲
    ╲
     ╲__●  ← 稳定收敛到最低点
```

**经验值**：
- Adam优化器默认 η = 0.001，大多数情况下不需要调
- 典型范围：1e-4 ~ 1e-2

### 6.3 SGD / Mini-batch / Adam

| 方法 | 每次用多少数据计算梯度 | 特点 |
|------|---------------------|------|
| **Batch GD** | 全部训练数据 | 稳定但慢 |
| **SGD** | 1个样本 | 快但噪声大 |
| **Mini-batch GD** | 32/64/128个样本 | **实际标准做法** |

**Adam(Adaptive Moment Estimation)** — 目前最常用的优化器：

```
Adam = Momentum + RMSProp

Momentum: 记住之前走的方向，有"惯性"（不容易被单个噪声干扰）
RMSProp: 为每个参数自适应调整步长（走过的路陡就走慢点，平就走快点）
Adam: 两者结合 → 又稳又快

默认参数: lr=1e-3, β₁=0.9, β₂=0.999, ε=1e-8 （一般不需要改）
```

来源: Kingma & Ba, "Adam: A Method for Stochastic Optimization." ICLR 2015; Stanford CS231n Lecture 6

### 6.4 完整的训练循环

```
训练一个神经网络的完整循环：

for epoch in range(300):           ← 遍历300次完整训练集
  for batch in train_loader:       ← 每次取一个mini-batch（如64个样本）
    
    # 1. 前向传播
    ŷ = model(batch.x)             ← 输入→网络→预测
    
    # 2. 计算损失
    loss = MSE(ŷ, batch.y)        ← 预测 vs 真实答案
    
    # 3. 反向传播
    loss.backward()                ← PyTorch自动计算所有梯度
    
    # 4. 梯度下降更新权重
    optimizer.step()               ← w = w - η × ∂L/∂w
    
    # 5. 清零梯度（PyTorch要求）
    optimizer.zero_grad()
```

**每一次循环，权重都朝着减小损失的方向微调一点点。经过成千上万次迭代，网络逐渐学会了从输入到输出的映射。这就是"学习"。**

## 第7章：过拟合与正则化

![fig7](/images/NeuralNetwork-Basics-DeepDive/fig7.png)


### 前置知识
训练循环（第6章），损失的概念（第4章）。

### 7.1 训练集 / 验证集 / 测试集

```
你有1000道数学题：

训练集 (Training Set) — 700道：学生用来学习和练习的题
验证集 (Validation Set) — 150道：老师用来检查学习进度的小测验
测试集 (Test Set) — 150道：期末考试，之前从没见过的题

为什么要分开？
  如果学生提前看到了考试题 → 考满分不代表真的学会了
  同理，如果模型在训练时"见过"测试数据 → 准确率高不代表真的泛化了
```

### 7.2 过拟合（Overfitting）= 死记硬背

**过拟合(Overfitting)** — 模型在训练集上表现很好，但在新数据上表现差。

```
类比：
  一个学生做了100道数学题，把每道题的答案都背下来了。
  考试时遇到一道见过的原题 → 满分
  遇到一道稍微变化的新题 → 完全不会
  
  这个学生"过拟合"了——他记住了答案，但没学会方法。
```

**怎么发现过拟合**：

```
训练损失 ↓ ↓ ↓ ↓ ↓ ↓ ↓     ← 训练集上损失一直降
验证损失 ↓ ↓ ↓ ↑ ↑ ↑ ↑     ← 验证集上损失先降后升！
                ↑
            这个拐点之后就是过拟合
```

来源: Stanford CS231n Lecture 7 "Training Neural Networks II"

### 7.3 正则化方法

#### Dropout

```
训练时：随机"关闭"一部分神经元（如50%的概率输出0）

○──●──○──●──○      ●=激活  ○=关闭
│  │     │  │
●──○──●──○──●      每次前向传播随机一组
│     │     │
○──●──○──●──○

直觉：不让网络依赖任何单个神经元
      就像考试时随机抽走一半同学，剩下的也得能答题
      → 每个神经元都必须学到独立有用的特征
```

来源: Srivastava et al. "Dropout: A Simple Way to Prevent Neural Networks from Overfitting." JMLR 2014.

#### L2正则化（Weight Decay）

```
在损失函数中加入权重大小的惩罚：

L_total = L_data + λ × Σwᵢ²
          ↑          ↑
     正常损失    权重大小的惩罚

λ越大 → 越不允许大权重 → 模型更"简单"
直觉：惩罚复杂的方案，鼓励用简单的方式拟合数据
```

#### 早停（Early Stopping）

```
监控验证集损失，当它开始上升时停止训练

损失
  │╲
  │  ╲___        ← 验证损失
  │     ╲__↗
  │        ↑ 在这里停止！
  │   ╲
  │    ╲____     ← 训练损失继续下降
  │         ╲__
  └──────────→ epoch
```

最简单有效的正则化方法之一。

#### 数据增强（Data Augmentation）

通过对训练数据做随机变换来"制造"更多数据：
- **图像**：旋转、翻转、裁剪、颜色抖动
- **运动数据**：左右镜像、时间缩放、加噪声
- 效果：让模型见到更多变化，减少死记硬背
