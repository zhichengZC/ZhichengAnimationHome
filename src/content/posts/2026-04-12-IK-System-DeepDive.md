## 1. IK 基础理论

### 1.1 什么是逆向运动学（Inverse Kinematics）

**正向运动学（Forward Kinematics, FK）** 是从骨骼层级的根部出发，逐级计算每个子骨骼在世界空间中的最终位置和旋转。每个骨骼的变换 = 父骨骼的全局变换 × 自身的本地变换。这是一个确定性的、从根到叶的计算过程。

**逆向运动学（Inverse Kinematics, IK）** 则是反过来的过程：给定末端执行器（End Effector）的目标位置/旋转，反推中间骨骼链的旋转角度，使末端执行器到达目标。这是一个从叶到根的求解问题。

数学上，IK 问题通常是：

```
给定：
  - 骨骼链 B₀, B₁, ..., Bₙ（从根到末端）
  - 每个骨骼的长度 Lᵢ
  - 末端目标位置 T（可能还有目标旋转 R）

求解：
  - 每个关节的旋转 θᵢ
  - 使得 FK(θ₀, θ₁, ..., θₙ) ≈ T
```

### 1.2 IK 问题的特性

| 特性 | 说明 |
|------|------|
| **欠定义（Underdetermined）** | 当链的自由度 > 末端约束时，存在无限多解 |
| **过定义（Overdetermined）** | 当链的自由度 < 末端约束时，可能无精确解 |
| **非线性** | 关节旋转到末端位置的映射是非线性的（三角函数） |
| **局部最优** | 迭代方法容易陷入局部最优 |
| **不可达** | 当目标超出链的最大伸展范围时，需要优雅降级 |

### 1.3 常见 IK 求解策略分类

```
IK 求解策略
├── 解析法（Analytical / Closed-Form）
│   ├── Two-Bone IK（最常用，如 Limb IK）
│   └── 特定拓扑的精确解
├── 迭代启发式法（Iterative Heuristic）
│   ├── CCD（Cyclic Coordinate Descent）
│   └── FABRIK（Forward And Backward Reaching IK）
├── 基于优化的方法（Optimization-Based）
│   ├── Jacobian 转置法
│   ├── Jacobian 伪逆法
│   ├── Damped Least Squares（DLS）
│   └── Position-Based IK（PBIK）→ UE5 Full Body IK
└── 物理模拟法
    └── 基于约束的求解（如 XPBD）
```

### 1.4 Two-Bone IK 的解析解

Two-Bone IK 是游戏中最常用的 IK，因为它有精确的解析解且计算极快。典型应用：手臂 IK、腿部 IK。

对于两根骨骼 `Upper`（长度 L₁）和 `Lower`（长度 L₂），给定根位置 `P₀` 和目标位置 `T`：

```cpp
// 步骤 1：计算距离
float DistToTarget = FVector::Dist(RootPos, TargetPos);

// 步骤 2：余弦定理求关节角度
// 对于三角形 P₀-Joint-T
float CosAngle = (L1*L1 + L2*L2 - DistToTarget*DistToTarget) / (2.0f * L1 * L2);
CosAngle = FMath::Clamp(CosAngle, -1.0f, 1.0f); // 防止浮点误差
float JointAngle = FMath::Acos(CosAngle);

// 步骤 3：使用 Pole Vector 确定弯曲平面
// Pole Vector 决定了关节弯曲的方向（比如膝盖朝前弯）
```

### 1.5 CCD 算法原理

CCD（Cyclic Coordinate Descent）是一种迭代启发式方法：

```
算法 CCD-IK:
  输入: 骨骼链 [B₀, B₁, ..., Bₙ], 目标 T, 精度 ε, 最大迭代 MaxIter
  
  for iter = 1 to MaxIter:
    for i = n-1 down to 0:  // 从末端前一个骨骼到根
      // 计算当前末端位置 E
      E = Bₙ.GlobalPosition
      
      // 计算从当前骨骼到末端的向量
      ToEnd = E - Bᵢ.GlobalPosition
      // 计算从当前骨骼到目标的向量
      ToTarget = T - Bᵢ.GlobalPosition
      
      // 计算需要的旋转
      Rotation = RotationFromTo(ToEnd, ToTarget)
      
      // 应用旋转限制（可选）
      Rotation = ClampRotation(Rotation, Bᵢ.RotationLimit)
      
      // 旋转当前骨骼
      Bᵢ.Rotate(Rotation)
      // 更新所有子骨骼的全局变换
      
    // 检查收敛
    if Distance(Bₙ.GlobalPosition, T) < ε:
      return SUCCESS
  
  return REACHED_MAX_ITERATIONS
```

**优点**：简单、每次迭代成本低、支持任意长链  
**缺点**：收敛慢、可能产生不自然的卷曲运动

### 1.6 FABRIK 算法原理

FABRIK（Forward And Backward Reaching Inverse Kinematics）是另一种迭代启发式方法，但基于位置而非旋转：

```
算法 FABRIK:
  输入: 骨骼链位置 [P₀, P₁, ..., Pₙ], 骨骼长度 [L₁, ..., Lₙ], 目标 T
  
  保存根位置: RootPos = P₀
  
  for iter = 1 to MaxIter:
    // === 前向阶段（Forward Reaching）===
    // 从末端到根，拉向目标
    Pₙ = T  // 将末端设置到目标位置
    for i = n-1 down to 0:
      // 从 Pᵢ₊₁ 向 Pᵢ 方向，以距离 Lᵢ₊₁ 放置 Pᵢ
      Dir = Normalize(Pᵢ - Pᵢ₊₁)
      Pᵢ = Pᵢ₊₁ + Dir * Lᵢ₊₁
    
    // === 后向阶段（Backward Reaching）===
    // 从根到末端，恢复根位置
    P₀ = RootPos  // 恢复根位置
    for i = 1 to n:
      // 从 Pᵢ₋₁ 向 Pᵢ 方向，以距离 Lᵢ 放置 Pᵢ
      Dir = Normalize(Pᵢ - Pᵢ₋₁)
      Pᵢ = Pᵢ₋₁ + Dir * Lᵢ
    
    // 检查收敛
    if Distance(Pₙ, T) < ε:
      return SUCCESS
  
  return REACHED_MAX_ITERATIONS
```

**优点**：收敛快、结果自然、支持多末端、直观  
**缺点**：旋转约束的支持不如 CCD 自然

### 1.7 PBIK（Position-Based IK）原理

UE5 的 Full Body IK 基于 PBIK 方法：

```
核心思想:
  1. 将骨骼系统建模为"质点-距离约束"系统
  2. 每个骨骼 = 一个质点（有质量/刚度）
  3. 骨骼间的连接 = 距离约束
  4. 末端执行器 = 位置约束
  5. 使用约束投影法迭代求解

每次迭代:
  1. Pre-Pull: 整体平移/旋转骨骼链趋向目标
  2. 约束投影: 满足骨骼长度约束
  3. 旋转限制: 应用每个关节的旋转限制
  4. Over-Relaxation: 超松弛加速收敛
```

### 2.3 选择指南：用哪个 IK？

```
场景决策树:

需要 IK 吗？
├─ 简单的手/脚位置约束（2-3 骨骼链）
│   └─ → IKRig LimbSolver 或 ControlRig Two-Bone IK
│
├─ 脚步贴地 + 骨盆调整
│   └─ → AnimNode_FootPlacement (AnimationWarping Plugin)
│
├─ 长链 IK（蛇、触手、尾巴、机械臂）
│   ├─ 单链 → CCDIK 或 FABRIK (ControlRig RigUnit)
│   └─ 多链 → MultiFABRIK (ControlRig RigUnit)
│
├─ 全身 IK（多个末端同时求解）
│   └─ → Full Body IK (IKRig PBIK Solver)
│
├─ 头部/眼球注视目标
│   └─ → RigUnit_AimBone (ControlRig)
│
├─ 动画重定向（不同骨骼拓扑间传递动画）
│   └─ → IK Retargeter (IKRig Plugin)
│
└─ 自定义求解需求
    └─ → 继承 FIKRigSolverBase 实现自定义 Solver
```

```
不适合:
  ❌ 人形腿/臂（用 LimbSolver 更好）
  ❌ 需要快速收敛的场景
  ❌ 全身多链同时求解
```

## 5. Limb IK — 肢体求解器

### 5.1 IKRig Limb Solver

这是 UE5 中最常用的 IK 求解器，专为 2-3 骨骼的肢体（手臂、腿）设计。

**源码路径**：`Engine/Plugins/Animation/IKRig/Source/IKRig/Public/Rig/Solvers/IKRigLimbSolver.h`

```cpp
USTRUCT(BlueprintType)
struct FIKRigLimbSolver : public FIKRigSolverBase
{
    UPROPERTY()
    FIKRigLimbSolverSettings Settings;
    
    // 运行时接口
    virtual void Initialize(const FIKRigSkeleton& InIKRigSkeleton) override;
    virtual void Solve(FIKRigSkeleton& InIKRigSkeleton, const FIKRigGoalContainer& InGoals) override;
    
private:
```

### 5.2 FIKRigLimbSolverSettings 详解

`FIKRigLimbSolverSettings` 继承自 `FLimbSolverSettings`：

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `StartBone` | `FName` | — | IK 链起始骨骼（如 Hip） |
| `GoalName` | `FName` | — | 关联的 IK Goal 名称 |
| `EndBone` | `FName` | — | IK 链末端骨骼（如 Foot） |
| `ReachPrecision` | `float` | 0.01 | 到达精度 |
| `HingeRotationAxis` | `EAxis::Type` | None | 铰链轴（None=自动推断） |
| `MaxIterations` | `int32` | 12 | 最大迭代 |
| `bEnableLimit` | `bool` | false | 启用旋转限制 |
| `MinRotationAngle` | `float` | 15.0 | 最小旋转角度 |
| `bAveragePull` | `bool` | true | 平均拉力分配 |
| `PullDistribution` | `float` | 0.5 | 拉力分配比（0=根端，1=末端） |
| `ReachStepAlpha` | `float` | 0.7 | 每步到达比例 |
| `bEnableTwistCorrection` | `bool` | false | 扭转修正 |
| `EndBoneForwardAxis` | `EAxis::Type` | Y | 末端骨骼前向轴 |

### 5.3 求解流程

```
LimbSolver 求解流程:
1. 获取根、中间、末端三个骨骼的全局变换
2. 计算链总长度和目标距离
3. 如果目标不可达:
   → 沿方向拉伸到最大长度
4. 使用解析 Two-Bone IK:
   → 余弦定理求中间关节角度
   → Pole Vector 确定弯曲平面
5. 可选: 扭转修正
6. 可选: 旋转限制
7. 更新所有子骨骼的全局变换
```

## 6. Full Body IK (PBIK)

### 6.1 概述

UE5 的 Full Body IK 基于 **PBIK（Position-Based IK）**，这是一种基于位置约束的迭代求解方法。它可以同时处理多个末端执行器，实现全身 IK。

**源码路径**：`Engine/Plugins/Animation/IKRig/Source/IKRig/Public/Rig/Solvers/IKRigFullBodyIK.h`

> **注意**：`Plugins/Experimental/FullBodyIK/` 下还有一个基于 Jacobian 的实验性全身 IK（`FRigUnit_FullbodyIK`），已在 **5.0 废弃**。不要使用。

### 6.2 架构

```

├── Settings: FIKRigFBIKSettings           // 全局求解器设置
```

### 6.3 FIKRigFBIKSettings — 全局设置

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `RootBone` | `FName` | — | 求解器根骨骼（通常为 Pelvis/Hips） |

| `SubIterations` | `int32` | 0 | 子链预求解迭代（在主迭代前） |
| `MassMultiplier` | `float` | 1.0 | 全局质量乘数（高=刚硬，需更多迭代） |

### 6.6 PreferredAngles 的重要性

```
问题: 膝盖/肘部反向弯曲
原因: PBIK 不知道生物力学正确的弯曲方向

解决方案: 使用 PreferredAngles
  - 先将 Iterations 设为 0
  - 移动 Effector 向根部靠近（压缩链）
  - 观察骨骼弯曲方向
  - 设置 PreferredAngles 为期望方向
  - 恢复 Iterations

推荐值（示例）:
  膝盖: PreferredAngles = (90, 0, 0)  // 向前弯曲
  肘部: PreferredAngles = (0, -90, 0) // 向下弯曲
```

### 7.8 求解器栈执行顺序

```
IKRig 求解流程:
1. 设置输入姿态（全局空间）
2. 设置所有 IK Goal 的目标位置/旋转
3. 解析 Goal 的最终变换（考虑空间和 Alpha）
4. 按顺序执行求解器栈:
   ┌─────────────────────────────────────┐
   │ Solver 1: Body Mover               │  ← 先移动躯干
   │ Solver 2: Full Body IK             │  ← 全身 IK
   │ Solver 3: Limb IK (Left Leg)       │  ← 左腿精确 IK
   │ Solver 4: Limb IK (Right Leg)      │  ← 右腿精确 IK
   │ Solver 5: Pole Solver (Knee)       │  ← 膝盖方向修正
   └─────────────────────────────────────┘
5. 输出最终全局姿态
```

### 9.8 Foot Placement 求解流程

```
FootPlacement 每帧执行流程:

1. GatherPelvisDataFromInputs()    → 读取骨盆输入数据
2. GatherLegDataFromInputs()       → 读取每条腿的输入数据
   └── 计算脚速度、对齐 Alpha、到地面距离
3. CalculateFootMidpoint()         → 计算脚中点
4. ProcessCharacterState()         → 处理角色状态
   └── 平滑地面法线、更新角色速度
5. ProcessFootAlignment()          → 处理每条腿
   ├── WantsToPlant()              → 判断是否需要植入
   ├── DeterminePlantType()        → 判断植入/解除/重植
   ├── AlignPlantToGround()        → 对齐到地面
   ├── UpdatePlantOffsetInterpolation() → 插值偏移
   └── UpdatePlantingPlaneInterpolation() → 插值植入平面
6. SolvePelvis()                   → 求解骨盆偏移
   ├── 计算每条腿需要的骨盆高度
   ├── 选择最佳骨盆高度（基于 PelvisHeightMode）
   ├── 应用 Heel Lift Ratio
   └── 水平重心调整
7. UpdatePelvisInterpolationRootSpace() → 骨盆插值
8. FinalizeFootAlignment()         → 每条腿最终调整
   ├── 检查过伸/过压
   ├── 脚跟抬起
   └── 分离平面约束
9. 输出骨骼变换
```

## 11. IK 在怪物动画 Pipeline 中的应用

### 11.1 怪物 IK 需求分析

```
怪物动画 Pipeline 中的 IK 需求:
┌──────────────────────────────────────────────────────────────┐
│ 需求层                  │ IK 方案                            │
│─────────────────────────┼──────────────────────────────────────│
│ 脚步贴地                │ FootPlacement + IKRig LimbSolver   │
│ 头部追踪玩家            │ ControlRig AimBone                 │
│ 触手/尾巴               │ CCDIK / FABRIK (ControlRig)         │
│ 多足怪物                │ Full Body IK (多末端)               │
│ 爬墙/天花板             │ FootPlacement 自定义法线            │
│ 体型缩放               │ IK Retargeter (Scale Op)            │
│ 攻击目标适配            │ Motion Warping + LimbSolver         │
│ 死亡布娃娃 → IK 姿态   │ PBIK 恢复                           │
└──────────────────────────────────────────────────────────────┘
```

### 11.2 怪物脚步贴地方案

```
方案: State Tree + Anim SM + FootPlacement

State Tree:
  ├── Idle 状态 → FootPlacement (强锁定)
  ├── Walk 状态 → FootPlacement (弱锁定)
  ├── Run 状态  → FootPlacement (最小锁定)
  ├── Attack    → FootPlacement 关闭
  └── Stagger   → FootPlacement 关闭

AnimBP 结构:
  Source Pose → State Machine → FootPlacement → Output

FootPlacement 配置:
  - PlantSpeedMode = Manual (使用自定义曲线)
  - LegDefinitions: 根据怪物腿数量配置
  - 四足怪: 4 组 LegDefinition
  - 蜘蛛类: 8 组 LegDefinition (性能注意!)
```

### 11.3 长链怪物部件（触手/尾巴）

```
方案: ControlRig + CCDIK/FABRIK

ControlRig Blueprint:
  1. 输入: 目标位置（来自 GAS 或游戏逻辑）
  2. CCDIK/FABRIK 求解触手链
  3. 添加噪声/程序化动画
  4. 输出: 骨骼变换

优化:
  - LOD 0: 完整 IK 求解 (MaxIterations=12)
  - LOD 1: 降低迭代 (MaxIterations=6)
  - LOD 2: 关闭 IK, 纯 FK 播放
```

### 11.4 多足怪物全身 IK

```
方案: IKRig Full Body IK

IKRig 配置:
  RootBone: pelvis / body_center
  Goals:
    - foot_FL_goal (前左脚)
    - foot_FR_goal (前右脚)
    - foot_BL_goal (后左脚)
    - foot_BR_goal (后右脚)
  
  Solver Stack:
    1. Full Body IK Solver
       - Iterations: 15-20
       - RootBehavior: PrePull
       - OverRelaxation: 1.3
    
  Goal Settings:
    每个脚 Goal:
      - StrengthAlpha: 1.0
      - PullChainAlpha: 0.8
      - PinRotation: 0.5
    
  Bone Settings:
    Spine 骨骼: RotationStiffness = 0.3
    Hip 骨骼: PreferredAngles 设置正确弯曲方向

游戏逻辑:
  每帧:
    1. 射线检测每只脚的地面位置
    2. 步态生成器决定哪只脚移动
    3. 更新 IK Goal 位置
    4. IKRig 自动求解
```

### 11.5 与 GAS 的集成

```
IK + GAS 集成模式:

GameplayAbility:
  ├── ActivateAbility()
  │   ├── 获取攻击目标位置
  │   ├── 计算 IK Goal 位置
  │   └── 通过 Gameplay Tag 触发 IK 调整
  │
  ├── AnimNotify_IKTarget
  │   ├── 设置 IK Goal 变换
  │   └── 过渡 Alpha（平滑开启/关闭）
  │
  └── EndAbility()
      └── 恢复 IK 到默认状态

数据流:
  GAS Ability → GameplayEvent → AnimBP → IKRig Goal → Solve
```

## 12. 性能优化与调试

### 12.1 性能开销分析

| IK 类型 | 单次求解开销 | 主要瓶颈 |
|---------|------------|----------|
| Two-Bone / Limb IK | 极低（解析解） | 几乎可忽略 |
| CCDIK (10 骨骼, 8 迭代) | 低 | 迭代数 × 骨骼数 |
| FABRIK (10 骨骼, 8 迭代) | 低 | 迭代数 × 骨骼数 |
| Full Body IK (50 骨骼, 20 迭代) | 中-高 | 迭代数 × 骨骼数 × 约束数 |
| Foot Placement (2 腿) | 中 | 射线检测 + 弹簧插值 |
| Foot Placement (8 腿) | 高 | 多条射线检测 |

### 12.2 LOD 策略

```
IK LOD 分级策略:

LOD 0 (近距离):
  - 所有 IK 开启
  - Full Body IK: Iterations = 20
  - FootPlacement: 完整功能
  - ControlRig IK: MaxIterations = 12

LOD 1 (中距离):
  - Full Body IK: Iterations = 10
  - FootPlacement: 简化（关闭地面对齐插值）
  - ControlRig IK: MaxIterations = 6

LOD 2 (远距离):
  - 关闭所有 IK
  - 纯 FK 播放
  - 可选: 简单的骨盆高度调整

LOD 3+ (极远):
  - 最简化的动画
  - 可能完全关闭骨骼动画
```

### 12.4 常见问题排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|---------|
| 膝盖/肘反弯 | 缺少 PreferredAngles | 设置 FBIK BoneSettings |
| 脚穿地面 | Trace 配置不当 | 调大 EndOffset 和 SweepRadius |
| IK 抖动 | 插值参数不当 | 增加 Stiffness/Damping |
| 骨骼过伸 | MaxExtensionRatio 太高 | 降低到 0.5 以下 |
| 全身 IK 不收敛 | Iterations 不够 | 增加迭代或启用 SubIterations |
| IK 在某骨骼无效 | 骨骼不在 RequiredBones 中 | 检查 Solver 的骨骼配置 |
