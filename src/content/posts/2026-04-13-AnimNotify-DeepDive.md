## 概述与核心概念

### 什么是 AnimNotify？

**AnimNotify**（动画通知）是 UE5 动画系统中最重要的**事件桥梁**机制之一。它允许动画资产（AnimSequence、AnimMontage 等）在播放到特定帧时触发游戏逻辑。

核心设计哲学：

| 维度 | 描述 |
|------|------|
| **定位** | 动画数据到游戏逻辑的事件管线 |
| **触发源** | 动画资产中标记的时间点或时间段 |
| **触发目标** | 拥有该 AnimInstance 的 Actor / Component |
| **设计模式** | 观察者模式（Observer Pattern）+ 时间轴标记 |
| **线程安全** | 主要在游戏线程执行，部分可在 Worker 线程预处理 |

### 两大核心类

UE5 的通知系统围绕两个基类构建：

```
UAnimNotify            → 瞬时通知（单帧触发）
UAnimNotifyState       → 持续通知（时间段触发，有 Begin / Tick / End）
```

它们都继承自 `UObject`，但有着截然不同的生命周期和使用场景。

### 通知 vs 事件 vs 曲线

在 UE5 动画系统中，有多种"从动画驱动游戏逻辑"的机制：

| 机制 | 精度 | 数据载荷 | 时序 | 典型用途 |
|------|------|----------|------|----------|
| **AnimNotify** | 单帧 | 自定义（通过子类） | 触发式 | 播放音效、生成粒子、触发攻击判定 |
| **AnimNotifyState** | 时间段 | 自定义 + Tick | 持续式 | Trail 拖尾、武器碰撞窗口、无敌帧 |
| **AnimCurve** | 逐帧插值 | float | 连续 | 混合权重、IK 权重、材质参数 |
| **SyncMarker** | 标记点 | 名称 | 同步用 | 步伐同步、动画对齐 |
| **MetaData** | 全局/片段 | 自定义 | 非实时 | 编辑器信息、分类标签 |

### 触发时序详解

瞬时通知的触发遵循**区间检测**逻辑：

```
时间轴:  0.0 ----[PrevPos]--------[NotifyPos]--------[CurrPos]---- 1.0

判定条件: PrevPos <= NotifyPos < CurrPos  →  触发！
```

关键规则：

| 规则 | 说明 |
|------|------|
| **左闭右开** | 通知位置 ∈ [PrevPos, CurrPos) 时触发 |
| **循环处理** | 循环动画会拆分为两段区间检查首尾边界 |
| **快进仍触发** | 帧率过低跳过通知位置，默认仍会补触发 |
| **逆播不触发** | 反向播放时默认不触发通知（可配置） |
| **精度限制** | 30FPS 下精度 ±17ms，60FPS 下 ±8ms |

**结论**：对于精确的攻击判定窗口，不应使用单帧 AnimNotify，而应使用 AnimNotifyState。

## AnimNotifyState 详解

### 基类 API

```cpp
// UAnimNotifyState
// Engine/Source/Runtime/Engine/Classes/Animation/AnimNotifies/AnimNotifyState.h
UCLASS(abstract, Blueprintable, hidecategories=Object)
class ENGINE_API UAnimNotifyState : public UObject
{
    GENERATED_BODY()
public:
    virtual void NotifyBegin(
        USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation,
        float TotalDuration, const FAnimNotifyEventReference& EventReference);

    virtual void NotifyTick(
        USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation,
        float FrameDeltaTime, const FAnimNotifyEventReference& EventReference);

    virtual void NotifyEnd(
        USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation,
        const FAnimNotifyEventReference& EventReference);

    // 蓝图版本
    UFUNCTION(BlueprintImplementableEvent)
    bool Received_NotifyBegin(...) const;
    UFUNCTION(BlueprintImplementableEvent)
    bool Received_NotifyTick(...) const;
    UFUNCTION(BlueprintImplementableEvent)
    bool Received_NotifyEnd(...) const;
};
```

### 三阶段生命周期

```
时间轴:  ----[Begin]======[Tick...Tick...Tick]=======[End]----

帧 1:  [PrevPos]---[Begin]---[CurrPos]  → NotifyBegin()
帧 2:  [PrevPos]-----[CurrPos]          → NotifyTick()
帧 N:  [PrevPos]---[End]---[CurrPos]    → NotifyTick() + NotifyEnd()
```

### 关键行为

| 行为 | 描述 |
|------|------|
| **Begin 保证** | 从中间开始播放且起始位置在范围内，Begin 仍触发 |
| **End 保证** | 动画被中断（Montage Stop），End 仍会被调用 |
| **Tick 频率** | 与动画更新频率一致（受 LOD/URO 影响） |
| **嵌套安全** | 多个 NotifyState 可重叠，各自独立 |
| **跨 Section** | Montage 跨 Section 跳转时，活跃的 State 收到 End |

### Active Notify State 管理

引擎内部维护 `ActiveAnimNotifyState` 列表：

```cpp
// 伪代码 - 每帧更新逻辑
void UAnimInstance::TriggerAnimNotifies(float DeltaSeconds)
{
    TArray<FAnimNotifyEvent*> NewActiveStates = CollectActiveStates();

    // 离开的 → End
    for (auto* State : ActiveAnimNotifyState)
        if (!NewActiveStates.Contains(State))
            State->NotifyStateClass->NotifyEnd(...);

    // 新进入的 → Begin；持续中的 → Tick
    for (auto* State : NewActiveStates)
    {
        if (!ActiveAnimNotifyState.Contains(State))
            State->NotifyStateClass->NotifyBegin(...);
        else
            State->NotifyStateClass->NotifyTick(...);
    }

    ActiveAnimNotifyState = NewActiveStates;
}
```

## 通知触发机制与时序

### 完整调用链

```
AWorldSettings::Tick()
└── USkeletalMeshComponent::TickAnimation()
    └── UAnimInstance::UpdateAnimation()
        ├── UpdateAnimationInternal() → 计算动画时间
        └── TriggerAnimNotifies(DeltaSeconds)
            ├── GetAnimNotifiesFromDeltaPositions() → 收集区间通知
            ├── 合并 + 排序（按时间升序）
            ├── 过滤（概率 / LOD / Server）
            ├── 执行瞬时通知 UAnimNotify::Notify()
            ├── 管理 NotifyState（Begin / Tick / End）
            └── 广播委托
```

### 通知排序规则

1. 按触发时间升序排列
2. 时间相同时，按 Notifies 数组索引排序
3. NotifyState 的 Begin/End 优先于 Tick
4. 不同动画资产的通知合并后统一排序

### Branching Point vs Queued

Montage 中有两种通知触发模式：

| 特性 | Queued（默认） | Branching Point |
|------|---------------|-----------------|
| **触发时机** | 帧末尾统一触发 | 动画求值时立即触发 |
| **延迟** | 可能 1 帧 | 零延迟 |
| **精确度** | 帧级 | 亚帧级 |
| **性能开销** | 低 | 较高 |
| **典型用途** | 音效、粒子 | Montage Section 跳转决策 |

### 循环动画中的通知

```
动画长度: 1.0s, 循环播放
PrevPos = 0.95, CurrPos = 0.05 (已循环)
→ 拆分为 [0.95, 1.0) 和 [0.0, 0.05) 两段检查
→ 两段区间内的通知都会触发
```

## C++ 自定义 AnimNotify

### 实战示例：攻击判定通知

```cpp
// MyAnimNotify_DamageWindow.h
UCLASS(meta=(DisplayName="Damage Window"))
class MYGAME_API UMyAnimNotify_DamageWindow : public UAnimNotify
{
    GENERATED_BODY()
public:
    UMyAnimNotify_DamageWindow();

    virtual void Notify(USkeletalMeshComponent* MeshComp,
        UAnimSequenceBase* Animation,
        const FAnimNotifyEventReference& EventReference) override;

    virtual FString GetNotifyName() const override;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Damage")
    float DamageAmount = 50.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Damage")
    TSubclassOf<UDamageType> DamageType;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Damage",
              meta=(ClampMin="0.0"))
    float DetectionRadius = 100.0f;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Damage")
    FName DetectionSocket = "weapon_tip";

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Damage")
    bool bIgnoreSelf = true;
};

// MyAnimNotify_DamageWindow.cpp
UMyAnimNotify_DamageWindow::UMyAnimNotify_DamageWindow()
{
    NotifyColor = FColor::Red;
#if WITH_EDITORONLY_DATA
    bShouldFireInEditor = false;
#endif
}

void UMyAnimNotify_DamageWindow::Notify(
    USkeletalMeshComponent* MeshComp, UAnimSequenceBase* Animation,
    const FAnimNotifyEventReference& EventReference)
{
    Super::Notify(MeshComp, Animation, EventReference);
    if (!MeshComp || !MeshComp->GetOwner()) return;

    AActor* Owner = MeshComp->GetOwner();
    UWorld* World = Owner->GetWorld();
    if (!World) return;

    FVector DetectionLocation = MeshComp->GetSocketLocation(DetectionSocket);
    TArray<FOverlapResult> Overlaps;
    FCollisionShape Shape = FCollisionShape::MakeSphere(DetectionRadius);

    if (World->OverlapMultiByChannel(Overlaps, DetectionLocation,
        FQuat::Identity, ECC_Pawn, Shape))
    {
        for (const FOverlapResult& Overlap : Overlaps)
        {
            AActor* HitActor = Overlap.GetActor();
            if (!HitActor || (bIgnoreSelf && HitActor == Owner)) continue;
            FDamageEvent DamageEvent;
            HitActor->TakeDamage(DamageAmount, DamageEvent,
                Owner->GetInstigatorController(), Owner);
        }
    }
}
```

## C++ 自定义 AnimNotifyState

### 实战示例：武器碰撞窗口

> **重要设计注意**：`UAnimNotifyState` 的实例是在动画资产级别共享的（类似 CDO），**不是**每个播放该动画的 Actor 各自拥有一份。因此**绝不能将运行时可变状态（如 HitActors、上一帧位置等）存储为 NotifyState 的类成员变量**，否则多个角色同时播放同一动画时会互相污染数据。正确做法是将每实例数据存储在 Actor 或其组件上。以下示例使用一个轻量级的 `UActorComponent` 来持有每实例状态。

```cpp
// WeaponCollisionDataComponent.h — 用于存储每实例的武器碰撞运行时数据
UCLASS()
class MYGAME_API UWeaponCollisionDataComponent : public UActorComponent
{
    GENERATED_BODY()
public:
    TArray<FVector> PreviousTracePoints;
    TMap<TWeakObjectPtr<AActor>, float> HitActors;
    float ElapsedTime = 0.0f;

    void Reset()
    {
        PreviousTracePoints.Reset();
        HitActors.Reset();
        ElapsedTime = 0.0f;
    }
};

// AnimNotifyState_WeaponCollision.h
UCLASS(meta=(DisplayName="Weapon Collision Window"))
class MYGAME_API UAnimNotifyState_WeaponCollision : public UAnimNotifyState
{
    GENERATED_BODY()
public:
    virtual void NotifyBegin(USkeletalMeshComponent* MeshComp,
        UAnimSequenceBase* Animation, float TotalDuration,
        const FAnimNotifyEventReference& EventReference) override;
    virtual void NotifyTick(USkeletalMeshComponent* MeshComp,
        UAnimSequenceBase* Animation, float FrameDeltaTime,
        const FAnimNotifyEventReference& EventReference) override;
    virtual void NotifyEnd(USkeletalMeshComponent* MeshComp,
        UAnimSequenceBase* Animation,
        const FAnimNotifyEventReference& EventReference) override;

    UPROPERTY(EditAnywhere, Category="Collision")
    FName WeaponBaseSocket = "weapon_base";

    UPROPERTY(EditAnywhere, Category="Collision")
    FName WeaponTipSocket = "weapon_tip";

    UPROPERTY(EditAnywhere, Category="Collision", meta=(ClampMin="2", ClampMax="20"))
    int32 TracePointCount = 5;

    UPROPERTY(EditAnywhere, Category="Collision", meta=(ClampMin="0.1"))
    float TraceRadius = 10.0f;

    UPROPERTY(EditAnywhere, Category="Damage")
    float BaseDamage = 30.0f;

    UPROPERTY(EditAnywhere, Category="Damage")
    bool bAllowMultiHit = false;

    UPROPERTY(EditAnywhere, Category="Damage",
              meta=(EditCondition="bAllowMultiHit", ClampMin="0.05"))
    float MultiHitInterval = 0.2f;

private:
    UWeaponCollisionDataComponent* GetOrCreateData(AActor* Owner) const;
};

// 实现
UWeaponCollisionDataComponent* UAnimNotifyState_WeaponCollision::GetOrCreateData(AActor* Owner) const
{
    if (!Owner) return nullptr;
    auto* Data = Owner->FindComponentByClass<UWeaponCollisionDataComponent>();
    if (!Data)
    {
        Data = NewObject<UWeaponCollisionDataComponent>(Owner);
        Data->RegisterComponent();
    }
    return Data;
}

void UAnimNotifyState_WeaponCollision::NotifyBegin(...)
{
    Super::NotifyBegin(MeshComp, Animation, TotalDuration, EventReference);
    if (!MeshComp || !MeshComp->GetOwner()) return;
    auto* Data = GetOrCreateData(MeshComp->GetOwner());
    if (!Data) return;
    Data->Reset();

    FVector BasePos = MeshComp->GetSocketLocation(WeaponBaseSocket);
    FVector TipPos = MeshComp->GetSocketLocation(WeaponTipSocket);
    Data->PreviousTracePoints.SetNum(TracePointCount);
    for (int32 i = 0; i < TracePointCount; ++i)
    {
        float Alpha = (float)i / (float)(TracePointCount - 1);
        Data->PreviousTracePoints[i] = FMath::Lerp(BasePos, TipPos, Alpha);
    }
}

void UAnimNotifyState_WeaponCollision::NotifyTick(...)
{
    Super::NotifyTick(MeshComp, Animation, FrameDeltaTime, EventReference);
    if (!MeshComp || !MeshComp->GetOwner()) return;
    AActor* Owner = MeshComp->GetOwner();
    UWorld* World = Owner->GetWorld();
    auto* Data = GetOrCreateData(Owner);
    if (!Data) return;
    Data->ElapsedTime += FrameDeltaTime;

    FVector BasePos = MeshComp->GetSocketLocation(WeaponBaseSocket);
    FVector TipPos = MeshComp->GetSocketLocation(WeaponTipSocket);
    TArray<FVector> CurrentPoints; CurrentPoints.SetNum(TracePointCount);
    for (int32 i = 0; i < TracePointCount; ++i)
        CurrentPoints[i] = FMath::Lerp(BasePos, TipPos,
            (float)i / (float)(TracePointCount - 1));

    FCollisionQueryParams QueryParams;
    QueryParams.AddIgnoredActor(Owner);

    for (int32 i = 0; i < TracePointCount; ++i)
    {
        if (!Data->PreviousTracePoints.IsValidIndex(i)) continue;
        TArray<FHitResult> Hits;
        if (World->SweepMultiByChannel(Hits, Data->PreviousTracePoints[i],
            CurrentPoints[i], FQuat::Identity, ECC_Pawn,
            FCollisionShape::MakeSphere(TraceRadius), QueryParams))
        {
            for (const FHitResult& Hit : Hits)
            {
                AActor* HitActor = Hit.GetActor();
                if (!HitActor || HitActor == Owner) continue;
                if (float* LastTime = Data->HitActors.Find(HitActor))
                {
                    if (!bAllowMultiHit) continue;
                    if (Data->ElapsedTime - *LastTime < MultiHitInterval) continue;
                }
                Data->HitActors.Add(HitActor, Data->ElapsedTime);
                FDamageEvent DamageEvent;
                HitActor->TakeDamage(BaseDamage, DamageEvent,
                    Owner->GetInstigatorController(), Owner);
            }
        }
    }
    Data->PreviousTracePoints = CurrentPoints;
}

void UAnimNotifyState_WeaponCollision::NotifyEnd(...)
{
    Super::NotifyEnd(MeshComp, Animation, EventReference);
    if (!MeshComp || !MeshComp->GetOwner()) return;
    if (auto* Data = GetOrCreateData(MeshComp->GetOwner()))
        Data->Reset();
}
```

## Notify 与 Montage 的交互

### Section 跳转对通知的影响

```
Montage:  [Section A] → [Section B] → [Section C]

A → B（正常推进）: A 中 NotifyState 收到 End，B 通知从头触发
A → C（跳过 B）:   B 的通知完全不触发
B → A（倒回）:     B 中 NotifyState 收到 End，A 重新从头触发
Montage 被中断:    所有活跃 NotifyState 收到 End
```

## 多线程与异步考量

### 关键规则

```
1. 通知触发始终在游戏线程执行
2. 并行动画求值（骨骼变换）在 Worker 线程
3. 不要在通知中读取正在并行计算的骨骼数据
4. GetBoneTransform() 返回的是上一帧的最终结果（安全）
```

### URO 对通知的影响

```
距离 < 10m:  每帧更新   → 通知正常
距离 10~30m: 每 2 帧    → 延迟 1 帧
距离 30~60m: 每 4 帧    → 延迟 3 帧
距离 > 60m:  每 8 帧    → 延迟 7 帧

关键：URO 不会跳过通知！只是延迟，下次更新时补触发。
```

## 性能优化与最佳实践

### 性能开销估算

```
每帧通知处理 CPU 开销（近似）:
- 区间检测:         ~0.1μs / 通知
- 蓝图通知执行:     ~5-20μs / 次
- C++ 通知执行:     ~0.5-5μs / 次
- NotifyState Tick: ~0.5-5μs / 帧

10 个通知/帧:    ~10-50μs（可忽略）
100 个通知/帧:   ~100-500μs（开始有感）
1000+ 通知/帧:   ~1ms+（需要优化）
```

| 概率触发 | 非关键通知使用随机概率 | ⭐⭐ |
| 延迟执行 | 昂贵操作延迟到下一帧 | ⭐⭐ |
| Profiling | 用 Unreal Insights 监测 | ⭐⭐⭐ |

## 常见陷阱与调试技巧

### 陷阱 1：通知不触发

```
排查清单:
□ 动画资产中确实添加了通知？
□ 动画正在播放？（AnimBP Debug 模式检查）
□ NotifyTriggerChance > 0？
□ NotifyFilterLOD 设置是否合理？
□ bTriggerOnDedicatedServer 在服务器端是否为 true？
□ 动画是否被 LOD / URO 跳过更新？
□ Montage 是否在正确的 Slot 播放？
□ AnimBP 是否正确分配给 SkeletalMeshComponent？
```

### 陷阱 2：通知重复触发

```
可能原因:
1. 多个 SkeletalMeshComponent 播放同一动画
2. AnimComposite 中源动画重叠
3. Montage 循环 Section 导致重复进入
4. 动画混合过程中两个动画都触发通知

解决: 去重机制（HitActor 列表）、NotifyTrack 隔离
```

### 陷阱 3：NotifyState End 未被调用

```
可能原因: Actor 被 GC、MeshComponent 被禁用、ForceStop
解决: Actor::EndPlay 中检查未关闭的状态、OnMontageEnded 委托备份
```

### 陷阱 4：混合时双重触发

```
Anim_A (weight: 0.7) + Anim_B (weight: 0.3)
→ 两者的通知都会触发！混合权重不影响通知触发
→ BlendSpace、Layered Blend 中都适用
→ 可能导致脚步声双重触发
```
