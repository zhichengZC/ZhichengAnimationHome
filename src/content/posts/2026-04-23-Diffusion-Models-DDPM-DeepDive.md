## 第 0 章 前置知识回顾 [L0 核心]

### 0.1 高斯分布与重参数化（从 VAE 带过来）

**高斯分布（Gaussian / Normal distribution）** 写作 `N(μ, σ²)`，两个参数：均值 μ、方差 σ²（标准差 σ）。采样一个样本的写法：

```
x ~ N(μ, σ²)    # "x 服从均值 μ、方差 σ² 的高斯分布"
```

**重参数化技巧（Reparameterization trick）** 是 VAE 里学过的老朋友——把 "从 `N(μ, σ²)` 采样" 改写成 "先从 `N(0, 1)` 采样 ε，再线性映射"：

```
x = μ + σ · ε,   ε ~ N(0, 1)
```

**为什么 VAE 里要这么写**：因为 μ、σ 是网络输出，直接采样会切断梯度；这样改写后，梯度能从 x 穿过 μ、σ 回到网络。**扩散模型里也要这么做——不是为了梯度，而是为了"从高斯里抽样时，把随机性和均值/方差分开记账"**。后面你会看到 `x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε` 就是这个套路。

### 0.2 KL 散度的直觉（不严格）

**KL 散度（Kullback-Leibler divergence）** 衡量两个分布 p、q 的"距离"（不对称，严格讲不是距离）。直觉：**如果你手头的真实分布是 p，但你被迫用 q 去编码/建模，额外付出的"信息损失"就是 KL(p || q)**。

- `KL(p || q) = 0` ⇔ p = q 处处相等
- `KL(p || q) > 0` 永远非负
- 两个高斯分布之间的 KL 有闭式解（VAE 里用过，扩散模型推导里也会用）

**扩散模型里你见到 KL 时的唯一反应**：那是"让网络预测的反向分布 p_θ 去逼近真实反向分布 q"的度量。不需要推导，认识即可。

### 0.3 ELBO 与 L_simple（先给结论，推导放附录）

VAE 里学过 **ELBO（Evidence Lower Bound，证据下界）**：一个让网络最大化的目标函数，近似"让模型把真实数据的概率估得越高越好"。DDPM 的训练目标也是从 ELBO 推出来的，但 **Ho 等人 2020 的大贡献之一，就是证明了扩散模型的 ELBO 可以化简成一个极其简单的 MSE**：

```
L_simple = E_{t, x_0, ε} [ ||ε - ε_θ(x_t, t)||² ]
```

**翻译**：随机抽一个时间步 t、一个干净样本 x_0、一个标准正态噪声 ε；把 x_0 加噪到 x_t；让网络 ε_θ 去预测那个 ε，预测值和真值的 MSE 就是损失。**就是预测噪声，就这么简单**。所有扩散模型的训练代码都是这一行在跑循环。

> **TA 只需知道**：训练目标 = MSE 预测噪声。ELBO 推导附录 B。

### 0.4 和 VAE/GAN 的一句话类比

| 模型 | 核心思路 |
|---|---|
| **VAE** | 一步压缩到潜变量 z，再一步重建 |
| **GAN** | 生成器 G 和判别器 D 对抗博弈 |
| **扩散模型** | **把生成分成 T 步小增量：每步只做一点点"去噪"** |

**扩散模型的哲学**：不再追求"一锤定音"——一次性从噪声生成完美图像/动作太难；不如把任务拆成 T=1000 个简单子任务，每个子任务只要去一点点噪，累积起来就是从纯噪声生成完美样本。**这跟 AI 动画里的渐进式生成思路天然契合**。

## 第 1 章 为什么要学扩散模型 [L0 核心]

### 1.1 一句话说清扩散模型是什么

**扩散模型（Diffusion Models）= 一类生成模型，通过学习"逐步去噪"来生成样本**。训练时教网络"给定加了噪的图像/动作，预测里面的噪声"；推理时从纯高斯噪声出发，反向去噪 T 步得到干净样本。

**它解决什么问题**：
- GAN 训练不稳定、模式崩溃严重——扩散模型**天然稳定**（MSE 损失，没有对抗博弈）；
- VAE 生成结果模糊——扩散模型**细节非常锐利**（多步精细化累积）；
- 归一化流（normalizing flow）表达能力受限——扩散模型**架构完全自由**（想用什么网络都行）。

**代价**：**采样慢**（需要跑 T 次网络前向，T 常取 1000）。DDIM / Latent Diffusion / consistency models 都在解决这个问题。

### 1.2 TA 为什么必须学扩散模型

**现状（2024-2026）**：text-to-motion 领域的 SOTA 几乎**全是扩散模型**。你以后要做的事情大概率是：

1. **跑 MDM / MotionDiffuse / MLD 生成动作**——文本 prompt 喂进去，生成 120 帧 **SMPL**（Skinned Multi-Person Linear，参数化人体骨骼模型，24 个关节 + shape 参数）动作序列。
2. **训练你们公司自己的 motion diffusion 模型**——用你们的动捕数据 + text 标注，fine-tune 一个 MDM 变体。
3. **motion inpainting / inbetweening**——给定首尾两个 pose，扩散模型填中间 60 帧。EDGE（舞蹈生成）、PhysDiff（物理感知）都是这类工作。
4. **扩散 + 物理**——DiffuseBot、PhysDiff 把扩散和 PPO/物理仿真结合。
5. **Stable Diffusion 的运动版**——你们公司可能已经用 SD 生成过贴图/概念图；motion 扩散就是同样的思想换到了动作空间。

**一句话**：**扩散模型是目前 AI 动画里"从文本/音乐/关键帧生成新动作"这条链路的主算法**。Motion Matching 是检索 + 混合，扩散模型是真正的生成。

### 1.3 和前几天学过的生成模型怎么接

你已经学过的生成模型路线（4 月以来）：

- **VAE**（04-20）：编码器 + 解码器；潜变量 z 连续；生成模糊但稳定
- **GAN**（04-21）：生成器 + 判别器对抗；生成锐利但训练不稳定
- **RL**（04-22）：策略梯度学 π(a|s)；和 VAE/GAN 方向不同，是决策而非生成
- **今天：扩散模型**：**把生成分成 T 步**；既锐利又稳定；采样慢是代价

**和 VAE 的共同点**：都用 ELBO / KL / 重参数化；都是似然派（不对抗）。
**和 VAE 的区别**：VAE 一步编码到 z；扩散模型 T 步加噪（没有编码器，前向过程是**固定的高斯过程**，无参数）。
**和 GAN 的对比**：GAN 对抗训练 → 不稳定、mode collapse；扩散模型 MSE 训练 → 稳定、mode coverage 好。
**和 RL 的接口**：DeepMimic/AMP 用 PPO 学控制策略；而 MDM 用扩散生成 kinematic 动作序列，再丢给物理控制器——**两条路线在 PhysDiff/DiffuseBot 里汇合**。

## 第 2 章 前向过程：怎么把干净数据变成纯噪声 [L0 核心]

> **本章目标**：钉死 **"为什么扩散模型能从 x_0 一步跳到任意 x_t"** 这个核心机制。这个闭式公式不理解，后面 L_simple 训练、反向采样全都悬空。**必须会手算**。

### 2.1 问题出发：生成的困难和逆向思路

如果让你从零生成一张人脸照片，你会怎么做？

**方案 A（GAN）**：直接训一个生成器 G，输入随机向量 z，输出整张人脸。问题：这个函数太复杂，训练难收敛、mode collapse。

**方案 B（VAE）**：编码器压到 z，再一步解码。问题：解码器一次性复原细节太苛刻，结果模糊。

**方案 C（扩散模型）** ⭐：**反过来想——先定义一个简单的"破坏过程"（加噪）把数据毁掉；再学一个"修复过程"（去噪）把噪声还原成数据**。关键洞见是：**破坏过程可以设计得非常简单（固定、无参数、一步一点点加高斯噪声）；而修复过程虽然难，但也只需每步"去掉一点点噪声"——这是一个容易得多的小任务**。

这就是扩散模型的核心哲学：**把难问题（一步生成）拆成 T 个简单子任务（每步微调）**。

> **Shader 类比**：就像你做一个褪色滤镜 shader，每帧把颜色往灰色拉一点点——跑 1000 帧后图像就变成纯灰色。前向过程就是这个固定的、每步一点点、总共 T 步的"加噪滤镜"；而去噪网络要学的是它的逆。

### 2.2 单步前向过程：q(x_t | x_{t-1}) [L0 核心]

前向过程定义为一个 **马尔可夫链（Markov chain）**——每一步 x_t 只依赖前一步 x_{t-1}，与更早的状态无关（回忆 RL 里的马尔可夫性，完全一样）。

每一步的分布是一个**高斯分布**：

```
q(x_t | x_{t-1}) = N(x_t;  √(1-β_t) · x_{t-1},   β_t · I)
```

**逐项翻译**：
- `q(x_t | x_{t-1})` ：**"给定上一步 x_{t-1}，这一步 x_t 的条件分布"**。q 表示它是"真实的前向过程"（无参数，人工设定），区别于后面反向过程里带学习参数的 p_θ。
- `N(x_t; μ, Σ)` ：x_t 服从均值 μ、协方差 Σ 的高斯。
- **均值 `√(1-β_t) · x_{t-1}`**：把上一步的信号**稍微缩小**一点（缩小系数略小于 1）。
- **方差 `β_t · I`**：往里面**加一点新鲜的高斯噪声**。I 是单位矩阵（每个维度独立同分布）。
- `β_t`：**噪声调度（variance schedule）**，一个预先定好的小数序列，t 从 1 走到 T。常用 linear：β_1 = 1e-4、β_T = 0.02，T = 1000，中间线性插值。

**写成重参数化形式（VAE 的老朋友）**：

```
x_t = √(1-β_t) · x_{t-1} + √β_t · ε_{t-1},    ε_{t-1} ~ N(0, I)
```

**一句话直觉**：**每步把信号稍微压缩一点、混进一点独立的高斯噪声**。信号和噪声的比例由 β_t 控制。β_t 越大，这一步噪声加得越猛。

**为什么用 `√(1-β_t)` 而不是直接 `(1-β_t)`**？答：为了保持方差稳定。你可以验证：如果 x_{t-1} 的方差已经是 1（标准化过），那么 x_t = √(1-β_t) · x_{t-1} + √β_t · ε 的方差也是 `(1-β_t) · 1 + β_t · 1 = 1`。**方差守恒，不会随着 t 增大而爆炸或衰减**。这在工程上非常关键——否则 1000 步之后数值就炸了。

### 2.3 马尔可夫链的总体前向：q(x_{1:T} | x_0)

把 T 步拼起来，就得到从 x_0 到 x_T 的完整加噪轨迹：

```
q(x_{1:T} | x_0) = ∏_{t=1}^T q(x_t | x_{t-1})
```

**直觉**：这就是一条"从干净数据 x_0 一路加噪到纯噪声 x_T"的固定轨道。T 足够大（比如 1000）、β 选得对，x_T 就几乎等于 `N(0, I)`——信号被完全抹掉。

**朴素模拟就是"一步一步跑 T 次"**——能跑，但**训练时你不想这样**，因为你每次要随机抽一个 t 来训练网络，总不能每次都从 x_0 跑到 t 吧？这就引出下一节的关键闭式。

### 2.4 前向过程的闭式一步到位（本章核心）[L0 核心] ⭐

这是 DDPM 论文的第一个关键发现（Ho et al. 2020，NeurIPS [A]）：**任意时刻 x_t 都可以由 x_0 一步算出来，不需要真跑 t 次马尔可夫链**。

定义两个辅助变量：

```
α_t   = 1 - β_t
ᾱ_t   = ∏_{s=1}^t α_s   （α 的累积乘积，"alpha bar"）
```

则有**闭式前向公式**：

```
q(x_t | x_0) = N(x_t;  √ᾱ_t · x_0,   (1 - ᾱ_t) · I)
```

**重参数化形式（实际代码里的样子）**：

```
x_t = √ᾱ_t · x_0 + √(1 - ᾱ_t) · ε,     ε ~ N(0, I)
```

**这个公式为什么重要**：
1. **训练时直接从 x_0 跳到任意 x_t**，不用真跑马尔可夫链，一行代码搞定；
2. 它揭示了 x_t 就是 **"信号 × √ᾱ_t  +  噪声 × √(1-ᾱ_t)"** 的线性组合——ᾱ_t 就是"剩余信号能量"，1-ᾱ_t 就是"累积噪声能量"；
3. 当 t → T 时，`ᾱ_T ≈ 0`，所以 `x_T ≈ ε`——确实退化成纯高斯。

> **推导草图（跳过也可以）**：两个独立高斯加起来还是高斯，方差可加。把 x_t = √(1-β_t) · x_{t-1} + √β_t · ε 递归展开，就能把所有中间 ε 归并成一个大 ε，系数就是 √ᾱ_t 和 √(1-ᾱ_t)。完整推导见附录 B。

### 2.5 手算一：5 步调度表（必须会）[L0 核心] 🔢

**题目**：设 T=5，采用线性调度 `β_t ∈ {0.1, 0.15, 0.2, 0.25, 0.3}`（为了手算取大数，真实工程里 β 小得多）。求 α_t、ᾱ_t 表格，并算出当 x_0 = 1.0 时 x_5 的信号系数和噪声系数。

**第 1 步：算 α_t = 1 - β_t**

| t | β_t | α_t = 1-β_t |
|---|---|---|
| 1 | 0.10 | 0.90 |
| 2 | 0.15 | 0.85 |
| 3 | 0.20 | 0.80 |
| 4 | 0.25 | 0.75 |
| 5 | 0.30 | 0.70 |

**第 2 步：算 ᾱ_t = α_1 × α_2 × ... × α_t**

- ᾱ_1 = 0.90
- ᾱ_2 = 0.90 × 0.85 = **0.765**
- ᾱ_3 = 0.765 × 0.80 = **0.612**
- ᾱ_4 = 0.612 × 0.75 = **0.459**
- ᾱ_5 = 0.459 × 0.70 = **0.3213**

**第 3 步：算信号系数 √ᾱ_t 和噪声系数 √(1-ᾱ_t)**

| t | ᾱ_t | √ᾱ_t（信号） | 1-ᾱ_t | √(1-ᾱ_t)（噪声）|
|---|---|---|---|---|
| 1 | 0.9000 | 0.9487 | 0.1000 | 0.3162 |
| 2 | 0.7650 | 0.8746 | 0.2350 | 0.4848 |
| 3 | 0.6120 | 0.7823 | 0.3880 | 0.6229 |
| 4 | 0.4590 | 0.6775 | 0.5410 | 0.7355 |
| 5 | 0.3213 | 0.5668 | 0.6787 | 0.8239 |

**第 4 步：代入 x_0 = 1.0**

```
x_5 = √ᾱ_5 · x_0 + √(1-ᾱ_5) · ε
    = 0.5668 · 1.0  +  0.8239 · ε
    = 0.5668 + 0.8239 · ε
```

**解读**：走完 5 步后，信号从 1.0 衰减到 0.5668（还剩 56.7%），噪声系数涨到 0.8239。**如果你把 T 从 5 换成 1000、β 取正常的 linear schedule**，最终 √ᾱ_T ≈ 0.0068，噪声系数 ≈ 1.0——信号被彻底抹掉，x_T ≈ ε 就是标准正态。这就是前向过程的"从干净到纯噪"。

> **重要**：**把这张表和公式在笔记本上亲手算一遍**。理解了 ᾱ_t 是"信号残余率"，后面 L_simple 的 MSE、CFG 的权重推导都会顺。

### 2.6 噪声调度的选择 [L1 扩展]

工程上常见三种 β 调度：

1. **Linear schedule（DDPM 2020）**：β 从 1e-4 到 0.02 线性增长。简单、效果 OK。
2. **Cosine schedule（Improved DDPM 2021）**：Nichol & Dhariwal [A] 观察到 linear 在低分辨率数据上**末尾太猛**——ᾱ_T 掉得太快，浪费了步数。改成 cosine 之后 ᾱ_t 曲线更平缓，FID 显著提升。**MDM 用 cosine**。
3. **Quadratic / sigmoid** 等自定义调度，看数据集手调。

**TA 层面你需要知道**：调度不是玄学，是一个**提高采样质量的超参数**。拿到代码先看默认哪种，改数据集时可能要调。**不要从零设计调度——用论文默认的就行**。

<summary>① 如果 T=1000，β_t 全部取 0.001（常数），ᾱ_T 大概是多少？</summary>

这个值还挺大——说明 β 常数 0.001 太温柔，T=1000 步走完信号还剩 37%，x_T 还不够接近纯噪声。真实的 linear schedule 末尾 β_T 会到 0.02，ᾱ_T ≈ 4.8e-5，信号基本被抹光。
</details>

<details>
<summary>② 为什么 √(1-β_t) 和 √β_t 要开根号，不能直接用 (1-β_t) 和 β_t ？</summary>

因为标准差才能线性叠加到信号和噪声上去。如果 x_{t-1} 的方差是 1，那么 x_t = √(1-β_t) · x_{t-1} + √β_t · ε 的方差 = (1-β_t)·1 + β_t·1 = 1，方差守恒。如果写成 (1-β_t)·x_{t-1} + β_t·ε，方差就会随着 t 指数衰减到 0，信号被挤压没了。
</details>

## 第 3 章 反向过程：让网络学会去噪 [L0 核心]

> **本章目标**：把 DDPM 的 **反向过程**、**训练目标 L_simple**、**一步反向采样公式** 三件事钉死，包含一次完整的一步反向采样手算。

### 3.1 反向过程的直觉（为什么能学）

前向过程是固定的"加噪滤镜"——从 x_0 毁到 x_T。**反向过程就是要把它反过来**：给定 x_T（纯噪声），一步步去掉噪声，最终得到一张符合真实数据分布的 x_0。

数学上，反向过程也是个马尔可夫链：

```
p_θ(x_{t-1} | x_t) = N(x_{t-1};  μ_θ(x_t, t),   Σ_θ(x_t, t))
```

**重点看这里**：
- **p_θ** 带 θ，说明是**有参数的学习模型**（网络），区别于 q（固定的前向）。
- 输入 x_t 和时间步 t，输出均值 μ_θ 和方差 Σ_θ（DDPM 里一般固定 Σ，只学均值）。
- 这是"给定当前带噪状态 x_t，预测上一步更干净的 x_{t-1}"。

**关键定理（不推导）**：**当 β_t 足够小时，真实的反向过程 q(x_{t-1} | x_t, x_0) 也是一个高斯分布**。这是 Feller 1949 的结果，DDPM 论文引用。**所以用高斯参数化 p_θ 是合理的**——分布族选对了，剩下就是学均值和方差。

### 3.2 训练目标的化简：从 ELBO 到 L_simple [L0 核心]

DDPM 论文（Ho 2020 [A]）从最大化 log-likelihood 出发，推出 ELBO，再化简得到**训练实际用的简化目标**：

```
L_simple = E_{t, x_0, ε} [ ||ε - ε_θ(x_t, t)||² ]
```

**逐项翻译**：
- **E_{t, x_0, ε}**：对三个随机变量取期望——t 从 `{1, ..., T}` 均匀抽；x_0 从训练集里抽；ε 从 `N(0, I)` 抽。
- **x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε**：用前向闭式一步加噪到 x_t（训练时不用真跑马尔可夫链）。
- **ε_θ(x_t, t)**：**噪声预测网络**——输入带噪样本 x_t 和时间步 t，输出对"里面混了哪个 ε"的预测。
- **||·||²**：MSE 损失。

**整个训练过程一句话**：**随机抽 t、随机抽 x_0、随机抽 ε、算 x_t、让网络 ε_θ(x_t, t) 去预测 ε，算 MSE，反传更新**。没了。

> **和 VAE 训练对比**：VAE 训练要在 loss 里写 `recon_loss + KL_term`，而 DDPM **只有一个 MSE**——简洁到夸张。这就是 Ho 2020 最令人震撼的发现：**一个看起来复杂的分层变分模型，训练代码最后就是一行 MSE**。

**L_simple 的三条直觉解读**：
1. **网络只需学"里面有多少噪声"**，不需要学"完整的数据分布"——后者无限难，前者只是回归一个高斯变量，容易得多。
2. **每个 t 都是一个子任务**：小 t 时 x_t 接近干净数据，让网络"去一点点噪"；大 t 时 x_t 接近纯噪，让网络"粗暴找大方向"。T 个子任务难度差异很大，但都是同一个网络在学（条件于 t）。
3. **和 VAE 的共同点**：都用重参数化 + MSE（VAE 的 reconstruction loss 就是 MSE）；区别是 VAE 一步到 z，DDPM 是 T 步。

### 3.3 网络输入/输出长什么样

**输入**：
- `x_t`：和 x_0 同维度的张量（图像就是 H×W×C；动作就是 frames × joints × feats）。
- `t`：一个整数（0..T-1），需要编码成向量——常用 **sinusoidal embedding**（复用 Transformer 的 positional encoding 思想），再过两层 MLP 映射到某个维度。

**输出**：
- `ε_θ(x_t, t)`：**和 x_t 同维度**的张量——预测里面混了哪个噪声。

**主干网络的选择**：
- **图像扩散（DDPM/SD）**：用 **U-Net**——CNN encoder-decoder 带跳连；每层注入 t 的 embedding（加到特征上或做 AdaIN）。
- **运动扩散（MDM）**：用 **Transformer**——把 frames 当成 token，t 和 text 当成额外 token 拼在开头。**不用 U-Net，因为运动数据是时序而非 2D 空间**。

![fig4](/images/Diffusion-Models-DDPM-DeepDive/fig4.png)

### 3.4 反向采样公式：从 x_T 生成 x_0

训练完了，怎么采样？DDPM 推出的**反向一步采样公式**：

```
x_{t-1} = (1/√α_t) · ( x_t  -  (β_t / √(1-ᾱ_t)) · ε_θ(x_t, t) )   +   σ_t · z
```

其中 `z ~ N(0, I)`（只在 t>1 时加；t=1 时 z=0），`σ_t` 是反向过程的方差，DDPM 取 `σ_t² = β_t`（最简单）或 `σ_t² = β̃_t`（posterior variance，效果稍好）。

**逐项读懂**：
1. `x_t  -  (β_t / √(1-ᾱ_t)) · ε_θ`：**从 x_t 里扣掉网络预测的"一点噪声"**。系数 β_t / √(1-ᾱ_t) 是 DDPM 推导出来的"本步该扣多少"。
2. `× (1/√α_t)`：把扣噪后的信号**放大回去**（补偿前向那一步 `√(1-β_t)=√α_t` 的压缩）。
3. `+ σ_t · z`：**再注入一点新鲜噪声**。这一项是让采样过程保持随机性的关键——**没有这项，采样就退化成确定性的、所有轨迹都一样**。

> **直觉类比**：就像你一步步把 Shader 的"褪色滤镜"反过来做——先扣掉一部分灰、放大信号、再加一点抖动让结果多样。跑 T=1000 次，从 x_T ≈ 纯噪 走到 x_0 = 干净样本。

**DDPM 采样伪代码**：

```
x_T ~ N(0, I)
for t = T, T-1, ..., 1:
    eps_hat = eps_theta(x_t, t)
    mu = (1/sqrt(alpha_t)) * ( x_t - (beta_t/sqrt(1-alpha_bar_t)) * eps_hat )
    if t > 1:
        z ~ N(0, I)
        x_prev = mu + sigma_t * z
    else:
        x_prev = mu    # final step: no noise injection
    x_t = x_prev
return x_0  # = x_prev from t=1
```

**这是 DDPM 原版采样，需要跑 T=1000 次前向**——慢。解决方案：**DDIM（§3.7，L1 章节）**、**Latent Diffusion（§4.6，L1 章节）**。

### 3.5 手算二：一步反向采样（必须会）[L0 核心] 🔢

**题目**：沿用 §2.5 的 5 步调度表。假设当前 t=3，x_3 = 0.80（一维标量），网络预测 ε̂ = ε_θ(x_3, 3) = 0.50。求 x_2。

**查表**（§2.5）：β_3 = 0.20，α_3 = 0.80，ᾱ_3 = 0.612。

**第 1 步：算 √α_3、√(1-ᾱ_3)、β_3 / √(1-ᾱ_3)**
- √α_3 = √0.80 ≈ 0.8944
- 1 - ᾱ_3 = 0.388，√(1-ᾱ_3) ≈ 0.6229
- β_3 / √(1-ᾱ_3) = 0.20 / 0.6229 ≈ 0.3211

**第 2 步：算均值 μ**
```
μ = (1 / 0.8944) × ( 0.80  -  0.3211 × 0.50 )
  = 1.1180 × ( 0.80 - 0.1606 )
  = 1.1180 × 0.6394
  ≈ 0.7148
```

**第 3 步：加方差项**。σ_3² = β_3 = 0.20，σ_3 ≈ 0.4472。因为 t=3>1，要加 z ~ N(0,1)。假设本次采样抽到 z = -0.3（随机的；下一次采样结果就不同）：

```
x_2 = 0.7148  +  0.4472 × (-0.3)
    = 0.7148  -  0.1342
    ≈ 0.5806
```

**解读**：
- 输入 x_3 = 0.80（带噪），网络说"里面有 ε=0.5 的噪声"
- 经过"扣噪 + 放大 + 注入新噪声"三步，得到 x_2 ≈ 0.581
- 如果 z 抽到 +0.3，x_2 ≈ 0.849——**不同的 z 给不同的 x_2**，这就是采样的随机性来源
- 继续往前跑，得到 x_1、x_0

> **练习**：用 x_2 = 0.5806，假设 ε̂_2 = 0.4，算 x_1。**自己动手算一遍**，这个公式你就彻底懂了。

### 3.6 训练和采样的完整流程图

```
Training:
    repeat:
        x_0 ~ dataset
        t ~ Uniform{1, ..., T}
        eps ~ N(0, I)
        x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * eps
        loss = || eps - eps_theta(x_t, t) ||^2
        loss.backward()   # update network parameters theta

Sampling:
    x_T ~ N(0, I)
    for t = T, T-1, ..., 1:
        eps_hat = eps_theta(x_t, t)
        if t > 1:
            z ~ N(0, I)
        else:
            z = 0
        x_prev = (1/sqrt(alpha_t)) * ( x_t - (beta_t/sqrt(1-alpha_bar_t)) * eps_hat ) + sigma_t * z
        x_t = x_prev
    return x_0   # = x_prev from t=1
```

**这就是 DDPM 的完整骨架**——训练 3 行、采样 3 行。图像扩散、运动扩散、音乐扩散本质都是这套骨架换个主干网络和数据格式。

![fig5](/images/Diffusion-Models-DDPM-DeepDive/fig5.png)

### 3.7 DDIM：加速采样（概念补）[L1 扩展]

**动机**：DDPM 要跑 1000 步，太慢。Song 等人 2021（ICLR [A]）提出 **DDIM（Denoising Diffusion Implicit Models）**，核心贡献：

1. **重新定义反向过程为非马尔可夫的隐式模型**，数学上等价地（共享同一个 ε_θ 网络）让采样可以**跨步跳**：20 步、50 步、100 步都能出结果。
2. **当方差参数 η=0 时采样变成确定性**——相同 x_T 每次都得到相同 x_0（对 latent interpolation 很有用）。

**工程要点**：
- **DDIM 不用重新训练**——它和 DDPM 共用同一个训好的 ε_θ 网络。只是采样公式不一样。
- **MDM 默认用 DDIM** 50 步采样；Stable Diffusion 常用 DDIM/DPM-Solver 20-50 步。
- **采样质量：DDIM(50) ≈ DDPM(1000)**，但速度快 20 倍。

**TA 只需知道**：训练归训练（还是 L_simple），采样可以换成 DDIM。**改 scheduler.py 一行代码的事**，不用懂证明。

### 3.8 本章自测

<details>
<summary>① 为什么 L_simple 里只用 MSE 就够了，不像 VAE 还要加 KL 项？</summary>

Ho 2020 证明：从 ELBO 出发的 DDPM 完整目标 L_VLB，其中**每一步的 KL 都能解析写成 MSE 的形式**（两个高斯的 KL 加上常数），所以最后汇总就是一个大 MSE。同时他们实验发现**丢掉那些随 t 变化的权重系数（用常数 1 替代）反而训得更好**——于是得到更简洁的 L_simple。细节推导见附录 B。
</details>

<details>
<summary>② 采样时为什么 t=1 的最后一步不加 z？</summary>

因为 t=1 的 x_0 就是最终输出，已经不再需要"保持多样性"；加 z 反而会给干净数据引入额外噪声。数学上，最后一步实际就是取均值 μ 作为 x_0（这对应于"返回 posterior 的均值"）。
</details>

## 第 4 章 条件扩散与 Classifier-Free Guidance [L0 核心]

> **本章目标**：讲清怎么让扩散模型"听话"——根据文本、类别、姿态等条件生成。**CFG 公式和 w 权重的手算必须会**，因为你以后每次跑 MDM / Stable Diffusion 都会调这个 guidance scale。

### 4.1 问题出发：怎么控制扩散模型生成特定内容？

第 3 章的 DDPM 训练出来后只能做**无条件生成**——给它纯噪声，它吐一张某种风格的人脸/一段风格随机的舞蹈。但 TA 真正想要的是：**"给一句文本 `walk forward then turn left`，生成 120 帧对应动作"**。这需要把**条件 c**（文本、类别、音乐特征、关键帧等）注入生成过程。

两种主流做法：
1. **Classifier Guidance（CG）**——外挂一个分类器 p(c|x_t) 用梯度引导，Dhariwal & Nichol 2021 [A]。
2. **Classifier-Free Guidance（CFG）**——**不用外挂分类器**，直接在扩散网络里一起训条件和无条件两个分支，Ho & Salimans 2021 [A]。

**CFG 是现在的事实标准**（Stable Diffusion、MDM、MotionDiffuse 全用它）。本章重点讲 CFG。

### 4.2 Classifier Guidance（CG）：历史背景 [L1 扩展]

**先说 CG 是为了理解 CFG 的"为什么"**。

**CG 的核心思路**（Dhariwal & Nichol 2021）：
1. 先训一个无条件扩散模型 ε_θ(x_t, t)；
2. 单独训一个**分类器** p_φ(c | x_t)——输入带噪图像，输出是哪一类的概率；
3. 采样时在每一步用**分类器的梯度** `∇_{x_t} log p_φ(c|x_t)` 来"把去噪方向推向目标类"。

**问题**：
- 需要单独训一个**能识别带噪图像**的分类器——这个分类器本身就不好训（噪声层级太多）。
- 两阶段训练，复杂。
- 分类器梯度不稳定，容易过度放大。

**CG 主要用于 ImageNet 类别条件生成，在文本条件上做不出来**（没有好的"噪声鲁棒的 text classifier"）。

### 4.3 Classifier-Free Guidance（CFG）：优雅的替代 [L0 核心] ⭐

**CFG 的核心思路**（Ho & Salimans 2021 [A]）：**不要外挂分类器，让同一个扩散网络同时学"有条件"和"无条件"两个分支**。训练时随机丢掉条件（比如 10% 的概率把文本 embedding 换成特殊的 ∅ token）——这样网络同时学会：
- `ε_θ(x_t, t, c)`：看到条件 c 时的噪声预测
- `ε_θ(x_t, t, ∅)`：没有条件时的噪声预测

**采样时**，在每一步用**两个预测的线性组合**：

```
ε̃(x_t, t, c) = (1 + w) · ε_θ(x_t, t, c)  -  w · ε_θ(x_t, t, ∅)
```

其中 `w` 是**引导强度（guidance scale）**，工程上常取 3~15。

**公式的直觉解读**：
- 把它改写成 `ε̃ = ε_θ(c) + w · [ε_θ(c) - ε_θ(∅)]`；
- 括号里 `ε_θ(c) - ε_θ(∅)` 是 **"条件带来的方向修正"**——有文本和没文本时网络预测的差；
- `w` 放大这个修正——**越大越"忠于条件"，但超过某个阈值后会导致样本失真**（过曝、扭曲、动作抽搐）。

**另一种等价写法**（有些论文用）：

```
ε̃ = ε_θ(∅)  +  s · [ε_θ(c) - ε_θ(∅)]     其中 s = 1+w
```

两种写法**完全等价**，只是 scale 的定义差 1。Stable Diffusion 代码里叫 `guidance_scale = s = 1+w`，默认 7.5。

### 4.4 手算三：CFG 在 w=0, 3, 7.5 的对比（必须会）[L0 核心] 🔢

**题目**：假设某一步扩散中，同一个 x_t 和 t 下，两个分支预测结果是：
- `ε_θ(x_t, t, c) = 0.60`（条件分支，表示"按文本 prompt 来应该预测的噪声"）
- `ε_θ(x_t, t, ∅) = 0.40`（无条件分支，表示"普通去噪该预测的噪声"）

计算 w = 0, 1, 3, 7.5 时的 `ε̃`：

**w=0**（相当于不引导，只用条件分支）：
```
ε̃ = (1+0) × 0.60  -  0 × 0.40  =  0.60
```
→ 退化成"单纯用条件预测去噪"，效果等同于直接用 ε_θ(c)。

**w=1**：
```
ε̃ = 2 × 0.60  -  1 × 0.40  =  1.20 - 0.40  =  0.80
```
→ 比条件分支多偏离无条件 0.20。

**w=3**（MDM 默认）：
```
ε̃ = 4 × 0.60  -  3 × 0.40  =  2.40 - 1.20  =  1.20
```
→ 比条件分支的 0.60 放大了 2 倍。

**w=7.5**（**注意约定差异**：本文用 `ε̃ = (1+w)ε_c - w ε_∅` 的 w；Stable Diffusion 代码里的 `guidance_scale` 是 `s = 1+w`，SD 默认 `guidance_scale = 7.5` 对应本文 `w ≈ 6.5`。为保持和论文计算口径一致，这里仍按 `w = 7.5` 计算）：
```
ε̃ = 8.5 × 0.60  -  7.5 × 0.40  =  5.10 - 3.00  =  2.10
```
→ 比条件分支的 0.60 放大了 3.5 倍。

**观察规律**：
- w 越大，`ε̃` 被推得越远离 `ε_θ(∅)`、越靠向 `ε_θ(c)` 的"方向极限"；
- 文本条件的影响被"过曝式放大"；
- **放大太猛（w>15）会导致样本不再落在真实数据分布上**——图像过曝出现色块，动作出现关节抽搐、速度突变。

**实际工程选择**：

| 应用 | 推荐值 | 来源 |
|---|---|---|
| Stable Diffusion (文生图) | guidance_scale=7.0~7.5（即 `s`，对应本文 `w≈6.0~6.5`） | SD 官方默认 |
| MDM (文生动作) | w=2.5~3.0 | MDM 论文 [A] |
| MotionDiffuse | w=2.5 | MotionDiffuse 论文 [A] |
| EDGE (音乐生舞蹈) | w=2.0 | EDGE 论文 [A] |

**为什么文生图 w 比文生动作大**：图像的自然先验（"看起来像照片"）本身就很强，稍微过曝反而让条件更突出；而动作数据的"物理合理性"约束严格得多，w 大了立刻出现穿模、飘移、抽搐。**TA 调 MDM 时一般从 w=2.5 开始，往上调到 w=4 就停**。

### 4.5 训练 CFG 模型的技巧

**训练循环里唯一的改动**：每个 minibatch 随机把一部分样本的条件 c 替换成 ∅ 特殊标记（通常 10%）。伪代码：

```
for batch in dataloader:
    x_0, c = batch
    # drop condition with 10% probability
    drop_mask = (rand(batch_size) < 0.1)
    c[drop_mask] = null_token

    # standard DDPM training, but condition may be null
    t = uniform(1, T)
    eps = N(0, I)
    x_t = sqrt(alpha_bar_t) * x_0 + sqrt(1 - alpha_bar_t) * eps
    loss = || eps - eps_theta(x_t, t, c) ||^2
    loss.backward()
```

**就这一行改动，一个网络学会两个技能**：既能做有条件生成（90% 数据），也能做无条件去噪（10% 数据）。采样时用 CFG 公式组合就行。

### 4.6 Latent Diffusion：把扩散搬到潜空间 [L1 扩展]

**动机**：原始 DDPM 在像素空间跑，512×512 RGB 图像 = 786432 维，算 T=1000 步太贵。Rombach 等人 2022 CVPR [A] 的 **Latent Diffusion Model (LDM)**，也就是 **Stable Diffusion** 的原始论文，提出：

1. **先训一个 VAE**（或 VQ-VAE）把图像压到 64×64×4 = 16384 维的潜空间；
2. **在潜空间里跑扩散**——ε_θ 处理的是 latent，不是像素；
3. **推理时 VAE decoder 把 latent 映射回 512×512 像素**。

**效果**：计算量降到 ~1/48，生成质量不降反升（VAE 潜空间已经把冗余去掉了，扩散只学"语义偏差"）。

**运动版本**：**MLD（Motion Latent Diffusion, Chen 2023 CVPR [A]）** 就是把这个思路搬到动作——先训 motion VAE 把 120 帧动作压到 1×256 的 latent，再在 latent 上跑扩散。比原始 MDM 快 10 倍。

**TA 必须知道的三句话**：
1. **LDM = VAE + 扩散的组合拳**；
2. **Stable Diffusion 就是 LDM 的产品化**；
3. **MLD 是 MDM 的 latent 加速版本，部分项目会用 MLD 不用 MDM**。

### 4.7 本章自测

<details>
<summary>① CFG 训练时为什么要"随机丢掉条件"？如果不丢会怎么样？</summary>

不丢条件的话，ε_θ(x_t, t, ∅) 分支从来没训过——它根本不知道"没有条件时该预测什么"。采样时用 CFG 公式 `(1+w)ε_c - w ε_∅` 就会算出乱七八糟的结果。丢掉条件（10% 概率把 c 换成 ∅）强迫网络同时学会"有条件"和"无条件"两种模式。训练时用一个网络，采样时通过 CFG 把两种模式对比放大，就能实现"强条件引导"。
</details>

<details>
<summary>② 为什么 w 不能无限大？</summary>

CFG 本质是把"条件方向"从无条件基线方向外推。w 越大外推越远——但数据分布在这个方向上有物理/统计约束，外推超过某个阈值就跑出真实数据支集。视觉表现是：图像过曝、颜色溢出；动作抽搐、关节扭曲、速度爆炸。所以 w 是**超参**，要针对具体数据集和任务调，不是"越大越好"。
</details>

## 第 5 章 扩散模型在动画 TA 的真实落点 [L0 核心]

> **本章目标**：把扩散模型**从图像领域桥接到动作领域**。把 MDM（Motion Diffusion Model）讲到能画出架构图的程度；给 VAE/GAN/RL/Diffusion 四条路线的对比表；列出 TA 日常最可能接触的五类具体工作。

### 5.1 从图像扩散到运动扩散：数据格式的迁移

**图像数据**：`[H, W, C]`，例如 `[256, 256, 3]`。空间上相邻像素强相关，所以用 **U-Net（CNN+skip-connection）** 当主干。

**运动数据**：`[frames, joints, feats]`，典型是 `[120, 24, 9]`——120 帧、24 个关节、每个关节 9 维特征（**6D rotation**：用旋转矩阵前两列 6 个数表示朝向，连续无奇异性，比欧拉角/四元数更适合神经网络 + 3D 位置偏移）；或者 263 维的 **HumanML3D**（常用的 text-motion 数据集，14 小时动捕 + 44k 文本标注，把全身动作编成 263 维特征向量）表示。**时序维度 frames 比空间更重要**，所以用 **Transformer** 当主干——把每帧当成一个 token，跨帧自注意力捕捉时序依赖。

**加噪的对象变了**：不再往像素加高斯噪声，而是**往关节的 6D rotation / position 特征加噪**。`x_t` 是一个 `[120, 24, 9]` 的张量，加的 ε 也是同样 shape 的标准正态。前向闭式、L_simple、CFG 三大公式**一字不改**，直接移植。

> **TA 核心认知**：**扩散模型的数学框架与"什么数据"无关**。只要你能把数据表示成张量，就能用 DDPM。唯一要换的是主干网络（U-Net / Transformer / PointNet++ / 等等）。

### 5.2 MDM：Motion Diffusion Model 架构详解 [L0 核心]

**MDM**（Tevet et al. ICLR 2023 [A]）是 motion diffusion 的奠基之作。完整架构：

```
输入:
  文本 text, 例如 "a person walks forward then turns left"
  或空输入 ∅ (用于 CFG 的无条件分支)

处理:
  1. Text encoder (**CLIP** text encoder, 冻结，不参与训练。CLIP = Contrastive Language-Image Pre-training，OpenAI 2021 预训练的文本-图像对齐模型，把任意文本编码成 512 维向量): 
     text → text_embedding (512维)
  2. Time encoder: 
     t → t_embedding (sinusoidal → MLP → 512维)
  3. Concat:
     [t_emb + text_emb]   作为 condition token c

Transformer 主干:
  输入 token 序列: [c, x_t frame_1, x_t frame_2, ..., x_t frame_120]
                   长度 = 1 + 120 = 121 个 token
  8 层 Transformer encoder (无因果 mask, 全序列双向自注意力)
  每个 token 维度 = 512

输出:
  取 frame token 对应的输出: [frame_1_out, ..., frame_120_out]
  线性投影: 512维 → 263维 (HumanML3D 特征)
  = ε_θ 对每一帧的噪声预测
```

**四个关键设计决策**：

1. **Transformer 不是 U-Net**：时序建模用 Transformer 的自注意力更自然；全序列看一眼能捕捉长程依赖（走 1 秒再转弯的因果）。
2. **text 和 t 一起塞进 token 序列**：不是加到特征上，而是当作独立 token；这样注意力可以学"哪几帧最该听 text 的哪部分"。
3. **预测 x_0 而不是 ε**：**MDM 和 DDPM 原版有一处不同**——它让网络直接预测**干净的 x_0**，不是噪声 ε。两者数学上可互相换算：给定 x_t 和 ε̂，能算出 x̂_0；给定 x_t 和 x̂_0，也能算出 ε̂。**MDM 预测 x_0 的好处是方便加额外的几何 loss**——可以把 forward kinematics 算出的关节位置 loss、脚部触地 loss 直接套在 x̂_0 上。
4. **在 x̂_0 上加几何约束 loss**：MDM 的训练 loss 不止 MSE，还加了：
   - **Position loss**：让预测的 x̂_0 解出的关节位置接近真值；
   - **Foot contact loss**：让脚触地时速度为 0，防止"滑步"；
   - **Velocity loss**：相邻帧的变化平滑。

**MDM 训练 loss 总合**：

```
L_total = L_simple  +  λ_pos · L_pos  +  λ_foot · L_foot  +  λ_vel · L_vel
```

**TA 改 MDM 时最常动的就是这几个 λ 权重**——当生成的动作"脚滑"就调大 λ_foot；"抖动"就调大 λ_vel。

### 5.3 MDM 的变体和兄弟们

**MotionDiffuse（Zhang 2022 TPAMI [A]）**：和 MDM 几乎同期独立提出的 motion diffusion。主要区别是**预测 ε 不预测 x_0**（更接近原版 DDPM），在 HumanML3D 上指标略低于 MDM 但更稳定。

**MLD（Motion Latent Diffusion, Chen 2023 CVPR [A]）**：把 motion 先过 VAE 压到 1×256 的 latent，再在 latent 上跑扩散。**推理速度比 MDM 快 10 倍，质量接近**。大规模部署优选 MLD。

**EDGE（Tseng 2023 CVPR [A]）**：**音乐 → 舞蹈**扩散生成。条件 c 换成音乐特征（Jukebox embedding）。支持 **editability**——可以固定某些帧（关键帧锁定），只让扩散填中间，做 motion inpainting/inbetweening。TA 做舞蹈动画必看。

**PhysDiff（Yuan 2023 ICCV [A]）**：**扩散 + 物理仿真**。把 MDM 生成的 kinematic 动作丢给物理仿真器走一遍，把**物理不合理性（穿模、飘浮、脚滑）作为梯度回传修正扩散输出**。这是扩散和 RL/物理的真正结合点，给 TA 的启示：**"纯 kinematic 扩散"和"纯 RL physics"不是竞品，而是可以串联的两段流水线**。

**DiffuseBot（Wang 2023 NeurIPS [A]）**：**扩散生成软体机器人形态 + 物理演化**。更前沿，短期 TA 不会直接用，但启发 "扩散可以生成的不仅是动作，还可以是 controller 参数、角色形态"。

### 5.4 VAE / GAN / RL / Diffusion 四条路线对比表

四种生成/决策范式，TA 都会碰到。一张表对比：

| 维度 | **VAE** | **GAN** | **RL (PPO)** | **Diffusion** |
|---|---|---|---|---|
| 训练稳定性 | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 生成质量 | ⭐⭐ (模糊) | ⭐⭐⭐⭐ (锐利) | — (决策) | ⭐⭐⭐⭐⭐ (细节+多样性) |
| 模式覆盖 | ⭐⭐⭐⭐ | ⭐⭐ (mode collapse) | — | ⭐⭐⭐⭐⭐ |
| 推理速度 | ⭐⭐⭐⭐⭐ (1步) | ⭐⭐⭐⭐⭐ (1步) | ⭐⭐⭐⭐⭐ (在线) | ⭐⭐ (T步) |
| 训练代价 | 低 | 中 (对抗) | 高 (交互) | 中 (T×forward) |
| 条件引导 | 有 (CVAE) | 有 (cGAN) | 有 (state/goal) | **强 (CFG)** |
| 动画代表 | **VAE-GAN (MotionVAE)** | **AMP (WGAN-GP)** | **DeepMimic/AMP** | **MDM/MLD/EDGE** |
| TA 实战优先级 | 中 | 中 | **高** | **最高** ⭐ |

**要点总结**：
- **VAE**：潜空间插值/编辑适用，但单独用生成质量不够；和扩散结合（LDM/MLD）是最强组合。
- **GAN**：图像领域曾是 SOTA，但生成式工作逐步让位于扩散；在 AMP 里作为 style discriminator 仍占一席之地。
- **RL**：动画 TA 的决策方向主力（物理控制、实时适应）；不做生成，但和扩散可以串联（PhysDiff）。
- **Diffusion**：**当前 text-to-motion 的主力**；未来 2-3 年 AI 动画论文你会以它为主。

### 5.5 TA 日常最可能接触的五类扩散工作

**① 跑通 MDM inference**
```python
# pseudocode — not runnable as-is, illustrative only
model = load_pretrained_mdm()
text = "a person does a backflip"
text_emb = clip_text_encoder(text)
null_emb = clip_text_encoder("")  # empty string = unconditional branch
x_t = torch.randn([1, 120, 263])  # start from pure noise (empty motion)
w = 2.5
for t in reversed(range(1000)):
    eps_c   = model(x_t, t, text_emb)
    eps_unc = model(x_t, t, null_emb)
    eps_tilde = (1 + w) * eps_c - w * eps_unc   # CFG
    x_t = ddim_step(x_t, eps_tilde, t)          # one denoising step
x_0 = x_t  # final clean motion, shape [1, 120, 263] → decode to FBX
```
**首要任务**：读懂 HumanML3D 数据格式、能把输出 decode 回 SMPL 骨骼。

**② Fine-tune MDM 到公司风格数据**
用公司的动捕数据（比如 500 小时战斗/跳舞）替代 HumanML3D 训练集，继续训 MDM 10-50 epochs。需要你：数据清洗、text 标注、调 λ_foot。

**③ Motion inpainting / inbetweening**
固定首尾几帧，让扩散填中间。关键是**训练时就加 mask 训练** 或**采样时在每步 x_{t-1} 里用 Gaussian 注入已知帧**。EDGE 给出了范本。

**④ 音乐/语音 → 动作**
把 c 从 text 换成音乐特征（CLIP-music、Jukebox）或语音 MFCC。EDGE（舞蹈）、DiffuseStyleGesture（手势）都是这条线。

**⑤ 扩散 + 物理串联**
Step 1：MDM 生成 kinematic 动作；Step 2：DeepMimic/AMP 学一个 controller 去跟踪；Step 3：物理仿真出带动力学的动作。**PhysDiff** 是这条链路的代表。

### 5.6 本章自测

<details>
<summary>① MDM 选 Transformer 而不是 U-Net 的核心原因是什么？</summary>

运动数据的主要结构是"时序相关"（这一帧和前后几帧强相关），不是"空间相关"（像素的 2D 邻接）。Transformer 的全序列自注意力天然适合捕捉任意距离的时序依赖（走 30 帧后转弯的因果）；U-Net 的卷积感受野需要堆很多层才能覆盖 120 帧。此外 Transformer 把 text embedding 和 t embedding 当独立 token 塞进序列的做法更自然——和主序列用同一种机制（注意力）混合。
</details>

<details>
<summary>② MDM 让网络预测 x_0 而不是 ε 有什么好处？</summary>

两个原因。第一，x_0 是"干净的动作"，可以在它上面套物理/几何约束 loss（foot contact、关节位置、速度平滑），这些 loss 在 ε 上没法自然定义。第二，数值上 x_0 的 scale 比 ε 稳定（ε 的 scale 随 t 变化），训练更容易。代价是和原版 DDPM 的 L_simple 推导有一点差异，需要改写损失的权重。
</details>

## 第 6 章 总结与学习路线 [L0 核心]

### 6.1 全篇回顾（四条核心）

**核心一：前向过程（§2）**
- 单步：`q(x_t|x_{t-1}) = N(√(1-β_t) x_{t-1}, β_t I)`，每步压缩信号一点、加一点独立噪声。
- 闭式：`x_t = √ᾱ_t · x_0 + √(1-ᾱ_t) · ε`，**任意时刻一步到位**。
- 调度：linear（DDPM 原版）/ cosine（iDDPM / MDM）/ quadratic。

**核心二：反向过程与训练（§3）**
- 反向也是高斯：`p_θ(x_{t-1}|x_t) = N(μ_θ, Σ_θ)`，只学均值。
- 训练目标：`L_simple = ||ε - ε_θ(x_t, t)||²`，就是 **MSE 预测噪声**。
- 采样：`x_{t-1} = (1/√α_t)(x_t - (β_t/√(1-ᾱ_t)) ε̂) + σ_t z`。

**核心三：CFG（§4）**
- 训练时 10% 丢条件，让网络同时学条件和无条件两个分支。
- 采样：`ε̃ = (1+w)ε_c - w ε_∅`，w 越大越听条件但越易失真。
- MDM w≈3，Stable Diffusion 的 guidance_scale（即 s=1+w）≈7.5。

**核心四：MDM 应用（§5）**
- 主干 Transformer、数据 [120, 24, 9]、预测 x_0 + 几何 loss。
- 兄弟：MotionDiffuse、MLD、EDGE、PhysDiff。
- 五类 TA 工作：inference、fine-tune、inpainting、音乐驱动、扩散+物理。

读完本文，你必须能做到：

2. ✅ **手算一步反向采样**：给 x_t、β_t、α_t、ᾱ_t、ε̂，算出 x_{t-1}；
3. ✅ **解释 CFG 公式每一项**，并说清 w=3 和 w=7.5 的经验差异来源。

### 附录 B：从 ELBO 到 L_simple 的推导草图 [L2 研究向]

> **本附录为可选**。TA 日常工作不需要亲手推，但如果你要读 DDPM 原论文、或和 ML 研究员对接，这一页能帮你看懂公式跳转。

**起点**：DDPM 用变分下界 L_VLB 最大化 log-likelihood：

```
L_VLB = E_q [ -log p_θ(x_0) ] ≤ E_q [ -log p(x_T) - Σ_{t≥1} log (p_θ(x_{t-1}|x_t) / q(x_t|x_{t-1})) ]
```

把这个式子展开并整理，可以拆成 T+1 项 KL 散度：

```
L_VLB = L_T + Σ_{t=2}^T L_{t-1} + L_0
```

其中：
- **L_T = KL(q(x_T|x_0) || p(x_T))**：前向末端分布 vs 标准正态先验。因为 ᾱ_T ≈ 0，这项几乎为 0，可忽略。
- **L_{t-1} = KL(q(x_{t-1}|x_t, x_0) || p_θ(x_{t-1}|x_t))**：真实后验 vs 学习的反向分布。**关键一步**：Ho 2020 推导 q(x_{t-1}|x_t, x_0) 也是高斯，均值为 μ̃_t(x_t, x_0)，方差为 β̃_t（闭式）；两个高斯的 KL 就是一个 MSE 加常数。
- **L_0 = -log p_θ(x_0|x_1)**：最末一步重建项，也能写成 MSE 形式。

**关键参数化**：把 μ_θ 用 ε_θ 表示——`μ_θ(x_t, t) = (1/√α_t)(x_t - β_t/√(1-ᾱ_t) · ε_θ(x_t, t))`。代入 KL 里的 MSE：

```
L_{t-1} ∝ || ε - ε_θ(x_t, t) ||²
```

**丢掉权重系数**（Ho 2020 实证发现去掉反而好）：

```
L_simple = E_{t, x_0, ε} [ || ε - ε_θ(x_t, t) ||² ]
```

完成。**推导细节见 Ho 2020 论文附录**。

### 附录 C：Score-Based 视角与 SDE 形式 [L3 纯数学]

**Score** 指 `∇_x log p(x)`——概率密度的梯度场，指向"密度上升方向"。Song & Ermon（NCSN, NeurIPS 2019 [A]）提出：**训练网络预测每个加噪级别的 score，用 Langevin dynamics 采样**。Song 等人 2021（ICLR [A]）进一步证明：

> **DDPM（Ho 2020）和 Score-based（Song 2019）在数学上完全等价**——都是同一个随机微分方程（SDE）的离散化。差别只是参数化选择：DDPM 预测 ε，Score-based 预测 ∇_x log p。两者可互换。

**SDE 形式**（只给结论）：

- **前向 SDE**：`dx = f(x, t) dt + g(t) dw`，其中 dw 是维纳过程。
- **反向 SDE**：`dx = [f(x, t) - g(t)² ∇_x log p_t(x)] dt + g(t) dw̄`。

在反向 SDE 里，需要知道 **score function** `∇_x log p_t(x)`——这就是 score-based 模型学的东西。DDPM 的 ε_θ 和 score 的关系是 `∇_x log p_t(x) = -ε_θ(x, t) / √(1-ᾱ_t)`。

**TA 跳过理由**：SDE/Fokker-Planck 属于 ML 博士的测度论/随机过程工具；**你日常工程完全不用**。但读顶会论文时看到"我们从 SDE 视角解释..."要知道它和 DDPM 等价，不要怕。

### 附录 D：工程避坑清单

1. **训练 loss 不降**：检查 β schedule 是否合理（不要 β_T > 0.02）、学习率（2e-4 通常够）、是否有 `cosine` 可以试。
2. **采样结果像纯噪**：大概率是 `ᾱ_t` 计算错误（log 累乘精度问题，要用 `torch.cumprod(1-betas).clamp(min=1e-8)`）。
3. **CFG 调大就"糊"**：w 超过 15 基本废了。动作 w≈3，图像 w≈7.5。
4. **MDM 生成动作脚滑**：加大 λ_foot 和 λ_vel。
5. **DDIM 采样质量跌**：确保 scheduler 的 η=0（确定性）或 η=1（随机）选对；步数 50-100 通常足够。
6. **Latent Diffusion 效果一般**：检查 VAE 重建损失——如果 VAE 自己就重建不好，扩散再准也没用。
