## 第 0 章 前置知识回顾 [L0 核心]

> **前置于本章**：4/20 VAE 深讲的 §0–§3、§7.3；4/17 Conv1D 专题 1。

这一章不讲新东西，只是把后面要反复用到的 5 个符号/概念放在这里，方便你随时翻回来看。

### 0.1 三个一句话回顾

- **自编码器 AE**（4/20 §2.4）—— Encoder 把输入 x 压成低维向量 z，Decoder 把 z 解回 x'。训练目标是 x' ≈ x。
- **VAE 的关键突破**（4/20 §3.3）—— Encoder 不再输出一个点 z，而是输出一个**分布** `N(μ, σ²)`，再采样出 z。这让潜空间"连成片"，能插值生成。
- **重参数化技巧**（4/20 §4）—— `z = μ + σ · ε`（`ε ~ N(0,1)`）把随机性外置，让梯度能通过 μ 和 σ 回流到 Encoder。

### 0.2 本文会反复用到的符号

| 符号 | 中文 / English | 一句话含义 |
|---|---|---|
| `x` | 输入数据 / input | 一段动画、一张图、一段音频——这里主要想运动数据 |
| `z_e` | encoder 输出 / encoder output | 连续向量（D 维实数），Encoder 吐出来的"分析结果" |
| `z_q` | 量化后向量 / quantized vector | 从码本里挑出来的那一行，替换掉 `z_e` 送进 Decoder |
| `e_k` | 码本第 k 行 / codebook entry | 码本里的第 k 个候选向量，`k=1..K` |
| `K` | 码本大小 / codebook size | 码本有多少行，T2M-GPT 默认 512，MoMask 每级 512 |
| `D` | 嵌入维度 / embedding dim | 每个码字多少维，通常和 encoder 输出维度一致 |
| `sg[·]` | stop-gradient（梯度截断） | 前向照常走，反向当成常数（梯度设 0）—— PyTorch 的 `.detach()` |

> 📌 **首次出现术语**：**stop-gradient（梯度截断，sg）** 是个"假截断"算子 —— 在计算图里它前向不改数值，只是让反向传播时梯度在这里停下来不再往回流。后面 §3 STE 的数学会大量用它，先记住"sg 就是反向时的一堵墙"。完整用法在 §3.2 兑现，本节只做指针。

### 0.3 一句话抓住 VQ-VAE 的直觉

> **VQ-VAE = VAE + 把连续潜空间换成"从一张表里查最近邻"的离散化**。

你可以先把今天要学的所有东西想成：**如何把 VAE 里的那个连续 z（一个 32 维实数）替换成"从 512 行的码本里挑一行"，并且让整个网络还能训练**。后面一切公式都围绕这句话展开。

## 第 1 章 为什么需要离散潜空间 [L0 核心]

> 4/20 你学的 VAE 能把动画压成一个 32 维的连续向量。连续向量听起来信息足——但你有没有想过一个问题：**GPT / Transformer 吃进去的必须是一串整数索引**（token ID，例如 `[152, 88, 47, ...]`），它不会对着实数向量做"下一个词预测"。那 VAE 吐出来的连续 z 怎么喂进 Transformer？这就是今天要解决的核心矛盾。

![fig1](/images/VQVAE-Principles-DeepDive/fig1.png)
*图 1：连续潜空间 vs 离散码本潜空间的隐喻。左边是一滩水（任意实数），右边是一串可数的珠子（有限 K 个可选码字）。*

### 1.1 Transformer 只吃离散 token：一个被"忽视"的架构约束

回想你在 4/18 学的 Transformer。它的输入是什么？**一串 token ID**——每个 ID 从 0 到 vocab_size−1 之间取整。GPT-2 的 vocab_size ≈ 50257，每个位置就是 `[0, 50256]` 的一个整数。

Transformer 做"下一个 token 预测"时，输出的是**一个 vocab_size 维的概率向量**（softmax 之后），最大概率的那维就是下一个 token。整个机制**天生依赖离散 token ID + embedding lookup**。

这带来一个现实问题：如果你想**像写文字那样自回归地生成动作**（文字 → 动作），你必须先把动作**变成一串整数 ID**。不然 GPT 根本没法接。

> 📌 **首次出现术语**：**tokenize（token 化）** = 把连续或结构化数据切成离散单元（token）并编号。自然语言里 tokenize 是"分词 + 查词表"；动画里 tokenize 就是"切成时间片段 + 每片映射成码本索引"。

### 1.2 VAE 的潜空间为什么不能直接用？

直觉答案：VAE 的 z ∈ ℝ³² 是一个**连续实数向量**，整个空间是不可数的无限（ℝ³² 里有无穷个点）。你没法给每个点配一个整数 ID。

你可能想："那我把 z 的每个维度四舍五入成整数不就行了？" 听着像——但**粒度一控制不住立刻爆炸**。假设每维只保留 256 个值（round 到 [0, 255]），32 维就是 256³² ≈ 10⁷⁷ 种可能，你不可能学一张 10⁷⁷ 行的词表。

**VQ-VAE 的解法：不让 z 自由取值，而是强制它"从一张有限的表里挑一行"**。这张表就是"码本（codebook）"。

### 1.3 一个"吸附"动作：最近邻查表

假设我们有一张 K=3 行、D=2 维的小码本（方便手算）：

```
Codebook =  e_1 = [ 1,  2]
            e_2 = [ 0,  0]
            e_3 = [-1,  1]
```

Encoder 输出一个连续向量 `z_e = [0.8, 1.5]`。我们要把它**吸附**到码本最近的那一行。

**Hand-calc 1 —— 最近邻查表（第 1 个手算，必须会）** 🔢

逐行算欧几里得距离平方（平方是为了避免开根号，单调等价）：

- `‖z_e − e_1‖² = (0.8−1)² + (1.5−2)² = 0.04 + 0.25 = **0.29**`
- `‖z_e − e_2‖² = (0.8−0)² + (1.5−0)² = 0.64 + 2.25 = **2.89**`
- `‖z_e − e_3‖² = (0.8−(−1))² + (1.5−1)² = 3.24 + 0.25 = **3.49**`

最小的是 `e_1`，所以：

- **索引 `k* = 1`**（这就是可以喂给 Transformer 的整数 ID）
- **量化后向量 `z_q = e_1 = [1, 2]`**（送进 Decoder 重建的那一行）
- **量化误差 `z_q − z_e = [0.2, 0.5]`**（损失了多少连续信息）

一句话描述这个过程：**`z_e` 是 Encoder 吐出的"真实分析"，`z_q` 是把它"吸附"到最近码字后的离散替身**。语义差是 `z_q − z_e`，越小说明码本表达力越够。

### 1.4 离散 token 带来的两个附送红利

把连续 z 换成码本索引 `k*`，除了能喂 Transformer，还顺带白捡两样：

1. **压缩率暴涨**：一个 512 维的 float32 向量要 2KB；一个 index（假设 K=512）只要 9 bits ≈ 1 字节。**压缩比约 2000:1**。这是为什么音频编解码器（SoundStream）爱用 RVQ。
2. **天然可以做掩码学习**：Transformer 里的 BERT-style mask、GPT-style autoregressive、MoMask 的 Masked Modeling 全都以"离散 token + 分类预测"为前提。连续向量做"掩码预测"没有现成的 loss 可用（MSE 不是你想要的语义损失）。

### 1.5 本章自测

<details>
<summary>Q1：下列哪种模型"最需要"离散潜空间？A. 简单自编码器（AE） B. 用 Transformer 做自回归生成 C. 变分自编码器 VAE 做潜空间插值</summary>

**答：B**。AE 和 VAE 都在连续潜空间上工作，不需要离散化。B 的 Transformer 自回归预测本质上是"下一个 token 的分类问题"，没有离散 token 根本跑不起来。
</details>

</details>

## 第 2 章 VQ-VAE 架构：Encoder–Quantizer–Decoder [L0 核心]

> §1 我们建立了直觉：**VQ-VAE = VAE + 从码本里查最近邻**。这一章把架构落地到具体的三块组件，看清楚一段动画输入如何变成一串 token、再解回动作。

### 2.1 三块组件的职责

VQ-VAE 在架构上比 VAE 多一块——**量化器（Quantizer）**：

![fig10](/images/VQVAE-Principles-DeepDive/fig10.png)
*图 10：VQ-VAE 架构全图。动画帧序列进入 Encoder（Conv1D 64→128→256 堆叠）得到 `z_e`，送进 Quantizer 做最近邻查表（码本是 K×D 的嵌入表），得到离散替身 `z_q`，再经 Decoder（Deconv1D 256→128→Pose）还原动画帧。全流程 tensor shape 标注清晰。*

```
  x  ──►  Encoder  ──►  z_e  ──►  Quantizer  ──►  z_q  ──►  Decoder  ──►  x̂
 动画              连续向量              最近邻查表          离散替身              重建
(T×263)          (T'×D, 连续)                              (T'×D, 从码本挑)   (T×263)
```

三块各管什么：

| 组件 | 职责 | 实现 | 是否学习参数 |
|---|---|---|---|
| Encoder | 把连续高维输入压到低频序列 `z_e` | 1D 卷积堆叠（T2M-GPT 用 4 倍下采样） | ✅ |
| Quantizer | 把每个 `z_e[t]` 替换成码本里的最近 `e_k` | 最近邻查表 + 三项损失 + STE | ✅（码本是参数矩阵） |
| Decoder | 把离散替身解回原空间 | 1D 反卷积堆叠（上采样回到 T 帧） | ✅ |

![fig2](/images/VQVAE-Principles-DeepDive/fig2.png)
*图 2：VQ-VAE 三块流水线。灰色虚线框标出 argmin 不可导的"断点"，第 3 章 STE 就是用来缝合这个断点的。*

### 2.2 一段 4 秒动画走一遍完整流水线

用 T2M-GPT 的实际配置（来源：Zhang et al., CVPR 2023，[A]）：

- 输入：4 秒 @ 20 fps → 80 帧 × 263 维 = `(80, 263)`（HumanML3D 格式）
- Encoder：下采样 4× → `(20, 512)` 的连续潜序列（20 个 `z_e` 向量，每个 512 维）
- Quantizer：每个 `z_e[t]` 做一次最近邻查表 → 20 个 index `k*_t ∈ {0..511}`
- 送 Decoder 的是 `z_q` 序列：把每个 index 换回对应码字向量，拼成 `(20, 512)`
- Decoder：上采样 4× → 重建 `(80, 263)`

**一句话总结**：一段 4 秒动画在 VQ-VAE 里等价于 **20 个整数 token**。这就是 T2M-GPT 让文字模型"说动作"的基础。

### 2.3 `z_e` 和 `z_q` 的语义差是什么？

这是一对会反复出现的配对量，按 Concept Teaching Pattern 必须先讲清差异再讲公式。

- `z_e` 是 Encoder 根据**当前输入真实计算出来**的连续向量。
- `z_q` 是在 `z_e` 的邻居里找到最近码字后的替身，**必然等于码本里某一行**。
- **差 `z_q − z_e`** = 量化误差。越小说明"替身骗过 Decoder 的能力越强"，Decoder 重建质量越好。

所以你可以这么读三个量：`z_e` 是"真话"，`z_q` 是"码本里能说的最接近的话"，差是"信息损耗"。三项损失里每一项都在用不同方式压缩这个差。

### 2.4 章末自测

<details>
<summary>Q：为什么 Quantizer 的输出 `z_q` 必须是码本里的某一行，不能是别的向量？</summary>

因为 Transformer 要吃整数 ID。我们真正想送 Transformer 的不是 `z_q` 向量本身，而是**它对应的索引 `k*`**。一旦 `z_q` 必须等于 `e_k*`，`k*` 就成了这一帧的唯一 token，可以直接拿去做自回归建模。

## 第 3 章 直通估计器 STE：让 argmin 可训练 [L0 核心] ⭐

> §2 我们搞清楚了三块架构。但一个让 VQ-VAE 差点活不下去的问题冒出来：Quantizer 里的 `argmin_k ‖z_e − e_k‖` 是个**离散选择**，它的导数**几乎处处为零**——你把 `z_e` 挪动 0.001 不会改变选到哪个码字。梯度到这里就断了，Encoder 收不到任何反传信号，怎么学？本章讲清楚 **STE 是怎么"伪造"一条梯度通道**的。

### 3.1 argmin 为什么不可导：一张图看穿

想象 `z_e` 是一个在 2D 平面上滑动的点，三个码字 `e_1, e_2, e_3` 是地标。我们画出"哪个区域里 `e_1` 是最近邻"——这就是 **Voronoi 分区**（每块区域内最近邻都是同一个码字）。

![fig3](/images/VQVAE-Principles-DeepDive/fig3.png)
*图 3：2D 码本的 Voronoi 分区（K=8）。同色区域内部 argmin 的结果完全一样，梯度 = 0；只有跨越边界的瞬间才"跳变"，梯度是 Dirac δ 函数。*

**结论**：区域内部梯度恒为 0，边界上梯度是无穷大（Dirac δ）。这种函数**几乎处处不可导**，没法用常规反向传播。

### 3.2 STE 的数学魔法：一行代码解决问题

Bengio 2013 年提出的**直通估计器（Straight-Through Estimator, STE）** 思路极简：

> **前向传播照常走离散量化；反向传播时假装量化操作不存在，把梯度原样复制过去。**

实现这个思路的 PyTorch 一行代码：

```python
z_q = z_e + (z_q - z_e).detach()
```

**这行代码的作用需要逐符号展开**（这是 §3 必须看懂的核心）：

#### 前向传播：
- `(z_q - z_e).detach()` 就是 `z_q - z_e` 这个数值本身（detach 不改数值）
- 所以 `z_e + (z_q - z_e) = z_q` ✅ 前向结果就是量化后的 `z_q`，没改变。

#### 反向传播：
- 整个计算图是 `output = z_e + (常数)`（因为 detach 让 `(z_q - z_e)` 变成一个不参与反传的常数）
- 所以 `∂output / ∂z_e = ∂(z_e + 常数) / ∂z_e = 1`
- 这意味着 **Decoder 反传回来的 `∂L/∂z_q` 会原封不动传给 `z_e`**

用符号写是：

$$
\frac{\partial L}{\partial z_e} \;\xleftarrow{\text{STE}}\; \frac{\partial L}{\partial z_q}
$$

![fig4](/images/VQVAE-Principles-DeepDive/fig4.png)
*图 4：STE 的梯度流示意。黑色实线为前向数据流（经过 argmin 离散化），红色虚线为反向梯度流（"跳过" argmin，直接把梯度从 z_q 复制到 z_e）。*

### 3.3 Hand-calc 2 —— STE 正反向数值验证 🔢

用 §1 的数字：`z_e = [0.8, 1.5]`，最近邻 `e_1 = [1, 2]`，所以 `z_q = [1, 2]`。

假设 Decoder 反传回来的梯度是 `∂L/∂z_q = [0.3, -0.1]`（随便一个具体值，用来演示）。

**没有 STE 时**（argmin 是 0 梯度的硬截断）：
- `∂L/∂z_e = 0`
- Encoder 权重**完全收不到梯度**，没法学。

**有 STE 时**（`z_q = z_e + (z_q - z_e).detach()`）：
- `∂L/∂z_e = ∂L/∂z_q = [0.3, -0.1]` ✅
- Encoder 正常收到梯度，继续反传链式法则更新卷积权重。

**STE 的代价**：这个"复制"本质上是近似 —— 我们假装"量化没发生"，但实际量化误差 `z_q - z_e = [0.2, 0.5]` 是真实存在的。当误差很大时，STE 给的梯度方向其实不准。**这就是为什么我们需要下一章的 commitment loss——把量化误差压小，让 STE 的近似更靠谱**。

### 3.4 章末自测

<details>
<summary>Q1：为什么 STE 用 `.detach()` 而不是直接写 `z_q = z_e`？</summary>

因为前向必须走**离散量化后的 `z_q`**（它才是码本里那一行，和 Transformer 对应）。如果直接 `z_q = z_e`，前向跟"没量化"一样，码本就没意义了。`.detach()` 的巧妙在于：**数值上等于 `z_q`（前向对），梯度图上等于 `z_e`（反向能传）**。
</details>

<details>
<summary>Q2：如果 Decoder 给出的梯度是 `∂L/∂z_q = [1.0, 0.0]`，请写出经过 STE 后 `∂L/∂z_e` 的值。</summary>

`[1.0, 0.0]` —— STE 就是原样复制，不做任何数值改变。
</details>

## 第 4 章 三项损失的职责与 β 的选择 [L0 核心] ⭐

> §3 STE 解决了"梯度能传回 Encoder"。但还剩三个悬念：（a）**码本 `e_k` 自己怎么更新**？STE 只给了 Encoder 梯度，码本矩阵参数还是死的。（b）训练时为什么要加 **commitment loss**？不要它会怎样？（c）**β** 为什么原论文默认 0.25，而不是 1 或 0.1？这一章把三项损失一项项拆开，看清楚每项在给谁传梯度。

### 4.1 总损失长什么样：先给全图

VQ-VAE 原论文（van den Oord et al., NeurIPS 2017，[A]）的总损失：

$$
\mathcal{L}_{\text{VQ-VAE}} \;=\; \underbrace{\| x - \hat{x} \|^2}_{\text{重建 }L_{\text{recon}}}
\;+\; \underbrace{\| \mathrm{sg}[z_e] - e_k \|^2}_{\text{码本 }L_{\text{codebook}}}
\;+\; \beta \cdot \underbrace{\| z_e - \mathrm{sg}[e_k] \|^2}_{\text{承诺 }L_{\text{commit}}}
$$

每项给谁传梯度，一张表说清楚：

| 损失项 | 字面长得像什么 | `sg[·]` 锁住谁 | 梯度流向哪里 | 目的 |
|---|---|---|---|---|
| `L_recon` | `‖x − x̂‖²` | 无 | Decoder + Encoder（通过 STE） | 重建质量 |
| `L_codebook` | `‖sg[z_e] − e_k‖²` | 锁住 z_e | **只流向 `e_k`**（码本矩阵）| 让码字"追"当前 Encoder 输出 |
| `L_commit` | `‖z_e − sg[e_k]‖²` | 锁住 e_k | **只流向 `z_e`**（Encoder 权重）| 让 Encoder "承诺"靠近码字 |

![fig11](/images/VQVAE-Principles-DeepDive/fig11.png)
*图 11：三项损失在 50k 训练步内的典型走势。蓝线 Reconstruction 陡降（重建压力最大）；橙线 Codebook 缓降；绿线 Commitment 先短暂上升（Encoder 输出扩散时）再下降，最终和 Codebook 在同一量级趋稳。底部横幅给出总损失的权重组合 `L = L_recon + L_codebook + 0.25·L_commit`。*

### 4.2 `L_codebook` 和 `L_commit` 的语义差（关键对立）

这俩公式长得几乎一样，很多人第一次看都懵。按 Concept Teaching Pattern，先讲差异再讲公式：

> **`L_codebook` 和 `L_commit` 是一对方向相反的"拉扯力"——一股把码字拉向 Encoder 输出，一股把 Encoder 输出拉向码字**。它们在 `z_e` 和 `e_k` 之间形成一根弹簧，稳态时两者重合。

具体来说：
- **`L_codebook`** 的梯度只对 `e_k` 有效（z_e 被 sg 锁住）。它告诉码字：**"往 Encoder 输出的方向挪"**。
- **`L_commit`** 的梯度只对 `z_e` 有效（e_k 被 sg 锁住）。它告诉 Encoder：**"你的输出不要乱跑，承诺靠近已选中的码字"**。

![fig12](/images/VQVAE-Principles-DeepDive/fig12.png)
*图 12：两项损失形式对称、语义相反。左边 `L_codebook` 锁住 `z_e`（小锁图标），`e_k` 向 `z_e` 移动；右边 `L_commit` 锁住 `e_k`，`z_e` 向 `e_k` 移动。底部对比表直接列出三个关键差异：谁动、目标是谁、防止什么。*

![fig5](/images/VQVAE-Principles-DeepDive/fig5.png)
*图 5：三项损失在 2D 平面上的力矢量图。蓝色箭头 = `L_codebook` 把 e_k 拉向 z_e；红色箭头 = `L_commit` 把 z_e 拉向 e_k；绿色 = `L_recon` 通过 STE 影响 z_e。*

### 4.3 不要 `L_commit` 会怎样？—— 反证法思想实验

假设我们把 β 设为 0（去掉 commit loss），只留前两项。会发生什么？

1. Encoder 没有任何约束说"靠近码字"，它可以把 `z_e` 推到**任何地方**去最小化重建 loss；
2. 码字 `e_k` 一直在追 `z_e`，但 `z_e` 会飘得越来越远；
3. 训练变得极其不稳定，量化误差 `‖z_e − e_k‖` 爆炸，STE 的梯度近似质量崩溃；
4. 最终重建 loss 也学不下去——Encoder 和码本进入猫捉老鼠的恶性循环。

所以 **`L_commit` 的作用是"锚定"**：让 Encoder 知道"你不能随便飘，必须承诺向码字靠近"，这样 STE 的近似才有意义（§3.3 结尾的伏笔）。

### 4.4 Hand-calc 3 —— 三项损失的数值手算 🔢

续用前例：`z_e = [0.8, 1.5]`，`e_1 = [1, 2]`（选中），假设 Decoder 重建 `x̂` 使得 `L_recon = 2.50`（给定），β = 0.25。

计算 `L_codebook` 和 `L_commit`：

- `L_codebook = ‖sg[z_e] − e_1‖² = (0.8−1)² + (1.5−2)² = 0.04 + 0.25 = **0.29**`
- `L_commit = ‖z_e − sg[e_1]‖² = (0.8−1)² + (1.5−2)² = 0.04 + 0.25 = **0.29**`

注意两个数值**一样**（因为 `sg` 不改变数值，只影响梯度流向）。但意义完全不同：
- `0.29` 的梯度流向 `e_1` —— 把 `e_1` 从 `[1, 2]` 挪向 `[0.8, 1.5]`
- `0.29 × 0.25 = 0.0725` 是 `L_commit` 对总损失的贡献值；流向 `z_e` 的梯度是 `2β(z_e - e_k) = 2×0.25×([0.8,1.5]-[1,2]) = (-0.1, -0.25)`，把 `z_e` 从 `[0.8, 1.5]` 挪向 `[1, 2]`

**总损失** = `2.50 + 0.29 + 0.25 × 0.29 = 2.50 + 0.29 + 0.0725 = 2.8625`

### 4.5 β 为什么默认 0.25？

原论文做了 β ∈ [0.1, 2.0] 的消融，发现模型对 β 选择不敏感（性能几乎不变）。0.25 是作者给出的推荐值。直觉是：

- β 太小（<0.1）：Encoder 约束太弱，训练早期 `z_e` 乱飘；
- β 太大（>1.0）：Encoder 被过度绑定到码字上，灵活性下降，Decoder 拿到的 `z_q` 信息变少。
- **0.25** 是经验折中点。若你遇到严重的码本崩塌（§5），可以尝试调大 β 到 0.5 左右。

### 4.6 章末自测

<details>
<summary>Q1：`sg[z_e]` 和 `z_e` 在数值上相等吗？在梯度上呢？</summary>

**数值上完全相等**（sg 不改变前向结果）。**梯度上不等**：sg 把反传梯度截为 0，所以 `∂L/∂sg[z_e] = 0` 但 `∂L/∂z_e` 正常流。这正是 `L_codebook` 能"只更新 e_k 不碰 z_e"的机制。
</details>

<details>
<summary>Q2：若 `L_recon = 0.01`（已经学得很好）而 `L_commit = 0.50`（非常大），说明什么问题？</summary>

说明 Encoder 能把信息编码得让 Decoder 重建得很好，但它选的 `z_e` 离任何码字都远——可能码本容量不够，或者 β 太小没有把 Encoder 约束住。实践中会看到这种情况下码本利用率往往也很低（§5 码本崩塌的先兆）。
</details>

## 第 5 章 EMA 码本更新与码本崩塌 [L0 核心] ⭐

> 到 §4 为止，码本是通过 `L_codebook = ‖sg[z_e] − e_k‖²` 这一项用梯度下降更新的。但在实际训练动作 VQ-VAE 时你会看到两件怪事：第一，多数论文（T2M-GPT / MoMask）**扔掉了这一项**，改用一种叫 **EMA** 的更新方式；第二，训到中期你会发现 512 个码字里有几百个完全没被用过——这就是**码本崩塌**。本章讲清楚 EMA 为什么能替换 `L_codebook`，以及码本崩塌是怎么形成的。

### 5.1 梯度更新码本 vs EMA 更新码本：为什么要换？

**梯度更新路线**（§4 公式）：
- 用 `L_codebook` 的梯度更新 `e_k`：$e_k \leftarrow e_k - \eta \cdot 2(e_k - z_e)$
- 等价于"每步让 `e_k` 朝 `z_e` 走一小步"

**问题**：这个"一小步"的步长由学习率 η 决定，和 Encoder 的学习率耦合。实践中 η 不好调——Encoder 需要小学习率以稳定，但码本可能需要大学习率快速追上。一错就出问题。

**EMA 更新路线**（van den Oord 2017 附录 + VQ-VAE-2, Razavi 2019）：
- 不用梯度，直接统计"过去一段时间里被分配到 `e_k` 的所有 `z_e` 的平均值"
- 用这个**指数加权移动平均**（Exponential Moving Average）作为新的 `e_k`

两者的**差异**：EMA 不走 loss，不怕学习率耦合；它像在"维护每个码字的质心"，数值上更稳定。代价是需要额外存两个 buffer。

### 5.2 EMA 更新公式（先直觉，再公式）

**直觉**：每个码字 `e_k` 其实就是"最近一段时间里所有选中它的 `z_e` 的平均"。EMA 是用滑动平均来**增量**维护这个平均，不必存所有历史。

**公式**：对每个码字 k，维护两个辅助 buffer：

- **计数 `N_k`**：被分配到 k 的样本数（衰减累积）
- **累加 `m_k`**：被分配到 k 的样本值之和（衰减累积）

每步更新：

$$
N_k^{(t)} = \gamma \cdot N_k^{(t-1)} + (1-\gamma) \cdot n_k^{(t)}
$$

$$
m_k^{(t)} = \gamma \cdot m_k^{(t-1)} + (1-\gamma) \cdot \sum_{j \in S_k^{(t)}} z_e^{(j)}
$$

$$
e_k^{(t)} = \frac{m_k^{(t)}}{N_k^{(t)}}
$$

其中 $n_k^{(t)}$ 是这一步 batch 中被选到 k 的样本数，$S_k^{(t)}$ 是这些样本的集合，$\gamma$ 是衰减系数（通常 0.99，相当于 100 步的滑动平均）。

> 📌 **为什么 EMA 能去掉 `L_codebook`**：梯度更新里 `L_codebook` 本质上就是让 `e_k` 朝"被分到自己的所有 z_e 的平均"移动。EMA 直接给出这个平均值，就不用再靠梯度推了——相当于**用数值稳定的统计方法替代梯度下降**。此时总损失就简化为：
>
> $$ \mathcal{L}_{\text{EMA-VQ}} = L_{\text{recon}} + \beta \cdot L_{\text{commit}} $$
>
> ⚠️ **重要工程注意**：EMA 更新和 `L_codebook` 的梯度更新**不能同时作用于同一码本参数**。如果同时对 `e_k` 用 EMA 更新又让 `L_codebook` 的梯度流向 `e_k`，会导致码本参数被两股不同的力拉扯，训练极不稳定。T2M-GPT 和 MoMask 的实际实现都是**只用 EMA 更新码本，完全去掉 `L_codebook` 这一项**。

### 5.3 Hand-calc 4 —— EMA 一步更新 🔢

假设 `γ = 0.99`，初始 `N_1^{(t-1)} = 10`，`m_1^{(t-1)} = [10, 20]`（即 `e_1 = [1, 2]`）。

这一步 batch 里，码字 k=1 被 3 个样本选中，这 3 个样本的 `z_e` 分别是 `[0.8, 1.5]`、`[1.2, 2.1]`、`[0.9, 1.9]`。

- `n_1 = 3`
- `Σ z_e = [0.8+1.2+0.9, 1.5+2.1+1.9] = [2.9, 5.5]`

更新：

- `N_1^{(t)} = 0.99 × 10 + 0.01 × 3 = 9.9 + 0.03 = 9.93`
- `m_1^{(t)} = 0.99 × [10, 20] + 0.01 × [2.9, 5.5] = [9.9, 19.8] + [0.029, 0.055] = [9.929, 19.855]`
- `e_1^{(t)} = [9.929 / 9.93, 19.855 / 9.93] ≈ [1.0000, 1.9995]`

码字从 `[1, 2]` 几乎没动（因为 γ 大，3 个新样本影响有限），但方向正确：新样本均值 `[2.9/3, 5.5/3] ≈ [0.967, 1.833]` 略低于 `[1, 2]`，EMA 让 `e_1` 微微向下漂——正符合"码字追踪当前 Encoder 输出的质心"的直觉。

![fig6](/images/VQVAE-Principles-DeepDive/fig6.png)
*图 6：EMA 一步更新的数值演示。上排 bar chart 显示 N_k 和 m_k 的变化，下排显示 e_k 的前后对比。*

### 5.4 码本崩塌：马太效应的闭环

**定义**：训练过程中，大量码本向量从不被任何 `z_e` 选中，导致**有效码本远小于 K**。T2M-GPT 默认 K=512；在 K=1024 的实验中实测码本利用率只有 50–65%，近半数码字浪费（Zhang et al., CVPR 2023 §4.3）。

**形成机理**（马太效应）：

```
  初始分配不均（随机初始化 → 有些码字离 z_e 近，有些远）
       │
       ▼
  近的码字频繁被选 → 频繁收到 L_codebook 梯度 → 进一步靠近 z_e 分布
       │
       ▼
  远的码字永远不被选 → 永远收不到梯度 → 永远停在初始位置 → 越来越"被孤立"
       │
       ▼
  最终只有一小撮码字活跃，其他全部"死亡"
```

*图 7：码本崩塌的马太效应曲线。横轴是训练步数，纵轴是活码数量（蓝线）和 Perplexity（红线），可以看到活码数快速下滑到 50% 左右后稳定。*

![fig13](/images/VQVAE-Principles-DeepDive/fig13.png)

### 5.5 Perplexity：用一个数看懂码本健康度

**Perplexity（困惑度）** 来自信息论，对码字使用频率 $p_k$ 定义：

$$
\text{PPL} = \exp(H(p)) = \exp\left(-\sum_{k=1}^K p_k \log p_k\right)
$$

**为什么用 `exp(H)`？**——因为熵 $H$ 的单位是 nats（自然对数单位），`exp(H)` 把它转成"等效均匀分布下的有效类别数"。

- 如果 K=512 个码字使用频率**完全均匀**，则 $p_k = 1/512$，$H = \log 512$，**PPL = 512**。
- 如果只有 1 个码字被使用，$H = 0$，**PPL = 1**。
- 实践中 PPL / K > 0.7 算健康。

**对比刻度**：

| 场景 | PPL / K 比值 | 解读 |
|---|---|---|
| 理想均匀 | 1.0 | 码本满载 |
| 健康训练 | >0.7 | 多数码字都有使用 |
| 中度崩塌 | 0.3–0.5 | 半数码字死亡 |
| 严重崩塌 | <0.1 | 坍缩成少数几个簇 |

### 5.6 Hand-calc 5 —— Perplexity 手算 🔢

假设 K=4 个码字，统计出使用频率：
- 方案 A（均匀）：p = [0.25, 0.25, 0.25, 0.25] → `H = −4 × 0.25 × log(0.25) = log(4) ≈ 1.386`，`PPL = exp(1.386) = 4` ✅
- 方案 B（严重崩塌）：p = [0.97, 0.01, 0.01, 0.01] → `H ≈ −0.97 × log(0.97) − 3 × 0.01 × log(0.01) ≈ 0.030 + 0.138 ≈ 0.168`，`PPL ≈ exp(0.168) ≈ 1.18`

方案 B 的 PPL/K = 0.295，说明 4 个码字里实际只等效于 1.2 个在用。

![fig14](/images/VQVAE-Principles-DeepDive/fig14.png)
*图 14：放大到 K=512 的两种码本使用分布。左：健康码本，512 条概率柱几乎等高（$p_k ≈ 1/512$），PPL = 480 ≈ K；右：崩塌码本，前 30 条高耸其余全零，PPL = 28 ≪ K。中间公式框强调 `PPL = exp(H(p))` —— 困惑度就是"把熵的 nats 指数化得到的等效均匀类别数"。*

<summary>Q1：为什么用 EMA 更新码本时，`L_codebook` 这一项可以扔掉？</summary>

因为 EMA 本身就是一种"让码字追踪 Encoder 输出质心"的更新方式，数学上等价于 `L_codebook` 的无穷小步长梯度下降，但数值更稳定。`L_codebook` 的作用被 EMA buffer 更新取代了。保留它反而会和 EMA 冲突。

## 第 6 章 码本工程四件套 [L1 扩展]

> §5 讲了码本崩塌的根因。这一章给出**实际训练 VQ-VAE 必备的四个工程对策**，每个都是 T2M-GPT / MoMask 代码里的标配。L1 是因为 TA 不需要自己去推导它们，但读开源代码时必须看得懂这些补丁为什么存在。

![fig17](/images/VQVAE-Principles-DeepDive/fig17.png)
*图 17：码本工程四件套总览。左上 K-means 初始化（随机点→簇心覆盖数据分布）；右上死码重置（死码×被随机 z_e 替换复活）；左下 L2 归一化（向量被拉到单位球上，大小一致）；右下 Perplexity 监控曲线（PPL/K 实时崩塌预警）。*

### 6.1 对策一：K-means 初始化

**默认随机初始化的问题**：`e_k ~ N(0, 1)` 让码字散布在整个实数空间，但 Encoder 输出 `z_e` 可能只占一个局部区域（比如 `[0, 5]` 的方盒子）——一半码字从训练开始就注定没人用。

**K-means 初始化**：训练开始前，先跑一个 batch 的 Encoder，得到一堆 `z_e`，然后做 K-means 聚类，用 K 个簇心初始化码本。

效果（T2M-GPT 实测）：码本利用率从 50–60% 提升到 70–85%。

### 6.2 对策二：死码重置（Codebook Restart）

**检测**：定期（比如每 1000 步）统计每个码字的使用次数。连续多步使用次数为 0 的视为"死码"。

**重置**：把死码的位置替换成**当前 batch 里一个随机 `z_e`**（即"从 Encoder 当前输出里拉一个活的来替代死的"）。

```python
def reset_dead_codes(codebook, z_e, usage_count, threshold=1):
    dead_mask = usage_count < threshold
    if dead_mask.any():
        random_indices = torch.randint(0, z_e.shape[0], (dead_mask.sum(),))
        codebook.weight.data[dead_mask] = z_e[random_indices].detach()
        usage_count[dead_mask] = threshold
```

### 6.3 对策三：Codebook Factor / L2-Norm（Improved RVQGAN）

**问题**：码字向量的模长（magnitude）如果不受约束，长度大的码字天然更容易被选中（点积更大）。

**对策**：把码本向量强制归一化到单位球上：`e_k / ‖e_k‖`，距离计算用归一化后的向量。

这是 Kumar et al., NeurIPS 2023（"Improved RVQGAN"，[A]）的核心改动，也被 MoMask 采纳。

### 6.4 对策四：Perplexity 训练监控

把 `PPL / K` 作为 TensorBoard 上的第一项监控指标。一旦跌破 0.5 就预警，开始检查 β、学习率、重置频率。

### 6.5 本章自测

<details>
<summary>Q：如果你的训练曲线显示 L_recon 持续下降但 PPL/K 从 0.8 跌到 0.3，应该优先加哪个对策？</summary>

**死码重置**（对策二）。PPL 下降而 L_recon 下降说明模型在用更少的码字重建——典型的码本崩塌。死码重置是最直接的"把死码拉活"的手段。同时也要检查 K-means 是否用了。
</details>

## 第 7 章 VQ 家族：VQ-VAE-2 / RVQ / FSQ [L1 扩展]

> §1–§6 讲的是"单层 VQ-VAE"。但论文里你会看到 **VQ-VAE-2**（双层码本）、**RVQ**（残差量化，MoMask 用的）、**FSQ**（彻底换思路）。这一章把三种变体的**设计动机**讲清楚——什么时候用哪个，不用记公式细节。

### 7.1 VQ-VAE-2：空间上双层码本（Razavi et al., NeurIPS 2019, [A]）

**动机**：单层码本在图像上只能捕捉单一分辨率的信息。

**做法**：在 Encoder 里分出 Top / Bottom 两个分支：
- Bottom：高分辨率、细节（如边缘、纹理）
- Top：低分辨率、全局结构（如整体构图）

两个分支各配一本码本，Decoder 时联合使用。这是"**空间上的多尺度码本**"。

### 7.2 RVQ（Residual VQ）：深度上多层残差量化

RVQ 是 MoMask（CVPR 2024, [A]）的核心技术，源自 SoundStream（Zeghidour et al., IEEE/ACM TASLP 2021, [A]）。

**动机**：单层码本的量化误差 `z_q - z_e` 可能还是很大。那**把误差再量化一次**呢？

**做法**：串联多个码本 `Q_1, Q_2, ..., Q_L`（每本 K 行）：

```
z_e  ─►  Q_1  ─►  e_{k1}        剩余  r_1 = z_e − e_{k1}
         r_1  ─►  Q_2  ─►  e_{k2}    剩余  r_2 = r_1 − e_{k2}
         r_2  ─►  Q_3  ─►  e_{k3}    剩余  r_3 = r_2 − e_{k3}
         ...
         最终：z_q = e_{k1} + e_{k2} + ... + e_{kL}
```

每一级专门学"上一级残差"。MoMask 用 6 级 RVQ，每级 K=512，**理论等效容量上限** = 512^6 ≈ 1.8×10¹⁶（假设每级码本利用率均为 100% 且各级独立；实际等效容量通常远低于此理论上限，取决于各级码本的实际利用率）。参数只有 6 × 512 × D。

### 7.3 Hand-calc 6 —— RVQ 两级残差示意 🔢

设 `z_e = [3.0, 4.0]`。

**第 1 级**，码本 Q_1 = { [2, 3], [3, 4], [1, 5] }：
- 最近邻：`e_{k1} = [3, 4]`
- 残差 `r_1 = [3.0, 4.0] − [3, 4] = [0.0, 0.0]` ← 这种情况说明 Q_1 就够了

设另一个例子：`z_e = [2.8, 3.5]`。
- 第 1 级：最近的还是 `[3, 4]`，残差 `r_1 = [−0.2, −0.5]`。
- **第 2 级**，小码本 Q_2 = { [0.1, 0.1], [−0.2, −0.5], [0, 0] }：
  - 最近邻：`e_{k2} = [−0.2, −0.5]`
- 最终 `z_q = e_{k1} + e_{k2} = [3, 4] + [−0.2, −0.5] = [2.8, 3.5]` ← 完全重建

这就是 RVQ 的神奇：**多级组合 = 指数级容量扩展**。

> 📌 **教学简化说明**：上面的 Q_2 码本里恰好有一个码字 `[-0.2, -0.5]` 精确匹配了残差，实现了完全重建（z_q = z_e）。在实际训练中，残差空间的码本通常无法精确匹配每一个残差向量，仍会有量化误差。此处为了演示 RVQ 的原理而做了理想化简化。

### 7.4 FSQ：根本不要码本（Mentzer et al., ICLR 2024, [A]）

**动机**：码本 + commitment + EMA + 崩塌处理 = 一堆工程补丁。能不能简化？

**FSQ 的思路**：**不要码本，每个维度独立 round 到 L 个 bin**。

```
z_e 每一维 ∈ ℝ
  ↓  tanh 压缩到 [−1, 1]
  ↓  均匀离散化成 L 个 bin（例如 L=5：{−1, −0.5, 0, 0.5, 1}）
z_q 每一维 ∈ {L 个离散值}
```

**总等效码本容量** = L^D（每维 L 种，D 维独立）。比如 L=5, D=6：5^6 = 15625 个等效码字。

**神奇优势**：
- 没有码本矩阵 → 没有 codebook collapse
- 不需要 commitment loss / EMA
- 训练代码简化到几行

**代价**：每个"等效码字"不可学习，只能是均匀栅格点。表达力有上限。

### 7.5 四种量化方案对比表

| 方案 | 参数量 | 等效容量 | 训练复杂度 | 崩塌风险 | 代表应用 |
|---|---|---|---|---|---|
| 单层 VQ-VAE | K×D | K | 中 | 高 | T2M-GPT, 老版图像 |
| VQ-VAE-2 | 2×K×D | K²（分层）| 中 | 中 | DALL-E 1 |
| RVQ（L 层）| L×K×D | K^L | 高（级联训）| 低（每级分散）| MoMask, SoundStream |
| FSQ | 0 | L^D | **低** | **零** | 最新图像/音频 |

![fig8](/images/VQVAE-Principles-DeepDive/fig8.png)
*图 8：VQ 家族四方案植物标本式对比。单株蒲公英=Plain VQ（单簇码本）；双层伞菇=VQ-VAE-2（空间分层两套码本）；递减铃花藤=RVQ（深度残差多层码本，逐层细化）；透明晶格=FSQ（无码本，隐式网格量化）。四种内部组织方式不同，分别对应表格里的参数量/容量/典型用途三列。*

### 7.6 本章自测

<summary>Q：如果你要做一个新的动作量化器，在下列哪个场景下会选 FSQ？（A）有大规模数据，追求极致表达力；（B）小规模实验，想快速上手、不想调崩塌。</summary>

**B**。FSQ 零码本、零崩塌，工程负担最小。但表达力有硬上限，大规模追求质量时 RVQ 还是更灵活。

### 8.3 为什么 AMP 不用 VQ-VAE？

这是 4/15 留的坑，今天可以回答。**AMP（SIGGRAPH 2021）用的是 RL + GAN + 连续策略网络**，它不需要把动作 token 化给 Transformer。AMP 的输入是当前物理状态（关节角速度、接触点等），输出是下一帧的连续动作（关节扭矩），**全程连续空间，没有 token 化的需求**。

而 T2M-GPT / MoMask 是**文字到动作**的生成，输入天然是文字 token，输出也要是动作 token 才能用 Transformer——这是 VQ-VAE 的主战场。

一句话记住：**离散 token 是为自回归语言模型服务的；物理模拟 RL 不需要它**。

## 第 9 章 下节课预告与学习路线 [L0 核心]

### 9.1 离散路线 vs 扩散路线：一个核心取舍

你现在学完了两条生成路线的"入口概念"：

- **离散路线（今天 + 4/15）**：VAE → VQ-VAE → T2M-GPT / MoMask（文字 → 动作 token）
- **连续扩散路线（4/23）**：DDPM → MDM（条件噪声 → 动作序列）

两条路线的**语义差**：
- 离散路线像"说话"—— 一个 token 一个 token 往外吐，天然支持 LLM 预训练、超大文本模型、多模态统一。代价是码本设计麻烦（崩塌、失真）。
- 连续扩散路线像"打磨"—— 从噪声逐步去噪到干净动作，一步到位高质量。代价是采样慢、难加文本控制（需 CFG）。

**TA 眼里的选择**：
- 要做大规模文字驱动动作库（动画 demo 生成、镜头预演）→ 离散路线（T2M-GPT, MoMask）
- 要做物理一致、柔顺动作（游戏内过渡、角色驱动）→ 扩散路线（MDM, MLD）
- 要做 **Physics-RL**（DeepMimic, AMP）→ **都不用**，RL 里动作不需要 token 化，回到连续策略网络

## 附录 A：Gumbel-Softmax 对比 STE [L2 研究向]

> 只在你要读 VQ-Diffusion / MaskGIT / 某些离散 VAE 论文时需要。TA 日常工作用不到。

**Gumbel-Softmax**（Jang et al., ICLR 2017, [A]）给 argmax 一个**可导的"软化"近似**：

$$
y_k = \frac{\exp\left((\log \pi_k + g_k) / \tau\right)}{\sum_j \exp\left((\log \pi_j + g_j) / \tau\right)}, \quad g_k \sim \text{Gumbel}(0, 1)
$$

- 温度 τ 高（如 1.0）：输出接近均匀分布（软）
- 温度 τ 低（如 0.01）：输出接近 one-hot（硬）
- 训练时温度从高退火到低

**STE vs Gumbel-Softmax**：
- STE：粗暴但有效，几乎所有 VQ-VAE 实现都用它
- Gumbel-Softmax：理论更"干净"，有严格的 reparameterization，但多了一个温度超参要调

在运动 VQ-VAE 领域，**STE 是绝对主流**。只有在某些极端不稳定的数值场景下才会考虑 Gumbel-Softmax。

| 中文 | English | 一句话 |
|---|---|---|
| 码本 | Codebook | 一张 K×D 的可学习矩阵，每行是一个候选码字 |
| 码字 / 嵌入向量 | Codebook Entry / Embedding Vector | 码本里的一行 `e_k` |
| 量化 | Quantization | 把连续 `z_e` 替换成最近码字 `e_k` 的过程 |

| 困惑度 | Perplexity | `exp(H(p_k))`，码本使用均匀度指标 |
| 死码重置 | Codebook Restart / Reset | 把长期不被选的码字替换成当前 batch 的 `z_e` |
| 残差量化 | Residual Vector Quantization (RVQ) | 多级码本串联，每级量化上一级残差 |
| 有限标量量化 | Finite Scalar Quantization (FSQ) | 不用码本，每维独立 round 到 L 个 bin |
| Token 化 | Tokenization | 把连续 / 结构化数据切成离散单元并编号 |
| 下采样率 | Downsampling Rate | Encoder 把输入帧数压缩的倍数（T2M-GPT 4×） |
