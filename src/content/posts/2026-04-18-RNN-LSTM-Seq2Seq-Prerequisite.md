# 第1章：RNN —— 为什么需要"循环"神经网络？

![fig1](/images/RNN-LSTM-Seq2Seq-Prerequisite/fig1.png)

## 1.1 前置：序列数据到底特殊在哪

**序列数据（Sequential Data / 序列数据）** 指的是**元素之间有先后顺序，且顺序很重要**的数据。

在你的工作里，最典型的序列数据就是**动画**：

```
一段 2 秒的走路动画（60 帧）：

时间轴 ────────────────────────→
帧:    0   1   2   3   ...   58   59

每一帧是一个 66 维向量（22 关节 × 3 维旋转）

特点 1：顺序重要
  → 帧 0 → 帧 1 → 帧 2 是走路
  → 帧 2 → 帧 1 → 帧 0 是倒放

特点 2：前后依赖
  → 第 30 帧的姿势，依赖前 29 帧怎么走过来的
  → 孤立看一帧，你不知道这是起步还是收尾

特点 3：长度可变
  → 一段走路可能 60 帧，也可能 300 帧
  → 网络要能处理不同长度的输入
```

其他的序列数据例子：
- **文本**："我爱猫" —— 词的顺序决定语义
- **音频**：一段声波采样 —— 时间顺序
- **股价**：每天的收盘价 —— 时间序列
- **视频**：一帧一帧的图像 —— 时间序列

## 1.2 为什么全连接层 / CNN 处理序列不够用？

你上一课学过的两种网络，它们处理序列时各有硬伤。

### 问题 1：全连接层（Fully Connected / 全连接层）

```
如果把整段动画一次性喂给全连接层：

输入：60 帧 × 66 维 = 3960 维向量
    ↓
  [全连接层 W1：3960 × 1024]       ← 400万参数
    ↓
  [全连接层 W2：1024 × 512]
    ↓
  ... 输出

致命问题：
  1. 长度固定！训练时 60 帧，推理时来一段 120 帧就炸了
  2. 参数量爆炸（上面的第一层就 400 万）
  3. 不利用"时间局部性"——第 1 帧和第 59 帧连接的权重
     和第 1 帧与第 2 帧连接的权重是独立的，模型要从头学
     "相邻帧更相关"这个常识
```

### 问题 2：CNN（卷积神经网络）

CNN 可以处理可变长度的序列，而且**时间局部性**天然编码（卷积核就是看相邻几帧）。但它有另一个问题：

```
kernel_size=3 的 Conv1D：
  帧: 0  1  2  3  4  ... 58 59
     [■  ■  ■]  ← 一次看 3 帧

要让网络看到 59 帧前的信息，需要 29 层 Conv1D（累积感受野）
  → 模型又深又复杂

TCN 用膨胀卷积能改善，但本质上还是"从局部到全局"的堆积
  → 感受野有硬上限
  → 无法灵活地捕捉"跨越整段序列"的长程依赖
```

### 我们真正需要什么？

```
理想的序列模型应该有：
  ✅ 处理可变长度
  ✅ 参数量不随序列长度增长
  ✅ 能记住"历史"，并用历史帮助处理当前
  ✅ 模型结构简洁（不是无脑堆深度）
```

**RNN（Recurrent Neural Network / 循环神经网络）** 就是为此设计的。

## 1.3 RNN 的核心直觉：隐藏状态 = 笔记本

### 生活类比：看剧记笔记

想象你在看一部 10 集的悬疑剧，每集 1 小时。你有两种看法：

**看法 A（全连接层）**：一次性把 10 集灌进大脑，同时理解。
→ 现实：不可能，大脑装不下。

**看法 B（RNN）**：一集一集看，边看边在**小本子**上记重点。每集开始前翻一下本子，结合之前的剧情来理解本集。
→ 现实：可行，而且高效。

**RNN 就是"看法 B"的数学实现**：

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│  隐藏状态 h（Hidden State / 隐藏状态）                 │
│    = 你的"小本子"                                     │
│    = RNN 对历史信息的压缩总结                          │
│                                                     │
│  处理第 t 步时：                                      │
│    新笔记 h_t = 更新(旧笔记 h_{t-1}, 新输入 x_t)       │
│                                                     │
│  注意！                                              │
│  - 小本子的"大小"是固定的（比如 256 维向量）             │
│  - 每一步都用**同一套规则**更新笔记（参数共享）           │
│  - 理论上，本子能记住无限长的历史                        │
│    （实际会遇到梯度消失，第 2 章会讲）                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 图示

```
RNN 展开图（Unrolled View）：

  h_0     h_1     h_2     h_3           h_t
   │       │       │       │             │
   ↓       ↓       ↓       ↓             ↓
  [RNN]─→[RNN]─→[RNN]─→[RNN]─→ ... ─→[RNN]
   ↑       ↑       ↑       ↑             ↑
   │       │       │       │             │
  x_0     x_1     x_2     x_3           x_t

  每一个 [RNN] 方块的内部参数完全相同（共享）
  每一步的 h_t 被传递给下一步
```

**参数共享（Weight Sharing / 权值共享）** 是 RNN 的灵魂：
- 不管序列多长，都用同一个 RNN 单元处理
- 参数量和序列长度无关（和 CNN 类似）
- 训练时"看过"各种长度的序列，泛化能力强

## 1.4 RNN 的数学（公式 + 手算）

### 标准 RNN 公式（Elman RNN, 1990）

```
h_t = tanh(W_xh · x_t + W_hh · h_{t-1} + b_h)

y_t = W_hy · h_t + b_y    （可选：如果每一步都要输出）
```

**符号含义**：

| 符号 | 维度 | 含义 | 生活类比 |
|---|---|---|---|
| `x_t` | d_x | 第 t 步的输入 | "本集剧情" |
| `h_{t-1}` | d_h | 上一步的隐藏状态 | "旧笔记" |
| `h_t` | d_h | 当前步的隐藏状态 | "新笔记" |
| `y_t` | d_y | 当前步的输出（可选）| "本集观后感" |
| `W_xh` | d_x × d_h | 输入到隐藏的权重 | "如何读取新剧情" |
| `W_hh` | d_h × d_h | 隐藏到隐藏的权重 | "如何更新笔记" |
| `W_hy` | d_h × d_y | 隐藏到输出的权重 | "如何写观后感" |
| `b_h`, `b_y` | 偏置 | 偏移项 | 常数 |
| `tanh` | — | 激活函数（-1 到 1）| 让笔记值不会无限增长 |

### 为什么用 tanh 而不是 ReLU？

```
RNN 的 h_t 会被反复乘 W_hh（第 2 章会详细讲）
  → 如果用 ReLU：值可能无限增长（爆炸）
  → 如果用 tanh：值被限制在 -1 到 1 之间（稳定）

所以经典 RNN 几乎都用 tanh。
```

### 手算示例：3 步的单层 RNN

假设我们在做一个**股价预测**：输入最近 3 天的股价涨跌率，预测第 4 天。

```
输入（3 天股价涨跌率，每天 1 个数）：
  x_1 = 0.5     ← 第1天涨 5%
  x_2 = -0.3    ← 第2天跌 3%
  x_3 = 0.8     ← 第3天涨 8%

简化参数（实际由训练得到，这里手工设定）：
  W_xh = 0.6    ← 输入 1 维，隐藏 1 维
  W_hh = 0.4
  b_h  = 0.1
  h_0  = 0      ← 初始隐藏状态通常设为 0

激活函数：tanh
```

**第 1 步**（t=1）：

```
h_1 = tanh(W_xh · x_1 + W_hh · h_0 + b_h)
    = tanh(0.6 × 0.5 + 0.4 × 0 + 0.1)
    = tanh(0.30 + 0.00 + 0.10)
    = tanh(0.40)
    ≈ 0.3799        ← "看完第1天"的笔记
```

（用直觉记：tanh(0.4) ≈ 0.38，tanh 在 0 附近几乎是线性，值越大越接近 1）

**第 2 步**（t=2）：

```
h_2 = tanh(W_xh · x_2 + W_hh · h_1 + b_h)
    = tanh(0.6 × (-0.3) + 0.4 × 0.3799 + 0.1)
    = tanh(-0.18 + 0.152 + 0.10)
    = tanh(0.072)
    ≈ 0.0718        ← "结合了第1天和第2天"的笔记
```

**第 3 步**（t=3）：

```
h_3 = tanh(W_xh · x_3 + W_hh · h_2 + b_h)
    = tanh(0.6 × 0.8 + 0.4 × 0.0718 + 0.1)
    = tanh(0.48 + 0.0287 + 0.10)
    = tanh(0.6087)
    ≈ 0.5431        ← 结合了全部 3 天的历史笔记
```

**关键观察**：

```
每一步的 h_t 都"记得"之前所有步的信息，
因为它通过 W_hh · h_{t-1} 这一项被不断更新。

你可以把 h_t 看成整段历史的"压缩摘要"。
```

### 和全连接层/CNN 的根本区别

```
全连接层：
  y = f(W · x + b)            ← 一次性吃整个输入
  
CNN (Conv1D)：
  y_t = f(W · [x_{t-k}...x_{t+k}] + b)    ← 看局部窗口，窗口内的相对位置固定

RNN：
  h_t = f(W_xh · x_t + W_hh · h_{t-1} + b)   ← 吃当前输入 + 前一步的笔记
  ↑        ↑              ↑
  新笔记   当前输入        上一步的笔记（= 所有历史的摘要）
```

## 1.5 RNN 的三种用法

根据输入和输出的关系，RNN 可以用于三种模式：

```
模式 1：many-to-one（多对一）
───────────────────────────────
  用途：序列分类
  输入：一整段序列
  输出：单个结果

  例子：情感分析（一段话 → 正面/负面）
        动作识别（一段动作 → "跑步"）
  
    x_0 → x_1 → x_2 → x_3
     │     │     │     │
    h_0 → h_1 → h_2 → h_3 ──→ y ("走路")
                       ↑
                   只在最后一步输出


模式 2：one-to-many（一对多）
───────────────────────────────
  用途：序列生成（从种子开始）
  输入：单个开头
  输出：一整段序列

  例子：图像字幕（一张图 → 一句话）
        动画续写（一个关键帧 → 一段动作）
  
    x
    │
    h_0 → h_1 → h_2 → h_3
           │     │     │
          y_1   y_2   y_3


模式 3：many-to-many（多对多）
───────────────────────────────
  用途：每步都要输出
  输入：整段序列
  输出：等长的输出序列

  例子：词性标注（每个词标词性）
        运动降噪（每帧输入噪声版 → 输出干净版）
  
    x_0 → x_1 → x_2 → x_3
     │     │     │     │
    h_0 → h_1 → h_2 → h_3
     │     │     │     │
    y_0   y_1   y_2   y_3

第 4 种模式 many-to-many（不等长），就是 Seq2Seq，第 4 章讲。
```

# 第2章：RNN 的致命缺陷 —— 梯度消失

![fig2](/images/RNN-LSTM-Seq2Seq-Prerequisite/fig2.png)

> 这一章要回答：**RNN 理论上能记无限长历史，为什么实际只能记 10-20 步？**

## 2.1 直觉：为什么序列越长越"记不住"？

### 传话游戏的类比

```
10 个人排成一排玩传话游戏：

原话（位置 0）：
  "明天下午三点去图书馆还书"
       │
       │ 传给位置 1
       ↓
  "明天下午三点去图书馆"  ← 丢了"还书"
       │
       │ 传给位置 2
       ↓
  "明天三点去图书馆"        ← 又丢了"下午"
       │
      ...
       ↓
位置 9 听到：
  "明天去某地"              ← 几乎什么都没剩
```

**每一次传递，都会有轻微的信息损失。传递次数越多，累积损失越大。**

RNN 的隐藏状态传递也一样：

```
h_0 → h_1 → h_2 → h_3 → ... → h_100

每一步都是：
  h_t = tanh(W_xh · x_t + W_hh · h_{t-1} + b_h)

h_{t-1} 的信息要穿过 tanh 和 W_hh 的乘法，
每穿过一次都会"衰减一点"。

到第 100 步时，h_0 的原始信息已经几乎完全丢失。
```

## 2.2 反向传播穿过时间（BPTT）

要真正理解梯度消失，需要理解 RNN 是怎么训练的。

**BPTT（Back-Propagation Through Time / 通过时间反向传播）** 是 RNN 训练算法。

### 回顾：普通神经网络的反向传播

```
普通前馈网络：
  x → [Layer1] → [Layer2] → [Layer3] → y → Loss

反向传播时，梯度从 Loss 逐层传回：
  ∂Loss/∂W3 → ∂Loss/∂W2 → ∂Loss/∂W1
```

（如果你还记得 CNN 课讲过的 ResNet —— "每层梯度连乘导致消失"，一会你会看到 RNN 是完全一样的问题）

### RNN 的反向传播

```
RNN 展开后，每一个时间步都是"一层"：

  x_0     x_1     x_2           x_T
   │       │       │             │
  h_0 → h_1 → h_2 → ... → h_T → y → Loss

要更新参数 W_hh，需要计算：
  ∂Loss / ∂W_hh

梯度要从 Loss 一路反传到最早的时间步：
  Loss → h_T → h_{T-1} → h_{T-2} → ... → h_1 → h_0
```

**关键差异**：普通网络的梯度穿过"不同层"；RNN 的梯度穿过"同一层的不同时间步"。每一次穿越都要乘 **同一个 W_hh**。

## 2.3 梯度消失的数学证明

我不给你推严格的公式，只给你**能闭眼复述**的直觉证明。

### 链式法则的连乘

```
从 h_T 回传到 h_0 的梯度，大约是：

∂h_T / ∂h_0 ≈ (W_hh)^T · tanh'(·) · (W_hh)^T · tanh'(·) · ... · (W_hh)^T · tanh'(·)
                                                           ↑
                                                    共 T 个这样的乘积
```

化简一下，把所有项用它们的"典型值"代替：

```
总梯度 ≈ (W_hh)^T × (tanh')^T

其中：
  W_hh 是矩阵，这里用它的"最大特征值" λ 代表
  tanh' 在 0 附近 ≈ 1，但饱和区 ≈ 0（平均值大约 0.25）

所以：
  总梯度 ≈ λ^T × 0.25^T
```

### 两种灾难

**灾难 A：梯度消失（Vanishing Gradient / 梯度消失）**

```
如果 λ < 1（W_hh 的权重较小）：
  T = 10:   0.9^10 × 0.25^10 ≈ 3.3e-7       ← 梯度几乎消失
  T = 50:   0.9^50 × 0.25^50 ≈ 2.6e-33      ← 完全消失
  T = 100:  梯度 ≈ 0

后果：
  训练时，第 100 步的 Loss 无法把有效梯度传回第 1 步
  → 模型学不到"长距离依赖"
  → 它能学到"最近的 5-10 步"怎么影响输出，
    但学不到"100 步前的信息"怎么影响输出
```

**灾难 B：梯度爆炸（Exploding Gradient / 梯度爆炸）**

```
如果 λ > 1（W_hh 的权重较大）：
  T = 10:   1.5^10 ≈ 58        ← 还好
  T = 50:   1.5^50 ≈ 6.4e8     ← 爆炸
  T = 100:  1.5^100 ≈ 4e17     ← 数值溢出

后果：
  参数更新时变化太大，训练直接崩溃（Loss 变成 NaN）
  
解法：梯度裁剪（Gradient Clipping）—— 如果梯度范数超过阈值就缩放
  → 工程技巧，能缓解但治标不治本
```

### 和 ResNet 对比

```
还记得 CNN 课的 ResNet 吗？

ResNet 解法：h_out = h_in + f(h_in)     ← 加号！
  反向传播: ∂h_out/∂h_in = 1 + ∂f/∂h_in    
  1 这一项保底，梯度不会变 0

标准 RNN：h_t = tanh(W · h_{t-1} + ...)    ← 没有加号
  反向传播: ∂h_t/∂h_{t-1} = W · tanh'
  连续相乘，必然走向 0 或 ∞

→ 所以 LSTM 的设计思路，正是"引入一条加法路径"
→ 你在 Transformer 文档里看到的残差连接，也是同一思想的延伸
```

**这是一个非常重要的联系**：你在 CNN 课学的残差连接，和接下来要学的 LSTM 细胞状态，以及 Transformer 的残差连接，**都是同一个数学思想**：**用加法路径保护梯度不消失**。

## 2.4 为什么 RNN 最多能记 10-20 步

实验结论（Bengio et al., 1994 就指出了这个问题）：

```
标准 RNN 的"有效记忆长度"：
  简单任务：能记 5-10 步
  复杂任务：能记 10-20 步
  超过 20 步：基本失效

对动画的影响：
  30fps 的动画，20 帧 ≈ 0.67 秒
  → 只能记住不到 1 秒的历史
  → 无法理解"跨越整个走路周期"的模式
  → 需要 LSTM / Transformer 来做更长范围的建模
```

这就是为什么**标准 RNN 在现代几乎不再单独使用**——它的缺陷太明显了。

# 第3章：LSTM —— 用"门"解决长期记忆

![fig3](/images/RNN-LSTM-Seq2Seq-Prerequisite/fig3.png)
![fig4](/images/RNN-LSTM-Seq2Seq-Prerequisite/fig4.png)

## 3.1 直觉：流水账 vs 带索引的笔记本

**LSTM（Long Short-Term Memory / 长短期记忆网络）** 由 Hochreiter 和 Schmidhuber 在 1997 年提出。

### 为什么叫"长短期记忆"？

```
RNN: 只有一种"短期记忆"
  h_t 既要记最新的信息，又要记很久以前的信息
  短期记忆把长期记忆"盖掉"了
  → 结果：长期记忆消失

LSTM: 把记忆分成两条线
  h_t    ← 短期记忆（类似 RNN 的隐藏状态）
  C_t    ← 长期记忆（新增！叫"细胞状态"）
  
  + 三个"门"（Gate）决定什么信息进、出、留
```

### 生活类比：笔记本带索引

```
RNN 的笔记本：
  只有一页，每次看新剧情就把整页重写
  → 旧的重要剧情迟早被覆盖

LSTM 的笔记本：
  第 1 页：短期速记板（h_t）—— 每集都重写
  第 2 页：长期档案（C_t）—— 只有重要的才写进去

  三个门决定操作：
    遗忘门 f：要不要擦掉长期档案的某些内容？
    输入门 i：要不要把新信息写进长期档案？
    输出门 o：从长期档案里拿什么给短期速记板？
```

## 3.2 LSTM 的结构图

```
         ┌─────────────────────────────────────┐
         │        LSTM 单元内部                 │
         │                                     │
   C_{t-1}── × ──────────── + ─────────── C_t  │  长期记忆
         │   ↑              ↑                  │
         │ 遗忘门 f        输入门 i              │
         │   │              │                  │
         │   │              × ← tanh(候选 C̃)    │
         │   │                                 │
         │ ┌─┴────┬────┬────┐                  │
         │ │      │    │    │                  │
         │ σ      σ    σ    tanh               │
         │ │      │    │    │                  │
         │ └─┬────┴────┴────┘                  │
         │   │                    输出门 o      │
         │   │                    │            │
   h_{t-1}───┤          C_t ──→ tanh           │
   x_t ──────┤                    │            │
             │                    × ────── h_t │  短期记忆
             │                                 │
             └─────────────────────────────────┘

看不懂没关系，下面用公式一步步拆开讲。
```

## 3.3 三个门的公式

每个门都是一个"0 到 1 的权重"，决定信息通过多少。

### 遗忘门（Forget Gate）：要不要忘掉旧记忆？

```
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)

含义：对长期记忆 C_{t-1} 的每个维度，决定保留多少
  f_t 接近 1 → 完全保留
  f_t 接近 0 → 完全遗忘
  f_t = 0.5 → 保留一半

σ（sigmoid）把数值压到 0~1 之间，正好适合做"门"
```

### 输入门（Input Gate）：要不要接受新信息？

```
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)        ← 决定要写入多少新内容
C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)    ← 候选的新内容本身

含义：
  i_t 是"开关"（0~1）
  C̃_t 是"内容"（-1~1）
  两者相乘，就是"实际要写入的新内容"
```

### 细胞状态更新：长期记忆的变化

```
C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t
      ────────────   ───────────
      忘掉一部分       写入一部分
      
⊙ 表示逐元素相乘（Element-wise / 对应位相乘）
```

**这就是解决梯度消失的关键**：

```
C_t = f_t · C_{t-1} + ...    ← 加号！
  ↑                    ↑
与 ResNet、Transformer 的残差连接同理

当 f_t 接近 1 时：
  C_t ≈ C_{t-1} + (新内容)
  ∂C_t / ∂C_{t-1} ≈ 1
  → 梯度可以几乎无衰减地穿过时间！
```

### 输出门（Output Gate）：从长期记忆里读多少？

```
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)        ← 输出开关
h_t = o_t ⊙ tanh(C_t)                      ← 短期记忆输出

含义：
  把长期记忆 C_t 过一遍 tanh（压到 -1~1），
  再用输出门 o_t 选择性输出到 h_t
```

### LSTM 完整公式一览

```
┌───────────────────────────────────────────────────────────┐
│  LSTM 的 4 个关键计算                                      │
├───────────────────────────────────────────────────────────┤
│                                                           │
│  遗忘门:  f_t = σ(W_f · [h_{t-1}, x_t] + b_f)            │
│  输入门:  i_t = σ(W_i · [h_{t-1}, x_t] + b_i)            │
│  候选态:  C̃_t = tanh(W_C · [h_{t-1}, x_t] + b_C)        │
│  输出门:  o_t = σ(W_o · [h_{t-1}, x_t] + b_o)            │
│                                                           │
│  细胞更新: C_t = f_t ⊙ C_{t-1} + i_t ⊙ C̃_t              │
│  隐藏更新: h_t = o_t ⊙ tanh(C_t)                         │
│                                                           │
└───────────────────────────────────────────────────────────┘

参数量：4 套权重矩阵（W_f, W_i, W_C, W_o）
     大约是标准 RNN 的 4 倍
```

**记忆口诀**：
- **f** for **F**orget（遗忘）
- **i** for **I**nput（输入）
- **o** for **O**utput（输出）
- **C̃** 是候选内容（C 的波浪号）
- 三个 σ（门）+ 一个 tanh（候选内容）

## 3.4 LSTM 手算示例（完整版，所有数字过一遍）

我们用一个最小化的 LSTM，把所有公式手算一遍。

### 场景设定

模拟一个**游戏中判断"是否该攻击"**的 LSTM：

```
输入（每帧 2 个特征）：
  x_1 = [1.0, 0.5]   ← 帧1：敌人距离近，血量中等
  x_2 = [0.8, 0.2]   ← 帧2：敌人还在近，血量低
  x_3 = [0.3, 0.8]   ← 帧3：敌人远了，血量恢复

为了手算可行，所有维度都设为 1（实际中一般是几百）
所以 h, C, 所有门 都是 1 维标量

简化参数：
  W_f = [0.5, 0.2]    W_i = [0.4, 0.3]   
  W_C = [0.3, 0.5]    W_o = [0.6, 0.1]
  
  U_f = 0.1           U_i = 0.2
  U_C = 0.15          U_o = 0.25
  
  所有 b = 0.1

  [注：完整写法是 W · [h,x] + b，等价于 W_h·h + W_x·x + b
   这里把 W_h 拆出来单独命名为 U，更直观]

初始：h_0 = 0, C_0 = 0
```

### 第 1 步（t=1, x_1 = [1.0, 0.5]）

**遗忘门 f_1**：

```
f_1 = σ(W_f · x_1 + U_f · h_0 + b_f)
    = σ(0.5×1.0 + 0.2×0.5 + 0.1×0 + 0.1)
    = σ(0.5 + 0.1 + 0 + 0.1)
    = σ(0.7)

sigmoid 速查：
  σ(0) = 0.5
  σ(1) ≈ 0.73
  σ(2) ≈ 0.88
  σ(-1) ≈ 0.27
  σ(0.7) ≈ 0.668
  
f_1 ≈ 0.668    ← 要保留 66.8% 的旧记忆（虽然旧记忆是 0，但门的值记住）
```

**输入门 i_1**：

```
i_1 = σ(W_i · x_1 + U_i · h_0 + b_i)
    = σ(0.4×1.0 + 0.3×0.5 + 0.2×0 + 0.1)
    = σ(0.4 + 0.15 + 0 + 0.1)
    = σ(0.65)
    ≈ 0.657    ← 要接受 65.7% 的新信息
```

**候选细胞内容 C̃_1**：

```
C̃_1 = tanh(W_C · x_1 + U_C · h_0 + b_C)
    = tanh(0.3×1.0 + 0.5×0.5 + 0.15×0 + 0.1)
    = tanh(0.3 + 0.25 + 0 + 0.1)
    = tanh(0.65)

tanh 速查：
  tanh(0) = 0
  tanh(0.5) ≈ 0.462
  tanh(1) ≈ 0.762
  tanh(0.65) ≈ 0.572

C̃_1 ≈ 0.572    ← 候选的新内容
```

**输出门 o_1**：

```
o_1 = σ(W_o · x_1 + U_o · h_0 + b_o)
    = σ(0.6×1.0 + 0.1×0.5 + 0.25×0 + 0.1)
    = σ(0.6 + 0.05 + 0 + 0.1)
    = σ(0.75)
    ≈ 0.679    ← 要输出 67.9% 的细胞内容
```

**更新 C_1**：

```
C_1 = f_1 × C_0 + i_1 × C̃_1
    = 0.668 × 0 + 0.657 × 0.572
    = 0 + 0.376
    = 0.376       ← 长期记忆（当前状态）
```

**更新 h_1**：

```
h_1 = o_1 × tanh(C_1)
    = 0.679 × tanh(0.376)
    = 0.679 × 0.360
    = 0.244       ← 短期记忆 / 当前输出
```

**第 1 步完成**：
```
C_1 = 0.376    （长期记忆：累积了"帧1的近距离+中血量"信息）
h_1 = 0.244    （短期记忆/输出）
```

### 第 2 步（t=2, x_2 = [0.8, 0.2]）

**遗忘门 f_2**：

```
f_2 = σ(0.5×0.8 + 0.2×0.2 + 0.1×0.244 + 0.1)
    = σ(0.4 + 0.04 + 0.0244 + 0.1)
    = σ(0.5644)
    ≈ 0.637    ← 保留 63.7% 的旧记忆
```

**输入门 i_2**：

```
i_2 = σ(0.4×0.8 + 0.3×0.2 + 0.2×0.244 + 0.1)
    = σ(0.32 + 0.06 + 0.0488 + 0.1)
    = σ(0.5288)
    ≈ 0.629
```

**候选细胞内容 C̃_2**：

```
C̃_2 = tanh(0.3×0.8 + 0.5×0.2 + 0.15×0.244 + 0.1)
     = tanh(0.24 + 0.10 + 0.0366 + 0.1)
     = tanh(0.4766)
     ≈ 0.444
```

**输出门 o_2**：

```
o_2 = σ(0.6×0.8 + 0.1×0.2 + 0.25×0.244 + 0.1)
    = σ(0.48 + 0.02 + 0.061 + 0.1)
    = σ(0.661)
    ≈ 0.659
```

**更新 C_2**：

```
C_2 = f_2 × C_1 + i_2 × C̃_2
    = 0.637 × 0.376 + 0.629 × 0.444
    = 0.2395 + 0.2793
    = 0.5188

观察：长期记忆从 0.376 增加到 0.5188，
     因为"近距离+低血量"持续出现 → LSTM 正在"加深印象"
```

**更新 h_2**：

```
h_2 = o_2 × tanh(C_2)
    = 0.659 × tanh(0.5188)
    = 0.659 × 0.477
    = 0.314
```

**第 2 步完成**：
```
C_2 = 0.5188
h_2 = 0.314
```

### 第 3 步（t=3, x_3 = [0.3, 0.8]）

**遗忘门 f_3**：

```
f_3 = σ(0.5×0.3 + 0.2×0.8 + 0.1×0.314 + 0.1)
    = σ(0.15 + 0.16 + 0.0314 + 0.1)
    = σ(0.4414)
    ≈ 0.609    ← 保留 60.9% 的旧记忆
```

**输入门 i_3**：

```
i_3 = σ(0.4×0.3 + 0.3×0.8 + 0.2×0.314 + 0.1)
    = σ(0.12 + 0.24 + 0.0628 + 0.1)
    = σ(0.5228)
    ≈ 0.628
```

**候选细胞内容 C̃_3**：

```
C̃_3 = tanh(0.3×0.3 + 0.5×0.8 + 0.15×0.314 + 0.1)
     = tanh(0.09 + 0.40 + 0.0471 + 0.1)
     = tanh(0.6371)
     ≈ 0.563
```

**输出门 o_3**：

```
o_3 = σ(0.6×0.3 + 0.1×0.8 + 0.25×0.314 + 0.1)
    = σ(0.18 + 0.08 + 0.0785 + 0.1)
    = σ(0.4385)
    ≈ 0.608
```

**更新 C_3**：

```
C_3 = f_3 × C_2 + i_3 × C̃_3
    = 0.609 × 0.5188 + 0.628 × 0.563
    = 0.3160 + 0.3536
    = 0.6696
```

**更新 h_3**：

```
h_3 = o_3 × tanh(C_3)
    = 0.608 × tanh(0.6696)
    = 0.608 × 0.584
    = 0.355
```

### 三步汇总表

| t | x_t | f_t | i_t | C̃_t | o_t | C_t | h_t |
|---|---|---|---|---|---|---|---|
| 1 | [1.0, 0.5] | 0.668 | 0.657 | 0.572 | 0.679 | **0.376** | **0.244** |
| 2 | [0.8, 0.2] | 0.637 | 0.629 | 0.444 | 0.659 | **0.519** | **0.314** |
| 3 | [0.3, 0.8] | 0.609 | 0.628 | 0.563 | 0.608 | **0.670** | **0.355** |

### 关键观察（这段一定要读）

```
观察 1：C_t 是"累积性"的
  C_1 = 0.376 → C_2 = 0.519 → C_3 = 0.670
  即使 x_t 在变化，C_t 保持在一个"有记忆"的范围
  因为：C_t = f_t·C_{t-1} + ...
        当 f_t 接近 1 时，C_{t-1} 几乎全部保留

观察 2：h_t 是 C_t 的"过滤版"
  h_t = o_t × tanh(C_t)
  o_t 决定"要不要暴露当前记忆"
  
观察 3：门的值决定"信息流强度"
  f_t ≈ 0.6 → 保留 60% 旧记忆
  i_t ≈ 0.63 → 接受 63% 新信息
  网络通过训练调整这些门，学会"该记什么、该忘什么"

观察 4：梯度不消失的关键
  C_3 通过 C_t = f_t · C_{t-1} + ... 这一加法路径，
  和 C_1 直接"打通"。
  梯度从 Loss 能通过这条加法路径回传到 C_1，不衰减。
```

## 3.5 为什么 LSTM 能缓解梯度消失

用一句话解释：

```
LSTM 让梯度可以通过"加法路径"（C_t = f_t · C_{t-1} + 新内容）
几乎无衰减地穿过时间。

与 RNN 的本质区别：
  RNN:  h_t = tanh(W · h_{t-1} + ...)   ← 只有乘法路径
  LSTM: C_t = f_t · C_{t-1} + ...       ← 有加法路径！
```

但 **LSTM 不完美**：

```
1. 加法路径要生效，需要 f_t（遗忘门）接近 1
   → 如果 f_t 经常接近 0，长期记忆仍然会丢失

2. 参数量是 RNN 的 4 倍（4 套门的权重）
   → 训练慢，占内存

3. 仍然是串行处理
   → 必须 t=1 → t=2 → t=3，不能并行
   → 训练速度被限制

4. 极长序列（>1000 步）仍然困难
```

这些问题最终驱动了 Transformer 的出现。

## 3.6 GRU —— LSTM 的简化版（一带而过）

**GRU（Gated Recurrent Unit / 门控循环单元）** 由 Cho et al. 在 2014 年提出，是 LSTM 的简化版：

```
LSTM 有 3 个门（f, i, o）+ 细胞状态 C
GRU 只有 2 个门（r 重置门，z 更新门），没有单独的 C

特点：
  ✅ 参数量更少（比 LSTM 少 25%）
  ✅ 训练更快
  ✅ 小数据集上效果往往和 LSTM 相当
  
  ⚠️ 超大数据集上 LSTM 略优
  
现代用法：
  NLP 领域：已被 Transformer 取代
  小数据、低延迟场景：GRU 仍在使用
```

**对你的实际意义**：知道"有这么个东西 = LSTM 的轻量版"就够了。

## 4.1 动机：翻译任务的困境

**Seq2Seq（Sequence-to-Sequence / 序列到序列）** 架构由 Sutskever et al. 在 2014 年提出。

### 核心痛点：输入和输出长度不一样

```
场景：机器翻译

输入（中文）：  "我    爱    猫"             ← 3 个词
输出（英文）：  "I  love  cats  ."           ← 4 个词（标点也算）

标准 RNN 的模式：
  many-to-many（等长）：输入 3 步 → 输出 3 步
  但翻译要求"3 步输入 → 4 步输出"
  
这种"输入输出不等长"的任务，RNN 不能直接做！

类似的任务：
  - 对话生成（问题短，回答长）
  - 文本摘要（原文长，摘要短）
  - 图像字幕（输入 1 张图，输出一句话）
  - 语音识别（音频 1 秒 50 帧，文字 3 个字）
  - 动画生成（文本 10 个字，动画 60 帧）
```

### Seq2Seq 的解法：编码器 + 解码器

**直觉类比：翻译官的工作流**

```
翻译官翻译一句话，不是逐词对译，而是：

第 1 步（编码 / Encode）：
  听完整句原文，在脑子里形成"完整意思"（一个抽象概念）

第 2 步（解码 / Decode）：
  基于这个"完整意思"，用目标语言组织一句话
  一个词一个词地说出来

这就是 Seq2Seq 架构。
```

## 4.2 编码器-解码器架构

```
                ┌─────────── Seq2Seq ───────────┐
                │                               │
  "我 爱 猫"    │                               │   "I love cats"
      │         │                               │         ↑
      ↓         │                               │         │
  [编码器 RNN]  ───→  [Context 向量 c]  ───→   [解码器 RNN]
      │                                         │
  ↓   ↓   ↓                                ↓    ↓    ↓    ↓
 x1  x2  x3                              y1    y2    y3   y4
                                        "I" "love" "cats" "."
```

### 编码器（Encoder）

```
职责：把输入序列压缩成一个"上下文向量"

工作流程：
  逐步读入输入序列 "我", "爱", "猫"
  每一步更新隐藏状态 h_enc
  最后一步的 h_enc 就是"上下文向量 c"（Context Vector）

  h_0 → h_1 → h_2 → h_3 = c
         │     │     │
        "我"  "爱"  "猫"

c 包含了整个输入的压缩信息
```

### 解码器（Decoder）

```
职责：从上下文向量生成输出序列

工作流程：
  用 c 作为初始状态
  以 <START> 作为第一个输入
  每一步输出一个词，并把它作为下一步的输入

  h'_0 = c           ← 用 c 初始化
  输入: <START>       → 输出: "I"
  输入: "I"           → 输出: "love"
  输入: "love"        → 输出: "cats"
  输入: "cats"        → 输出: "."
  输入: "."           → 输出: <END>  ← 停止

这种"上一步的输出作为下一步的输入"的模式叫
  自回归生成（Autoregressive Generation / 自回归生成）
  
这是一个非常重要的概念！后面 GPT / MotionGPT 都是自回归的。
```

### 两种 RNN 的角色

```
编码器：
  many-to-one 模式
  每步读入 → 更新隐藏状态
  最后一步的隐藏状态 = Context Vector

解码器：
  one-to-many 模式
  Context Vector 作为初始状态
  自回归地生成输出序列
```

## 4.3 Context Vector 的深入理解

### 它到底是什么？

```
Context 向量 c（通常 256 维或 512 维）：

  c = 编码器读完整句输入后的最终隐藏状态
    = 一个固定大小的向量
    = 整个输入序列的"压缩摘要"
    
把它想象成：
  "一本 500 页的书，压缩成一张 A4 纸的摘要"
  
解码器的任务：
  "读这张摘要，重新用另一种语言写一本新书"
```

### 这就暗示了 Seq2Seq 的最大缺陷

```
┌─────────────────────────────────────────┐
│  问题：信息瓶颈（Information Bottleneck）│
│                                         │
│  输入长度 ↑ → Context 向量大小不变       │
│                                         │
│  10 个词的句子：摘要成 512 维向量，够用  │
│  50 个词的句子：摘要成 512 维向量，紧张  │
│  200 个词的段落：摘要成 512 维向量，崩溃 │
│                                         │
│  就像把 500 页的书硬塞进一张便利贴       │
│  → 信息严重丢失                         │
└─────────────────────────────────────────┘
```

**实际实验结果**（Cho et al., 2014）：

```
BLEU Score（翻译质量，越高越好）：

句子长度 10 词：   BLEU = 32   ← 不错
句子长度 30 词：   BLEU = 28   ← 下降
句子长度 50 词：   BLEU = 22   ← 明显差
句子长度 70 词：   BLEU = 15   ← 基本不能用

长句子翻译质量急剧下降，就是信息瓶颈的后果。
```

## 4.4 信息瓶颈的启示

Seq2Seq 的信息瓶颈，**直接催生了注意力机制**（第 5 章）。

逻辑链是这样的：

```
问题：解码器只能看到 Context 向量 c，看不到原始输入
  → 对长输入，c 不够用

自然的想法：
  "能不能让解码器不止看 c，还能回头看输入序列的每一步？"
  "能不能让解码器在翻译每个词时，重点关注输入中对应的部分？"

例子：
  翻译 "I love cats" 的 "love" 时
  → 解码器应该重点看输入的 "爱"
  → 翻译 "cats" 时，重点看 "猫"
  → 翻译 "I" 时，重点看 "我"

这就是"注意力"的雏形。
```

## 4.5 Seq2Seq 在动画里的启发

```
文本 → 动画（Text-to-Motion）：
  编码器：Transformer 读 "a person walks forward"
         → 得到文本的 Context
  解码器：自回归地生成动画帧
         每帧基于 Context + 前面生成的帧

动画 → 文本（Motion-to-Text）：
  编码器：Transformer 读一段动画
         → 得到动作的 Context
  解码器：自回归生成描述 "a person is dancing"

音乐 → 舞蹈（Music-to-Dance）：
  编码器：读音频特征
  解码器：生成舞蹈动作

这些都是 Seq2Seq 架构的现代变体，
只是把 RNN 换成了 Transformer。
```

**代表工作**：
- **T2M-GPT**（Zhang et al., 2023）：文本到运动的 GPT 式生成
- **Bailando**（Li et al., 2022）：音乐到舞蹈的生成

# 第5章：注意力机制的诞生（衔接 Transformer）

![fig7](/images/RNN-LSTM-Seq2Seq-Prerequisite/fig7.png)

## 5.1 Bahdanau 注意力的动机

2014 年，Bahdanau et al.（ICLR 2015）提出：

> 与其让编码器压缩成一个向量 c，不如**保留编码器每一步的隐藏状态**，让解码器根据当前需要**选择性地查看**这些状态。

### 架构对比

```
传统 Seq2Seq：
  编码器输出: 只用最后一步 h_enc^T = c
  解码器输入: 固定的 c
  
Bahdanau 注意力：
  编码器输出: 保留所有时间步 h_enc^1, h_enc^2, ..., h_enc^T
  解码器每一步：
    1. 看所有 h_enc^i
    2. 计算权重 α_i（对每个 h_enc^i 的关注度）
    3. 加权求和得到 c_t（动态的上下文，随时间变化！）
    4. 用 c_t 生成当前输出
```

### 图示

```
编码器输出（5 个隐藏状态）：
  h^1   h^2   h^3   h^4   h^5
   │     │     │     │     │
   └──┬──┴──┬──┴──┬──┴──┬──┘
      ↓     ↓     ↓     ↓
      对每个状态计算权重 α_1, α_2, α_3, α_4, α_5
      （权重和 = 1）
      
      加权求和: c_t = α_1·h^1 + α_2·h^2 + ... + α_5·h^5
      
解码器第 t 步:
  → 用 c_t 生成当前输出
  → 不同的 t 有不同的 α，从而得到不同的 c_t
```

## 5.2 "回看"机制的直觉

```
翻译 "the cat sat on the mat" → "猫 坐 在 垫子 上"

翻译第 1 个中文词 "猫" 时：
  注意力权重应该高度集中在英文的 "cat"
  α = [0.05, 0.8, 0.05, 0.03, 0.05, 0.02]
             ↑ 高权重对应 "cat"

翻译第 2 个中文词 "坐" 时：
  α = [0.02, 0.05, 0.85, 0.03, 0.02, 0.03]
                   ↑ 高权重对应 "sat"

翻译第 3 个中文词 "在" 时：
  α = [0.02, 0.02, 0.05, 0.80, 0.05, 0.06]
                         ↑ 高权重对应 "on"

→ 注意力自动学会"对齐"原文和译文！
```

## 5.3 从注意力到 Transformer

Bahdanau 注意力是**加在 RNN 上**的（RNN 仍然是主体）。但人们逐渐发现：

```
2014-2017 之间的技术演进：

2014 Bahdanau 注意力：RNN + Attention
  ↓
  发现：Attention 很强大，但 RNN 是瓶颈（慢、难训）
  ↓
2017 Transformer：完全丢掉 RNN，只用 Attention
  "Attention Is All You Need"

关键洞察：
  如果 Attention 可以让解码器看编码器的所有步骤，
  为什么不让每一步都可以看所有其他步骤？
  → 自注意力（Self-Attention）
```

这就是你接下来要读的 Transformer 文档的起点。

### 参考答案（建议自答后再对照）

```
1. 全连接层处理动画序列的问题：
   - 输入长度固定（训练时 60 帧，推理时 120 帧就崩溃）
   - 参数量爆炸（3960 × 1024 = 400 万参数）
   - 不利用时间局部性（相邻帧相关性强是常识，但模型要从零学）

2. 区别：
   - 隐藏状态 h_t 是"累积性"的——它包含了所有历史信息的压缩
   - 特征图是"局部性"的——它只反映卷积核看到的那个窗口
   - h_t 的意义依赖于前面所有时间步；特征图的意义只依赖于窗口内的像素

3. 梯度消失原因（一句话）：
   BPTT 时，梯度沿时间反传每一步都要乘 W_hh 和 tanh'，
   连乘 T 次后指数衰减到 0。

4. LSTM 保护长期记忆：
   关键是 C_t = f_t · C_{t-1} + i_t · C̃_t 这个加法更新公式。
   当 f_t 接近 1 时，C_{t-1} 几乎完全保留，
   梯度 ∂C_t/∂C_{t-1} ≈ 1，不会衰减。
   和 ResNet 的 "h = x + f(x)" 是同一思想：用加法路径保护梯度。

5. 三个门：
   - 遗忘门 f_t：决定丢掉多少旧的长期记忆（0=全忘，1=全记）
   - 输入门 i_t：决定接受多少新信息进入长期记忆
   - 输出门 o_t：决定从长期记忆输出多少到短期记忆
   用 sigmoid 因为它输出 0~1，正好是"门的开合比例"；
   tanh 输出 -1~1 有负值，不适合做"开关"；
   ReLU 输出无上界，也不适合。

6. Seq2Seq 适合翻译：
   因为输入输出可以不等长。
   编码器 many-to-one 吃完整句，解码器 one-to-many 生成新长度的输出。
   标准 RNN 只能 many-to-many 等长，处理不了"3 个中文词 → 4 个英文词"。

7. 信息瓶颈：
   编码器把整个输入压缩成一个固定大小的 Context 向量（如 512 维）。
   输入越长，这个向量装的"信息密度"越高，直到装不下。
   实验表明：50 词以上的句子翻译质量急剧下降。

8. 演进链：
   RNN 解决"如何处理变长序列"，但有梯度消失；
   LSTM 用门控保护长期记忆，解决梯度消失；
   Seq2Seq 用编码器-解码器处理"输入输出不等长"，但有信息瓶颈；
   Attention 让解码器动态回看编码器，解决信息瓶颈；
   最后 Transformer 把 RNN 完全扔掉，只保留 Attention。
```

## 附录 A：LSTM 与 RNN 梯度对比（可选深入）

如果你想更深入理解"为什么 LSTM 能缓解梯度消失"，可以看这个对比：

```
RNN:  h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b)
  反向传播: ∂h_t/∂h_{t-1} = W_hh · tanh'
  连乘 T 次: ∏ W_hh · tanh'   
  → 如果 W_hh 的特征值 < 1：指数衰减到 0
  → 如果 W_hh 的特征值 > 1：指数爆炸

LSTM: C_t = f_t · C_{t-1} + i_t · C̃_t
  反向传播: ∂C_t/∂C_{t-1} = f_t   (一阶近似，忽略 f_t 本身对 C_{t-1} 的依赖)
  连乘 T 次: ∏ f_t
  → f_t 是 sigmoid 输出，取值 0~1
  → 但训练后 f_t 通常接近 1（因为网络倾向于保留记忆）
  → 乘积不会指数衰减，信息可以长距离传递

关键数学差异：
  RNN 的传播因子是 W_hh · tanh'（可以是任何值）
  LSTM 的传播因子是 f_t（网络学到的、通常接近 1 的值）
```
